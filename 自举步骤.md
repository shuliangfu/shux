## 0. 你这一波要干什么（先选题，30 秒）

读 `analysis/自举进度.md` §3 前排。默认做 **R2 真迁**（一个 rest 模块）。

## 2. R2 模板（默认主线，把 `<MOD>` 换成模块名）

### 2.1 改哪些文件

| 文件 | 做什么 |
|------|--------|
| `compiler/src/runtime/<mod>.x`（或 labi 路径） | 业务符号 `#[no_mangle]` 完整实现 |
| `compiler/seeds/<mod>.from_x.c` | `#ifndef SHUX_*_FROM_X`：冷启动全 C；`#else`：前向声明 + `labi_*_slice_marker` |
| `compiler/seeds/<mod>_surface.from_x.c` | 用 `shux -E` 产物抛光生成，**勿手写第二套语义** |
| `tests/prove_module_selfhost.sh` | 注释标明 R2 full / H=0 |
| `compiler/scripts/g05_ensure_relink_prereqs.sh` | 多数只改注释；已 PREFER 则拓扑可不动 |

**常见坑（根源修，禁补丁）**：

| 坑 | 做法 |
|----|------|
| `slice` / 指针难迁 | 用 `*_buf(data,len)` |
| `va_list` / `diag_reportf` | 改 `diag_report_with_code` 固定 msg |
| 长字符串 `*u8` | 栈 `u8[N]` 字节赋值 |
| 全局 BSS 大数组 | **先 Cap-global-bss，勿硬迁** |

参考已闭合：`rt_fs_open`、`rt_parse_diag`。

### 2.2 本机验证（mac，commit 前）

```bash
MOD=<mod>          # 例：rt_parse_diag
MACRO=SHUX_RT_PARSE_DIAG_FROM_X   # 与 seed 里宏一致

cd compiler
SHUX_BIN=./shux; [ -x "$SHUX_BIN" ] || SHUX_BIN=./shux-c

# A. -E 出 C（须 exit 0、有业务函数体）
perl -e 'alarm 45; exec @ARGV' "$SHUX_BIN" -E \
  -L .. -L src/lexer -L src/ast -L src/parser -L src/typeck \
  -L src/codegen -L src/lsp -L src/driver -L src/preprocess \
  src/runtime/${MOD}.x 2>/tmp/${MOD}_e.err | grep -v '^DBG-' > /tmp/${MOD}_e.c
# labi 等模块：路径见 prove 表 或 analysis/自举进度.md

# B. 抛光 surface（头：POSIX prologue，去 -E 自带 #include）
# → 写入 seeds/${MOD}_surface.from_x.c

# C. FROM_X rest：业务 T=0（只剩 marker）
cc -Wall -Wextra -I. -Iinclude -Isrc -D${MACRO} -c -o /tmp/rest.o seeds/${MOD}.from_x.c
nm /tmp/rest.o   # 期望：仅 labi_*_slice_marker

# D. 冷启动：无宏仍有业务 T
cc -Wall -Wextra -I. -Iinclude -Isrc -c -o /tmp/cold.o seeds/${MOD}.from_x.c
nm /tmp/cold.o

# E. hybrid
cc -c -o /tmp/thin.o /tmp/${MOD}_e.c   # 或 surface 编译
cc -r -nostdlib -o /tmp/merged.o /tmp/thin.o /tmp/rest.o && nm /tmp/merged.o && echo SMOKE_OK

# F. prove
cd ..
bash tests/prove_module_selfhost.sh 2>&1 | tee /tmp/prove.txt | tail -40
# 该模块 IDENTICAL；全量 DIFF=0
```

## 3. Cap 模板（能力解锁，一句话版）

1. 定位根因在 **IR / typeck / codegen / glue 实际被链的实现**（见 AGENTS.md，禁下游补丁）。  
2. 修前红 → 修后绿；`prove` / `diff_single_link` 不回退。  
3. 结束时写清：**立刻可 R2 的模块列表**（最好同波或紧随做 R2）。  
4. 进度文 §8 记 Cap 名 + 解锁面；KPI 以真迁为准，不单靠 N。

---

## 4. M 模板（mega，一句话版）

1. 目标：`typeck` / `pipeline` 等 pin 的 `*_x.o` 能否被真实 `.x` 产物替换。  
2. 先 smoke 再产品；失败写阻塞（能力缺口 → 转 Cap）。  
3. 与 R2 **不互等**：M 绿 ≠ rest H 下降。
