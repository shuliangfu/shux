## 5. 架构性技术债（A-10 之后）

1. **import 内联重复**：每个引用 `core.mem` 的 `.sx` 各编一份 `mem_copy`/`mem_set`/… → 靠 `ld --allow-multiple-definition` 合并；长期应 **单独编 `core/mem.o` 再链接**。
2. **模块顶层 let 语义**：hoist 仅首函数 + const 立即数回落；**可变全局**（如原 `g_mem_fence_seq`）需 `.bss` 或统一 module-global 槽位模型。
3. **`pipeline_glue_standalone.o` 置首链接**：while/assign 等修复依赖此顺序；seed 文档化不足易回归。
4. **G-06 partial pin**：`asm_backend_partial.linux.x86_64.o` 未稳定入库 seeds/。
5. **G-01**：core 零 C 文件删除未按单文件 commit。

### 6.1 改代码后的推荐顺序

1. **Mac 本机**（分钟级）：relink → 相关单测 / `check` / hello 烟测  
2. **Docker Linux**（提交前）：`SHUX_DOCKER_TIMEOUT_SEC=600 ./tests/run-linux-a09-a11-gate.sh` 或 `SHUX_W3_RESUME_FROM=ensure …/gate-inner.sh`  
3. **Mac 再跑一遍** `run-bootstrap-crossplatform-gate.sh`（矩阵里 `macos_must` 项）

**纪律**：Linux Stage2 hash **只在 Docker 内**对比；darwin 种子 **只在 macOS 上**生成，不与 Linux hash 混比。

### 6.2 种子（当前阶段）

| 产物 | 何时刷 | 在哪刷 |
|------|--------|--------|
| `*_gen.*.c`（18 个） | 大改前端 `.sx` 且 asm 编不过 | Mac 或 Docker 均可（源码共用） |

| `bootstrap_shuxc.darwin.arm64` | 宣称 macOS 冷启动 / LEGACY 链变更 | **本机 Mac**（**现在不必专门刷**，W3 后再说） |

FreeBSD / linux-arm64 等：**未到阻塞项**，workflow 已备好，不必现在跑。

### 6.3 测试纪律（避免风扇狂转）

1. **Docker 命令一律带超时**：`SHUX_DOCKER_TIMEOUT_SEC=600 ./tests/lib/docker-linux-run.sh '...'`（超时 SIGTERM，+30s SIGKILL）。
2. **单条 compile/check**：gate 内 `SHUX_CASE_TIMEOUT=120`（`run-lang-unsafe-gate.sh` 已用 `timeout` 包装）。
3. **勿在常驻容器里裸跑 `shux check` 大文件**；跑完用 `docker ps` 看 CPU，异常容器 `docker stop <name>`。
4. **勿用 `SHUX_DOCKER_PERSIST=1` 跑未设超时的长任务**；ephemeral `docker run --rm` 更安全。
