# 自举方法（证明驱动 · 逐模块退役）

> **编写时间**：2026-07-13
> **目标**：用最小投入、最短路径完成 SHUX 自举，且每一步可证明无回归
> **金标准**：Linux x86_64；macOS 仅用于开发与 commit

---

## 0. 一句话结论

**逐模块自举 + 差分验证：让 `shux -E` 能处理 `compiler/*.x`，每完成一个模块立即退役对应 seed C，直到 seed 全部消失。**

自举的证明不是 `seed == .x`，是 `shux_v1 == shux_v2`（v1 用 seed 编译 .x，v2 用 v1 编译 .x，bit-identical）。

---

## 1. 自举的真正证明

```
seed_shux -E compiler/*.x   →  shux_v1
shux_v1   -E compiler/*.x   →  shux_v2
shux_v2   -E compiler/*.x   →  shux_v3
assert shux_v2 == shux_v3   →  自举完成，seed 可删
```

关键洞察：
- **seed 只用于第一次启动**，之后是 .x 自己编译自己
- **不需要 seed 完美等价 .x**，只需要 seed "足够好"能把 .x 编译出来
- **双权威是迁移中间态的必要代价**，但必须逐模块消除，不能永久维护

---

## 2. 当前低效根因

| 现状 | 代价 |
|------|------|
| 把精力花在"让 seed C 完美同步 .x" | 线性还债，每修一个 bug 省 1 处，但 seed 还在 |
| 修 bug 在 seed 层（parser_gen.c 等） | 治标，.x 层根因未解，下个 bug 还要同步两处 |
| 没有差分 harness | 漂移靠人发现，bug 在 std/ 暴露、根因在 compiler/ |
| weak/强符号双版本（macOS vs Linux） | 跨平台行为分叉隐蔽，本次 typeck bug 即此 |
| `-E-extern` 坏，手动同步 seed C | 人为错误空间大，本次 parser unsafe handler 即此 |

**核心问题**：在"全量同步 seed"而不是"逐模块退役 seed"。

---

## 3. 高效方法：逐模块自举

### 3.1 原则

1. **改 .x，不改 seed**：bug 修在 .x 层，seed 只是 .x 的历史快照
2. **差分验证**：`shux -E module.x` 的输出 vs seed `module_gen.c`，diff 为零才合格
3. **立即退役**：模块差分为零 → 该模块 seed 从构建链删除，不再手动同步
4. **从简单到复杂**：先自举最简单的 .x，逐步扩展到 parser/typeck/codegen

### 3.2 为什么这比"全量同步 seed"高效

| 全量同步 seed | 逐模块自举 |
|--------------|-----------|
| 每个模块都要同步两处 | 每完成一个模块，减一处同步负担 |
| 进展不可见（永远在还债） | 每步有明确进展（模块退役计数） |
| bug 在 std/ 暴露，根因在 compiler/ | bug 在差分时立即暴露，定位精确 |
| 双权威永久存在 | 单权威逐步建立 |

---

## 4. 标准迁移闭环（每个模块）

每个 `compiler/src/*.x` 模块自举必须走完这 7 步：

```
1. 选模块         → 选一个 compiler/src/*.x
2. shux -E        → shux -E module.x > /tmp/gen.c
3. 差分对比        → diff /tmp/gen.c compiler/module_gen.c
4. 修 .x + 修 shux → diff 非空则修，直到 diff 为空
5. 构建切换        → Makefile 改用 shux -E 生成，不再用 seed
6. seed 退役       → compiler/module_gen.c 从构建链删除
7. 跨链验证        → macOS + Ubuntu 都通过 run-all-bstrict.sh
```

**硬约束**：第 3 步 diff 不为零，禁止进入第 5 步。

---

## 5. 模块优先级（从简到繁）

| 优先级 | 模块 | 难度 | 候选理由 |
|--------|------|------|---------|
| 1 | `runtime_driver_diagnostic.x` | 低 | 大量简单 wrapper，无复杂控制流 |
| 2 | `lexer.x` | 中低 | 状态机，无递归 |
| 3 | `typeck.x` | 中高 | 有递归，但结构相对规整 |
| 4 | `parser.x` | 高 | 最复杂，最后处理 |
| 5 | `codegen.x` | 高 | 依赖前面所有模块 |

**策略**：先做 1-2 个简单模块，验证闭环可行；再逐步推进复杂模块。

---

## 6. 工具链（最小可行投入）

### P0：必须先建（没这些不做迁移）

#### 6.1 `diff_single_link.sh` — 单链差分

```bash
# 对每个 compiler/src/*.x：
#   shux -E module.x 的输出 vs seed module_gen.c
#   diff 非空 → 该模块尚未自举
#   diff 为空 → 该模块可退役 seed
```

**作用**：给自举进度画地图，知道哪些模块已自举、哪些还差。

#### 6.2 `diff_cross_link.sh` — 跨链差分

```bash
# 同一组 .x 测试程序：
#   macOS 链 shux -E test.x > /tmp/mac.out
#   Linux 链 shux -E test.x > /tmp/linux.out  (ssh ubuntu-server)
#   diff /tmp/mac.out /tmp/linux.out
#   非空 → 跨平台行为分叉（如 weak/强符号差异）
```

**作用**：抓住本次 typeck weak seed bug 这类跨链分叉。

#### 6.3 `prove_module_selfhost.sh` — 单模块自举验证

```bash
# 验证某模块完成 7 步闭环：
#   1. shux -E module.x 成功
#   2. cc -c 生成 .o 成功
#   3. nm 导出符号与 seed 一致
#   4. ld -r 合并无重复/无缺失
#   5. 跨链差分为空
#   6. run-all-bstrict.sh 通过
```

### P1：迁移过程中补建

- `tu_contract.json`：每个 TU 的 ABI manifest（symbol / 参数 / 返回 / struct size）
- `abi_fingerprint.sh`：ABI 指纹，删除 seed 前验收用

### P2：自举完成后

- `bootstrap_verify.sh`：v1 == v2 == v3 的 bit-identical 验证

---

## 7. seed 删除门槛

某模块 seed C 允许物理删除，必须**同时**满足：

1. `shux -E module.x` 稳定通过
2. 单链差分连续 **N=3** 次为空
3. 跨链差分（macOS vs Linux）为空
4. `nm` 导出符号与合同一致
5. `ld -r` 合并无重复/无缺失/无弱桩误覆盖
6. `run-all-bstrict.sh` 全绿
7. 该 seed 不再承担 rest 职责

**缺一不可。** 删除前必须在 macOS commit + Ubuntu 验证。

---

## 8. ABI 收敛原则

1. **单权威**：每个 ABI 面只能有一个实现，禁止 `.x` 一套、seed 一套、硬编码头再一套
2. **收敛不复制**：所有 ABI 声明从单一来源生成，禁止手动复制
3. **fingerprint 可解释**：ABI 指纹必须能定位到具体 symbol/参数/struct 变化，不是黑盒 hash
4. **weak/强符号消除**：自举完成的模块不允许有 weak/强双版本，只留一个

---

## 9. 执行顺序

### Phase 0：建基础设施（不迁模块）
1. 建 `diff_single_link.sh`，跑一次，画自举进度地图
2. 建 `diff_cross_link.sh`，跑一次，找跨链分叉
3. 建 `prove_module_selfhost.sh` 框架

### Phase 1：自举最简单模块
- 选 `runtime_driver_diagnostic.x`
- 走 7 步闭环
- seed 退役
- 验证：跨链 + run-all-bstrict

### Phase 2：逐模块扩展
- 按优先级 2-5 推进
- 每个模块走同一闭环
- 每完成一个，构建链减一个 seed

### Phase 3：全部模块自举
- 所有 `compiler/*.x` 都能用 `shux -E` 处理
- seed C 全部退役
- 构建链纯 .x

### Phase 4：自举验证
- `seed_shux -E *.x → v1`
- `v1 -E *.x → v2`
- `v2 -E *.x → v3`
- `v2 == v3` → 自举完成

---

## 10. 禁止事项

1. **禁止修 seed bug 而不修 .x** — bug 必须回溯 .x 根源
2. **禁止双权威长期并存** — 模块差分为零后必须立即退役 seed
3. **禁止无差分验证就退役 seed** — 7 步闭环缺一不可
4. **禁止跨平台行为分叉** — weak/强符号差异必须消除
5. **禁止把临时 scp 当正式构建链** — macOS → Ubuntu 只走 git
6. **禁止为"纯 .x"引入新双权威** — 收敛不复制
