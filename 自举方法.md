# 自举方法（证明驱动 · 分层退役 · 根因波次）

> **更新**：2026-07-14（对照 `5d483df2` STRING_LIT / socket 修复与现有 harness）  
> **目标**：最短路径、可证明无回归地完成 SHUX 自举；每一步有可执行判据  
> **金标准**：Linux x86_64；macOS 仅开发与 commit  
> **关联**：[`compiler/docs/SELFHOST.md`](compiler/docs/SELFHOST.md)（A/B/C/D 验收命令）、[`当前进度.md`](当前进度.md)

## 0. 一句话结论

**先修 typeck / ABI 根因波次（一次解锁 N 模块），再按「.o 符号等价」退役叶子 seed，大模块（parser/typeck/codegen）以 Stage2 行为等价收口，最终 `shux_v2 == shux_v3`。**

不要用「C 文本 diff 为空」当大模块的唯一门禁——那是可选的洁癖目标，不是最短自举路径。

---

## 1. 自举的真正证明（未变）

```
seed_shux -E compiler/*.x   →  shux_v1
shux_v1   -E compiler/*.x   →  shux_v2
shux_v2   -E compiler/*.x   →  shux_v3
assert shux_v2 == shux_v3   →  语义自举成立；再按 D-03 做 bit-identical（可选更强）
```

| 命题 | 含义 |
|------|------|
| seed 的职责 | **仅冷启动**，不是长期权威 |
| seed 与 .x 不必文本等价 | 只要 seed 能稳定编出正确 v1 |
| 双权威 | 迁移期不可避免；**模块退役后必须消掉** |
| 仓库内「黄金自举」 | 见 SELFHOST：**Stage2 哈希 + portable 行为**（D-01～D-04） |

**与 D 阶段关系**：本文件的「逐模块退役」是 **E（编译器主体去 seed C）** 的执行法；D（Stage2）已部分闭合时，E 仍要继续拆 pinned seed。

## 2. 方法是否合理？——评审结论

### 2.1 合理且应保留

| 原则 | 为什么对 |
|------|----------|
| 证明驱动、非口号 | 有 harness 才有地图 |
| 逐模块退役 seed | 降低双权威维护面 |
| 修根因不打补丁 | Agents.md 根源治理 |
| 金标准 Linux | 与现 CI / 历史门禁一致 |
| 禁止盲删 seed | 删除门槛必要 |

### 2.2 原文档的低效点 / 过时假设（必须改）

| 原主张 | 问题 | 修正 |
|--------|------|------|
| **C 源码 `diff` 为空才算自举** | `parser`/`codegen` 常有数千行无害差分；永远卡在「还债同步」 | **主门禁改为 `.o`/`nm` 等价**（已有 `prove_module_selfhost.sh`）；文本 diff 降为可选洁癖 |
| **严格「改 .x 不改 seed」** | 冷启动链 **pinned seed** + `patch_typeck_gen_lang007.py`；strict_minimal ≠ 全量 glue | 热路径 bug 修在 **实际被链接的实现**（seed/patch/glue/.x 对齐）；.x 仍是语义权威，seed 是快照+补丁 |
| **优先级：diagnostic → lexer → typeck → parser** | 忽略已近退役的 **driver 叶子**（fmt/check/test…）；typeck 先于 parser 也不匹配依赖 | **按退役 ROI 排序**：叶子先；mega 按依赖与共享根因波次 |
| **P0 含未实现的 `diff_cross_link.sh`** | 工具写了却不存在，Phase 0 悬空 | 降为 P1；跨链先用 Ubuntu 金标准 + 同脚本复跑 |
| **每模块独立 7 步闭环** | 未利用「一处 typeck 修 → 多模块转绿」（如 STRING_LIT） | 增加 **根因波次（Wave）** 轨，优先于单模块洁癖 |
| **与 Stage2 / SELFHOST 脱节** | 两套证明语言并行 | 终局证明统一到 D/E gate；模块退役是 E 的子任务 |

### 2.3 当前真实拓扑（写方法时必须心里有数）

```
bootstrap-driver-seed
  ├── typeck_x.o / codegen_x.o / parser_x.o …   ← 多来自 pinned *_gen.c + patch
  ├── pipeline_glue_strict_minimal.o           ← 冷启动 glue（≠ 整份 pipeline_glue.c）
  └── shux / shux-c

日常 G-05 relink：再链用户 asm partial + 同步 shux-c
```

含义：

1. 改 `pipeline_glue.c` 不等于冷启动立刻生效，除非 strict_minimal / seed 同源也覆盖。  
2. `typeck.x` 逻辑完整 ≠ 链上跑的是 regen 后的 typeck；默认 **pinned + patch**。  
3. `diff_single_link` 用 `shux-c -E` 画地图：测的是 **「当前编译器能否吃下 .x」**，不是「seed C 是否已删」。

## 3. 高效方法：三轨并行（比「纯逐模块」快）

```
        ┌─────────────────────────────────────┐
 Track W│  根因波次（typeck/codegen/ABI）       │  一次修，N 模块 -E 转绿
        └─────────────────────────────────────┘
                        │ 解锁
        ┌───────────────┴───────────────┐
 Track L│ 叶子退役（driver/lsp 小 TU）    │  nm/.o 等价 → 删 seed 依赖
        └───────────────────────────────┘
                        │
        ┌───────────────┴───────────────┐
 Track M│ Mega 收口（parser/typeck/codegen）│ Stage2 / 链接合同，非文本 diff
        └───────────────────────────────┘
                        │
                    v2 == v3 / D-03
```

### Track W — 根因波次（最高 ROI）

**定义**：一类 typeck/codegen/解析错误，阻塞 **≥2 个** 模块或整个 std 路径。

| 波次示例 | 症状 | 修在哪 | 杠杆 |
|----------|------|--------|------|
| W-string | `expected *u8, found ?` on `m=""` | seed string_lit + glue | diagnostic / diag.x 等 |
| W-keyword-param | extern 丢声明（`type:` 参数） | 源码改名 | net/tcp 等 |
| W-unsafe-expr | `if (unsafe { f() } != 0)` 生成 void 比较 | 源码改 let+unsafe 赋值 | std/net 多处 |
| W-extern-unsafe | `extern call requires unsafe` | 调用点 / 边界表 | compile/lsp |
| W-tail | implicit tail return | 显式 return / region-tail patch | ast 等 |
| W-escape | struct stack escape | 真逃逸改写法或误报修 typeck | emit 等 |

**纪律**：

1. 先 `diff_single_link` 聚合 **错误类**，再选波次，不要从清单第 1 行盲修。  
2. 波次修复必须落在 **实际链接路径**（seed patch / strict_minimal / .x 语义一致）。  
3. 每波次结束：Ubuntu 冷启动 + 复跑地图 + 相关最小用例。

### Track L — 叶子模块退役（最快减 seed 负担）

**对象**：小 TU、少 import、已 typeck OK、`.o` 符号可对齐的模块。

现有证据（进度表 / `prove_module_selfhost` 覆盖）：

- 优先：`fmt` / `check` / `test` / `build` / `run`（driver 叶）  
- 次优：`token`、小 lsp helper（`lsp_io_std_heap` 等）  
- diagnostic：typeck 已解锁后走 **.o/nm**，不必死磕 C 文本 0 diff

**主判据（退役）**：

1. `shux-c -E module.x` 稳定 exit 0（Ubuntu）  
2. `prove_module_selfhost.sh <name>`：**nm 差分在允许 rename 表内为空**  
3. Makefile / g05 改为使用 `.x→.o` 路径，**构建不再依赖该 seed 文件**  
4. 冷启动 + G-FFI-5（或等价门禁）仍绿  

**C 文本 diff 为空**：加分项，**不是** Track L 硬门禁。

### Track M — Mega 模块（parser / typeck / codegen）

**不要**用「-E 输出 vs 整份 seed 文本」当主目标——输入/输出耦合、噪声爆炸。

**改用**：

| 层 | 判据 |
|----|------|
| M1 | 该 mega 作为 **源** 能被当前编译器 `-E` / typeck 通过（可 import 齐） |
| M2 | 用当前编译器 regen 出的 `*_x.o` **能替换** pinned `.o` 链接出 shux，并通过 smoke |
| M3 | Stage2：`bootstrap-verify-stage2-bstrict` / D-03 哈希（Linux） |
| M4 | 仅当 M2+M3 稳定后，再考虑物理删除 seed 与 pin |

Mega 期间允许 **pinned seed + 幂等 patch**（如 `patch_typeck_gen_lang007.py`），但 patch 必须：

- 幂等  
- 有最小回归用例  
- 注释写清对应 .x 语义（禁止第三套硬编码逻辑）

## 4. 标准闭环（按轨不同）

### 4.1 叶子退役闭环（Track L）— 6 步

```
1. 选模块（ROI 表 / 地图 OK 且 diff 小或 nm 可证）
2. -E 绿（Ubuntu）
3. prove_module_selfhost：.o + nm
4. 构建切换：g05 / Makefile 用 .x 产物
5. 删除「构建对 seed 的硬依赖」（文件可暂留仓库作考古）
6. 门禁：bootstrap-driver-seed + 相关 gate
```

### 4.2 根因波次闭环（Track W）— 5 步

```
1. 地图聚类错误（同一 typeck/codegen 消息 ≥2 处）
2. 最小复现（10 行 .x）
3. 修实际链接路径 + 对齐 .x 语义
4. 全图复跑 + Ubuntu
5. 写入 当前进度.md 已修复表
```

### 4.3 Mega 闭环（Track M）— 对齐 Stage2

```
1. -E/typeck 可消费源
2. 试替换 *_x.o 链接 smoke
3. Stage2 / D-0x gate
4. 去 pin / 去 patch（最后）
```

## 7. seed 删除门槛（分级）

### 7.1 叶子 seed（Track L）

同时满足即可 **构建链退役**（仓库文件可后删）：

1. Ubuntu：`shux-c -E` 稳定  
2. `prove_module_selfhost` nm 通过（含约定 rename）  
3. 构建路径不再读取该 seed  
4. 冷启动 + 相关 gate 绿  
5. 连续 **2** 次 Ubuntu 验证（比原 N=3 略松：叶子小，2 次足够；Mega 仍用更严）

### 7.2 Mega seed（Track M）

1. 替换 `*_x.o` 链接 smoke 绿  
2. Stage2 / D-03（Linux）绿  
3. pin 关闭后冷启动仍可从「上一代 shux -E」重建  
4. 无 weak/强双版本行为分叉  
5. patch 脚本可删或变为空操作  

### 7.3 物理删文件

构建链退役稳定 **≥1 周** 或 **≥3 次** 全绿 CI 后再删，避免回滚无种子。

## 8. ABI / 双权威纪律（增强版）

1. **语义权威**：`.x`  
2. **冷启动权威**：pinned seed + **唯一** 幂等 patch 入口（禁止手改 gen.c 当第三权威）  
3. **Glue 权威**：`strict_minimal` 与全量 glue **同语义**；修 typeck 热路径要标明落点  
4. **ExprKind 序数**：STRING_LIT=**59** 等以 parser_asm / backend 为准，禁止 `ast.h` 与 glue 各写各的  
5. **关键字参数**：禁止 `type` 等保留字作参数名（net socket 已证伪）  
6. **表达式 unsafe**：禁止 `if (unsafe { call() } != 0)` 依赖 statement-expression；统一 `let rc; unsafe { rc = call(); }`

5. **禁止**用 scp 代替 git 同步正式构建  
6. **禁止**为「纯 .x」复制第三套 ABI 表  
7. **禁止**把 lexer 超时和 typeck FAIL 绑在同一 PR 盲修
