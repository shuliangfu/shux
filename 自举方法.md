## 0. 一句话结论

**T 已满后，主线不再是「再证明一个 thin 叶子」；而是：一次能力解锁（Cap）→ 批量真迁 hybrid rest（R）→ mega 去 pin（M）→ `shux_v2 == shux_v3`。**

`prove_module_selfhost` 的 **N** 仍是地图，**不是** C-zero 进度本身。  
**真进度** = 产品链接路径上 **C/rest 字节下降** + **mega pin 关闭**。

**补一句（2026-07-15）**：**工程轨绿（R2/prove/return-value/Stage2 子集）≠ 产品轨绿（import+调用 `-o` + 严格 bstrict）**。  
没有产品轨全绿，**不得**说「可以自举 / 自举过了」。14 号验收混轨是 15 号雪崩的根因之一。详见 [自举步骤.md](自举步骤.md) §0.A～§0.C。

### 0.1 两条轨道（必须分清）

| 轨道 | 测什么 | 典型绿点 | 能否单独放行自举 |
|------|--------|----------|------------------|
| **工程轨** | hybrid 真迁、surface 证明、冷启动 seed、Stage2 子集 | prove IDENTICAL、R2 H=0、return-value 42、bootstrap-verify | **否** |
| **产品轨** | 当前 g05 链出的 `shux_asm` 编用户/测试程序 | option/hello/stdlib-import `-o`、**run-all-bstrict** 无假绿 env | **是（必要）** |

自举终局需要 **两轨都绿**；R2 可以只报工程轨，但进度文必须单列产品轨状态。

## 1. 自举的真正证明（未变）

```
seed_shux -E compiler/*.x   →  shux_v1
shux_v1   -E compiler/*.x   →  shux_v2
shux_v2   -E compiler/*.x   →  shux_v3
assert shux_v2 == shux_v3   →  语义自举成立；再按 D-03 做 bit-identical（可选更强）
```

| 命题 | 含义 |
|------|------|
| seed 的职责 | **仅冷启动**，不是长期权威 |
| seed 与 .x 不必文本等价 | 只要 seed 能稳定编出正确 v1 |
| 双权威 | 迁移期不可避免；**模块退役后必须消掉** |
| 仓库内「黄金自举」 | SELFHOST：**Stage2 哈希 + portable 行为**（D-01～D-04） |

模块退役是 **E（编译器主体去 seed C）** 的执行法；D（Stage2）部分闭合后 E 仍要继续。

## 2. 为何必须改方法：旧轨在 2026-07 中旬的失效点

### 2.1 仍正确、必须保留

| 原则 | 为什么对 |
|------|----------|
| 证明驱动、非口号 | 有 harness 才有地图 |
| 修根因不打补丁 | Agents.md 根源治理 |
| 金标准 Linux | 与 CI / 历史门禁一致 |
| 禁止盲删 seed | 删除门槛必要 |
| nm/.o 优于 C 文本 diff | 大模块噪声爆炸 |
| 语义权威在 `.x` | 冷启动 seed/patch 只是快照 |

### 2.2 已过时 / 低效的做法

| 旧习惯 | 问题 | 新纪律 |
|--------|------|--------|
| **T 未满时主推 Track L** | 合理；但 **T=18/18 已达成** | T 降为维护指标，不占前排 |
| **N++ 当主 KPI** | hybrid **thin IDENTICAL** 可在 rest 仍全 C 时刷分 | 主 KPI 改为 **R（真迁）/ H（hybrid 面）/ M_pin** |
| **产品已 PREFER_X_O 仍逐叶 surface prove** | 1 函数 thin 证明 ≈ 地图卫生，几乎不减产品 C | **prove 只服务真迁或未产品化的叶**；禁止为 N 而 N |
| **纯「逐模块 6 步」** | 未利用「一处能力修 → 多 rest 可迁」 | **Cap 波次优先于单模块洁癖** |
| **先把 runtime 全 thin 再碰 mega** | pin 的 typeck/codegen 是最大双权威；与 hybrid 无关 | **M 与 R 并行**，不互相阻塞 |
| **「构建去 seed」= 删文件** | 冷启动仍需 seeds；产品 PREFER 才是退役 | 区分 **产品链退役** / **冷启动考古** / **物理删除** |

### 2.3 当前真实拓扑（心里要有数）

```
bootstrap-driver-seed
  ├── typeck_x.o / codegen_x.o / parser_x.o …   ← 多来自 pinned *_gen.c + patch
  ├── pipeline_glue_strict_minimal.o             ← ≠ 整份 pipeline_glue.c
  └── shux / shux-c

G-05 PREFER_X_O（产品日常）:
  thin.x → .o  +  hybrid rest seed (-DSHUX_*_FROM_X)
  → ld -r → runtime_driver_no_c.o / labi 等
```

含义：

1. **产品 hybrid 已切 thin** ≠ rest 已 .x；N 涨、**H 不降** 时自举几乎没前进。  
2. 改 `pipeline_glue.c` ≠ 冷启动立刻生效，除非 strict_minimal / seed 同源覆盖。  
3. `typeck.x` 完整 ≠ 链上跑的是 regen 产物；默认 **pinned + patch**。  
4. `diff_single_link` 测的是「当前编译器能否吃下 .x」，不是「C 是否已删」。

## 3. 高效方法：四轨（Cap / R / L / M）

```
        ┌──────────────────────────────────────────┐
 Track Cap│  能力解锁（typeck/codegen/ABI/字面量/池） │  一次修，多 rest 可真迁
        └──────────────────────────────────────────┘
                        │ 解锁
        ┌───────────────┴───────────────┐
 Track R│  真迁退役（.x 吃满 或 rest C=0） │  产品链接上 C 真正下降
        └───────────────────────────────┘
                        │ 并行
        ┌───────────────┴───────────────┐
 Track M│  Mega 去 pin（typeck/codegen/parser）│ Stage2 / 替换 *_x.o
        └───────────────────────────────┘
                        │
 Track L│  叶子 prove / 未产品化薄叶          │  仅当 ROI 明确；不刷 N
        └────────────────────────────────────┘
                        │
                    v2 == v3 / D-03
```

### Track Cap — 能力解锁（最高 ROI，接替旧 Track W 的主位）

**定义**：一类语言/编译器能力缺口，阻塞 **≥2 个** 模块的 **真迁**（不是只阻塞 `-E` 地图）。

| 波次（当前前排示例） | 症状 | 解锁面 |
|----------------------|------|--------|
| Cap-empty-str | `""` / 空串字面量 typeck 或 codegen 缺口 | `rt_util` rest（argv0_basename_is）等 |
| Cap-string-pool | 串池 / slen 封顶（历史 ≤64） | 长字面量 rest、诊断串 |
| Cap-global-bss | `u8[N]` 全局 / static BSS 不能真迁 | `rt_arena_buf` rest 等 |
| Cap-va-reportf | `reportf` / va_list 更深路径 | diagnostic 全量、驱动日志 |
| Cap-regen-sync | .x 改了但 pin/patch 未同源 | 一切 mega 相关热路径 |

**纪律**：

1. 先按 **「卡 rest 真迁的错误类」** 聚类，再开波；不要字典序扫文件。  
2. 修复落在 **实际链接路径**（seed patch / strict_minimal / .x 语义一致）。  
3. 每波结束：Ubuntu 冷启动 + 相关最小用例 + **列出被解锁的 R 候选**。  
4. **一条 Cap 未做完，禁止并行开另一条同层 Cap**（避免双权威分叉）；不同域（如 string vs BSS）可并行仅当落点文件不冲突。

### Track R — 真迁退役（C-zero 的唯一直线）

**定义**：产品构建路径上，该 TU 的 **业务符号** 全部来自 `.x→.o`，**不再**依赖 hybrid rest 里的同名 C 体。

| 层级 | 含义 | 是否算 R 完成 |
|------|------|----------------|
| R0 hybrid thin | 产品 PREFER thin.x + rest C | ❌ 仅拓扑切片 |
| R1 surface prove | nm IDENTICAL 公共面 | ❌ 地图，非真迁 |
| **R2 true-migrate** | rest 符号迁入 .x 或删除；rest seed 可空/可删宏 | ✅ |
| R3 冷启动一致 | 冷路径也可用同构 surface/full .x 产物 | 加分；终局需要 |

**主判据（R2）**：

1. Ubuntu：相关 `-E` / g05 绿  
2. 产品 `*_FROM_X` rest 中 **该业务符号不再 `#ifndef` 提供 C 体**（或 rest 文件对该模块为空）  
3. 冷启动 + G-FFI-5（或等价）绿  
4. 可选：full `.x` vs 旧 seed 的 nm 扩展证明  

**批量规则**：同一 Cap 解锁后，**同一 PR 或连续 PR 吃掉所有同因 rest**，不要解锁后只迁 1 个再回去刷 thin N。

### Track L — 叶子 prove（降级为辅轨）

**仍值得做 L 的情况**：

- 产品 **尚未** PREFER_X_O，prove 可推动首次切 hybrid/纯 .x  
- 模块 **本就是纯 thin / 无 rest**，prove 成本低、可删构建硬依赖  
- 真迁 R2 之后的 **回归锁**（防止 rest 回潮）

**不值得做 L 的情况**：

- 产品已 PREFER、rest 被 Cap 卡住、只扩 `*_surface.from_x.c` 刷 N  
- 1 个 trivial 符号、零能力推进  

**主判据**（若做）：`prove_module_selfhost.sh` nm IDENTICAL + 构建路径明确。  
C 文本 diff=0：**永不**作为 L/R 硬门禁。

### Track M — Mega 去 pin（与 R 并行的高杠杆）

**不要**等 runtime hybrid 全部 R2 再碰 mega。  
pin 的 typeck/codegen/parser 是 **冷启动与语义的最大双权威**。

| 层 | 判据 |
|----|------|
| M1 | 该 mega 源能被当前编译器 typeck / `-E` 消费 |
| M2 | regen 的 `*_x.o` **替换** pinned `.o` 链接 smoke 绿 |
| M3 | Stage2：`bootstrap-verify-stage2-bstrict` / D-03（Linux） |
| M4 | 关 pin / 空 patch；冷启动可从「上一代 shux -E」重建 |

Mega 期允许 **pinned + 唯一幂等 patch**，禁止手改 gen.c 成第三权威。

## 4. 标准闭环（按轨）

### 4.1 Cap 闭环 — 5 步

```
1. 从 hybrid rest / 失败 log 聚类「能力缺口」（≥2 模块或 ≥1 大 rest）
2. 最小复现（≤20 行 .x）
3. 修实际链接路径 + 对齐 .x
4. Ubuntu + 全图/相关门禁
5. 写出「解锁的 R 清单」并立刻进入 R，不回头刷 L
```

### 4.2 真迁 R 闭环 — 6 步

```
1. 选 rest：被刚完成的 Cap 解锁，或本就无能力缺口
2. .x 吃满符号（或拆纯 .x TU）；更新 FROM_X / rest 宏
3. g05 / 产品拓扑去掉该 rest C 体
4. 证明：链接符号来源 + 烟测 / G-FFI-5
5. 可选 prove full 或扩展 nm
6. 文档：H↓、R2 计数↑（不要只写 N↑）
```

### 4.3 叶子 L 闭环 — 仅 ROI 明确时

```
1. 确认「非刷分」：未产品化 或 纯叶 或 R2 回归锁
2. -E 绿 → prove nm → 构建切换
3. 门禁绿；文件可留作冷启动考古
```

### 4.4 Mega M 闭环

```
1. M1 可消费源
2. M2 替换 *_x.o smoke
3. M3 Stage2 / D-0x
4. M4 去 pin
```

| 序 | 动作 | 轨 | 理由 |
|----|------|-----|------|
| 0 | 维持门禁绿（bootstrap / G-FFI-5） | 基建 | 底线 |

- 为 N=44…52 把剩余 runtime hybrid 再 surface-prove 一遍  
- 以 parser C 文本 diff=0 当关卡  
- 等「所有 thin 证完」再开 mega

## 7. seed 删除门槛（分级）

### 7.1 产品链退役（R2 / 强 L）

1. Ubuntu 稳定  
2. 产品不再链接该 C 体（rest 空或宏去掉）  
3. 冷启动 + 相关 gate 绿  
4. 建议连续 **2** 次 Ubuntu 全绿  

### 7.2 Mega seed（M）

1. M2 smoke + M3 Stage2  
2. 关 pin 后冷启动可重建  
3. 无 weak/双版本行为分叉  
4. patch 可删或空操作  

### 7.3 物理删文件

产品链退役稳定 **≥1 周** 或 **≥3** 次全绿 CI；冷启动零状态路径仍要有一代 seed 或「上一代 shux」。

## 8. ABI / 双权威纪律

1. **语义权威**：`.x`  
2. **冷启动权威**：pinned seed + **唯一** 幂等 patch（禁止手改 gen 成第三权威）  
3. **Glue 权威**：`strict_minimal` 与全量 glue **同语义**  
4. **ExprKind 序数**：以 parser_asm / backend 为准（如 STRING_LIT=**59**）  
5. **关键字参数**：禁止 `type` 等保留字作参数名  
6. **表达式 unsafe**：`let rc; unsafe { rc = call(); }`，禁止 `if (unsafe{…}!=0)` 依赖 statement-expression  
7. **struct stack escape**：同帧两 `&local` 同传可放行；更长寿命指针同传仍拒——修在 glue 热路径，**pipeline_x 静态副本须 mega 重编**  
8. **hybrid 宏**：`SHUX_*_FROM_X` 只表达「thin 已从 .x 来」；**不得**暗示 rest 已真迁

## 9. 执行阶段（后 T 重划）

### Phase A — 地图与 typeck ✅

- [x] T=18/18  
- [x] 主要 W 波次（string / unsafe / tail / heap / escape / lexer…）  
- [x] driver 叶 / labi 叶 / 大量 hybrid thin 产品化 + N 证明  

### Phase B — 能力解锁 + 真迁（**当前主线**）

- [ ] Cap 前排清完（空串、池、BSS、va… 按实际阻塞排序）  
- [ ] runtime / labi / driver：**H → 0**（或仅剩刻意 🔒 syscall 壳）

- **出口**：产品 `runtime_driver_no_c` / labi **无业务 rest C**（syscall/crt 级除外）  

### Phase C — Mega 去 pin（与 B 并行）

- [ ] typeck M2→M3→M4  
- [ ] codegen → parser  
- **出口**：`SHUX_FORCE_REGEN_GEN=1`（或等价）冷路径可重建且 Stage2 绿  

### Phase D — 终局

- [ ] v1→v2→v3；SELFHOST D/E  
- [ ] seed 物理删除策略落地  
- [ ] G（nostdlib / 物理零 C）按独立里程碑，不阻塞 B/C 定义的「语义自举」

1. **禁止**只修 seed 不修 .x 语义  
2. **禁止**只修 .x 却不碰 **实际被链接** 的 seed/glue  
3. **禁止**以 C 文本 diff 卡住退役  
4. **禁止**无 Ubuntu 验证宣称完成  
5. **禁止**用 scp 代替 git 同步正式构建  
6. **禁止**为「纯 .x」复制第三套 ABI 表  
7. **禁止**把 lexer 超时和 typeck FAIL 绑同一 PR 盲修  
8. **禁止**在 T 已满、rest 未动时，用 thin surface prove **冒充**自举进度  
9. **禁止**等 hybrid 全部 thin 证完再开 mega

## 14. 现在立刻该做什么

1. **列 H**：产品 `runtime_driver_no_c` / labi 上 **仍有业务 rest C 的 TU**（不要只看 N）。  
2. **标 Cap**：每个 rest 卡在哪条能力（空串 / BSS / 池 / va / 大函数 IO…）。  
3. **开一条最高杠杆 Cap** → 同波 **批量 R2** 吃掉解锁面。  
4. **并行**：typeck（或当前最易）**M2 试替换 smoke**，不要等 H=0。  
5. **停止**：仅为 N+1 写 `*_surface.from_x.c`。  

地图与门禁在手，自举速度由 **Cap 杠杆 × R 批量 × M 去 pin** 决定，  
而不是由「谁先把下一个 hybrid thin 的 nm 刷成 IDENTICAL」决定。
