## M1 硬件直通(第一波 — 让 shux 能在 QEMU 点字)

| 状态 | ID | 项 | 范围 | 测试层 | 出口标志 |
|------|----|----|----|--------|--------|
| 🟡 | K1 | **内联汇编 `asm!`** | lexer/parser/typeck/codegen;模板 + in/out/clobber 约束 + options;**必须在 `unsafe{}`** | T1(parse/typeck)+ T2(执行) | `asm!("out %1,%0"... )` 写串口、`mov %cr3` 读出、`hlt` 不死机,QEMU 显字 |
|   |   | └ K1a 模板版(C 前端) | `asm!("template")` parse→typeck(强制 unsafe)→C codegen `__asm__("...")`;新增 `AST_EXPR_ASM`/`asm_tmpl` union | T1 | ✅ `asm!("nop")` 经 shux-c -E 出 `__asm__("nop")`,host cc 编译运行 rc=0;unsafe 外报 `asm! requires unsafe block`;多指令模板转义正确;无回归 |
| ☐ | L8 | **`asm!` 操作数补全** | `sym` 符号操作数、`label` 跳转操作数、clobber memory vs reg、`nomem/readonly/pure` 语义 | T1+T2 | `sym(idt)` 引用 IDT 地址、asm 内 `label:` 跳转可用 |
| ☐ | K2 | **语言级 `volatile`** | typeck 识别 `*volatile T`;codegen 标记不 DCE/不 GVN/不 autovec、emit 真实 load/store | T1(查产物未被消除)+ T2(VGA/串口) | `*volatile` 改 VGA 字符 QEMU 显字;编译产物确认 volatile 读未被删 |

> M1 出口:`asm!` + 语言级 volatile 落地,QEMU 上用纯 `.sx` 写串口/VGA 点字。**这是通往内核开发的最短捷径**——后端一时写不出 `mov %cr3` 原生编码,先用 `asm!` 顶。
>
> **K1 进度(2026-07-02)**:第一切片(K1a 模板版,C 前端)已落地并 T1 验证通过——`AST_EXPR_ASM` 走通 parse→typeck(强制 unsafe、类型 void)→C codegen(`__asm__("...")`),`asm!("nop")` 在 host 编译运行 rc=0。**仍待办**:K1b 操作数 in/out/clobber(= L8)、K1c asm 后端原生 emit(模板塞进 `.text`)、K1d `.sx` 前端移植(须 bstrict 重编,内存不安全环境外做)、K1e QEMU 执行 gate(特权指令 `hlt/out/mov %cr3` 在 QEMU 跑)。当前 C 前端为验证载体;真正内核用须移植到 `.sx` 前端(每日 shux)。

| ☐ | K11 | **地址空间 / 重定位** | x86_64 `enc_lea_rip_sym`;ELF writer 数据符号记 `R_X86_64_PC32`/`R_X86_64_64`;driver 接受 `-T linker.ld` 与基址 | T1(编码/重定位)+ T2(高半区) | 高半区加载后引用数据正确;`.boot` 钉低物理地址 |

| ☐ | K5 | **内核入口接管** | `#[entry]` 生成裸名 `_start`(naked)、设栈、清 `.bss`、`call kmain`、不返回则 `hlt`;或 `shux build --kernel --entry _start --linker-script` | T2 | QEMU `-kernel` 跳进 `_start` 进 `kmain` |
| ☐ | K6 | **boot 协议头** | `std.kernel.boot.multiboot2` / `stivale2`;`Header` 结构 + 构造宏;借 K4 落段、L13 const-eval 算 magic | T2 | QEMU 识别头并跳进 `_start` |

## 实施铁律

1. **严格按 M1→M2→M3→M4→M5+ 顺序**:后项尽量以前项出口为前提;K4 段控制提前到 M2(boot 头强依赖)。
2. **每项必带 gate**:K1–K11/L1–L13 每落地一项,同时落对应 T1/T2 烟测(见 [`内核级语言.md`](内核级语言.md) §8.2);验证难度极高,必须 gate 化防积债。
3. **本地 TDD**:T1/T2 全本地(零外部硬件);T3 真机只用于最后一公里(真实设备时序/厂商固件/性能),不阻塞主线。
4. **安全边界**:内核特性全部落在 `unsafe{}` 与 naked/asm 内,安全子集不外泄硬件危险面,与 shux「默认安全」一致。
5. **双线并行**:内核线(K1 起)与编译器自举收尾线(阶段 G)并行;会合时 shux 既能自编译、又能写内核。

5. **双线并行**:内核线(K1 起)与编译器自举收尾线(阶段 G)并行;会合时 shux 既能自编译、又能写内核。
6. **ABI 严格对齐**:naked 函数、ISR 栈帧、任务切换上下文必须与 shux 普通函数的调用约定**逐寄存器完美对齐**(被调用者/调用者保存集、栈帧布局、返回地址位置)。错一个寄存器就是「本地跑通、偶发崩在真机/特定编译选项」的调试地狱——这是内核侧最高危的隐性 bug,K3/K10/任务切换落地时必须以 shux 当前 ABI 为基准写对照表并 gate 校验。
7. **设计参考与风格**:volatile / naked / 段控制 / asm! 这些「能用但不优雅」的早期实现日后重构成本极高,设计时就参考 Rust(`asm!`/`naked`/`link_section`/`global_allocator`)、Zig(`@Type`/inline asm/sections)、C++20(`__attribute__`/`std::atomic`)的对应机制,但保持 shux 自己的显式 + 默认安全风格;不为抄而抄。

> **第一步:K1 `asm!`。** 一旦内联汇编落地,K2–K11/L1–L13 绝大部分可先用 `asm!` 顶上、绕过后端原生缺失,这是通往内核开发的最短捷径。
