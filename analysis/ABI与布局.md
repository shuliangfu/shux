# ABI 与布局（结构体 / 枚举 / 切片与 C 的对应关系）

> 结构体、枚举、切片在内存中的布局若与 C 或特定 ABI 一致，应写清；便于 FFI、调试与性能调优。本文档汇总与 C/ABI 的对应关系，并引用《内存契约》中已约定的内容。对应《自举前-目标与缺口分析》ABI/布局项。

| 目标 | 说明 |

## 二、结构体（struct）

- **默认对齐**：与 C 一致的自然对齐；字段顺序与 .sx 声明一致；字段间与结构体末尾可能含填充（除非使用 **allow(padding)** 禁止隐式填充并显式补字段，或使用 **packed**）。
- **Packed**：`struct Name packed { ... }` 对应 C 的 `struct Name { ... } __attribute__((packed))`，无填充、对齐 1。
- **详细规则**：见《内存契约》第二节（Struct 布局）。

---

## 三、枚举（enum）

- **当前实现**：无负载枚举，变体为命名常量；生成 C 时对应 C 的 `enum Name { Name_A, Name_B, ... }`。
- **大小与对齐**：与 C 枚举一致，按 **int**（4 字节）大小与对齐；与 C 互操作时可直接对应 `enum` 或 `int`。
- **变体顺序**：变体在 C 中的枚举值顺序与 .sx 中声明顺序一致（从 0 递增）。

---

## 四、切片（slice）T[]

- **布局**：固定为两字段：(1) 指向元素类型的指针；(2) 长度（usize）。
- **64 位**：16 字节（8 + 8），对齐 8；**32 位**：8 字节（4 + 4），对齐 4。
- **C 侧命名**：codegen 生成 `struct shux_slice_<elem> { elem* data; size_t len; }`；第一成员为指针、第二成员为长度，与《内存契约》Slice 固定 (ptr, len) 一致。
- **详细规则**：见《内存契约》第三节（Slice 固定 (ptr, len)）。

---

## 五、基本类型与指针

| .sx 类型 | C/ABI 对应 | 大小（64 位） | 对齐 |
|----------|------------|----------------|------|
| i32, u32, bool | int32_t, uint32_t, int | 4 | 4 |
| i64, u64, usize, isize | int64_t, uint64_t, size_t, ptrdiff_t | 8 | 8 |
| u8 | uint8_t | 1 | 1 |
| f32, f64 | float, double | 4, 8 | 4, 8 |
| *T | T* | 8 | 8 |
