## 0. 核心命题（读这一节就够）

Shux **不做 GC**，也 **不回到 C 式裸指针地狱**。我们要的是：

```text
编译期把「安全事实」压到 typeck 里  →  codegen 敢优化  →  性能比肩/超越 C
编译期把「分配/释放」压到作用域里  →  程序员少写、机器零追踪  →  体验接近「自动」
```

**「编译时自动内存管理」在 Shux 中的精确定义**：

| 层 | 名称 | 做什么 | 运行时成本 |
|----|------|--------|------------|
| **A** | 编译期安全 | region / linear / borrow / noalias / BCE | **0** |
| **B** | 编译期释放 | defer / owned / Arena deinit | **0**（静态展开） |
| **C** | 编译期 Allocator 绑定 | `with_arena` 块内自动 threading `Allocator` | **0**（单态化） |
| **D** | 编译期布局/向量 | SROA / SoA / loop autovec | **SROA v1 ✅** |
| **✗** | 运行时 GC/RC | — | **禁止默认** |

**与 Zig 的关系**：边界上同样 **显式**（API/FFI 传 `Allocator`）；块内 Shux **更自动**（scope 注入 + region 拒逃逸）；安全上 **更严**（slice 域 + Linear move）。性能 closure 后 **≥ Zig**，不是更慢。

## 1. 设计总纲：统一作用域效应（Scope Effects）

### 1.1 一个块，三种编译期效应

Shux 不把「内存安全」「分配策略」「资源释放」拆成三套无关机制，而收敛为 **作用域效应** —— 标签与对象 **仅存在于 typeck**，codegen 展开为等价 C/汇编，**不进 ABI 元数据表**。

```text
with_arena(cap) {          // 效应 ①：绑定 Allocator（分配域）
  region io {              // 效应 ②：绑定 slice 借用域
    let buf = read_ptr();  // u8[]<io>；Allocator = arena
    let v = vec_new();     // 编译期：vec.al = scope_allocator()
    parse(buf, v);         // callees 自动 inherit ①②；defer 在块尾 ③
  }
  defer arena_deinit();    // 效应 ③：释放调度
}
```

| 效应 | 语法 | typeck 管什么 | codegen 产出 |
|------|------|---------------|--------------|
| **借用域** | `region L { }` | slice 不得逃出 `L`；域 mismatch 报错 | 嵌套 `{ }`；无额外指令 |
| **分配域** | `with_arena { }` / `#[alloc]` | 块内分配走同一 `Allocator`；容器不得带错域逃逸 | 单态化 call；`Allocator` 作隐式参数或 struct 字段 |
| **释放域** | `defer` / `owned T` | 块出口逆序 deinit；与 Linear move 不冲突 | goto cleanup 或出口展开 |

**极致性能点**：三种效应叠加时，parse/typeck 热路径可做到 **一次 Arena bump + 零 per-op free + 零 slice UAF + 零 malloc 计数**（PERF-007 已验证 Arena 子集）。

### 1.2 与「自动」一词的边界

| 用户感知 | 实际机制 | 是否隐藏成本 |
|----------|----------|--------------|
| 「不用写 free」 | defer / owned / Arena 块 deinit | ❌ 释放点仍在 IR 可见 |
| 「不用每层传 alloc」 | scope Allocator 注入 | ❌ 边界 API 仍显式 |
| 「编译器保证不 UAF」 | region typeck | ✅ 完全静态 |
| 「自动向量化」 | noalias + BCE 证明通过才 autovec | ❌ 不满足则标量 |

**绝不引入**：全局隐式堆、GC 写屏障、运行时 Allocator 查表。

## 3. 编译期 Allocator 自动传递（MEM-C 层）

> **STD-112 已有 `Allocator` trait**；本节定义 **编译期如何 threading**，使块内代码 **不必** 像 Zig 那样每层手写 `allocator` 参数。

### 3.1 三层 Allocator 语义

| 层级 | 写法 | 语义 | 对标 Zig |
|------|------|------|----------|
| **L0 默认堆** | `vec_i32_new()` | 隐式 `allocator_heap()` | `GeneralPurposeAllocator` 默认 |
| **L1 作用域绑定** | `with_arena(n) { ... }` | 块内 `#[alloc]` API **编译期**绑定 Arena | 块内仍手写 `defer(a.deinit())` + 每层传 `allocator` |
| **L2 API 边界显式** | `fn f(al: Allocator, ...)` | 策略由调用方决定 | 公共 API 同样显式 |

**L1 是 Shux 相对 Zig 的核心增量**：Zig 的显式传递在 **每一层 call**；Shux 在 **块内 call 链** 由 typeck 单态化注入，**生成代码与手写传递相同**。

### 3.2 语法草案（v1）

```su
// ── 作用域 Arena：分配域 + 释放域一次绑定 ──
with_arena(65536) {
  let tokens = vec_u32_new();       // 改写：vec.al = scope_alloc()
  append_token(tokens, 42);         // #[alloc] fn 自动拿 scope_alloc()
  build_ast(tokens);
} // 编译期：defer arena64_deinit；拒 tokens 逃逸（Allocator 域检查）

// ── 与 region 叠加：借用域 + 分配域 ──
with_arena(4096) {
  region parse {
    let src: u8[] = read_file_slice();   // u8[]<parse> 若 stamp
    let names = map_str_new();           // bump from arena
    lex(src, names);
  }
}

// ── 库边界：仍显式（FFI / 公共 API）──
function parse_file(al: Allocator, path: u8[]): Ast { ... }

// 调用方
with_arena(1_000_000) {
  let ast = parse_file(scope_allocator(), path);
}
```

**`scope_allocator()`**：仅能在 `with_arena` 块内调用；typeck 展开为当前块绑定的 `Allocator` 常量/栈槽，**非**运行时 TLS。

### 3.3 `#[alloc]` 与容器携带 Allocator

两种等价实现路径（可并行）：

**路径 A — 隐式参数注入**（适合 free function）：

```su
#[alloc]
function vec_u32_push(v: *Vec_u32, x: u32): i32 {
  return allocator_realloc(scope_or_param_alloc(v), ...);
}
```

typeck：若在当前 `with_arena` 内且无显式 `al` 形参 → 注入 `scope_alloc()`；否则用 `v.al` 或 heap。

**路径 B — 容器字段**（适合 Zig ArrayList 风格，**推荐 std 统一**）：

```su
struct Vec_u32 {
  ptr: *u32;
  len: i32;
  cap: i32;
  al: Allocator;    // new 时写入；push 用 self.al
}
```

`vec_u32_new()` 在 `with_arena` 内 → `al = scope_alloc()`；在块外 → `al = allocator_heap()`。

### 3.4 typeck 规则（ Allocator 域）

| 规则 ID | 条件 | 结果 | 状态 |
|---------|------|------|------|
| AL-01 | `with_arena` 块内调用 `#[alloc]` / `default_allocator()` | 使用 `scope_alloc()` | ✅ `default_allocator()` |
| AL-02 | 块外调用 `#[alloc]` / `default_allocator()` | 使用 `allocator_heap()` | ✅ `default_allocator()` |
| AL-03 | 函数显式 `al: Allocator` 形参 | **以形参为准**，不注入 | ✅ 已有 |
| AL-04 | `Vec`/`Map` 带 `al` 字段跨块返回 | **报错**：`allocator region escape` | ✅ return Allocator / arena 容器 |
| AL-05 | `al.kind == arena` 容器要求 `realloc` | **报错**或显式 fallback heap（v1：报错） | ✅ arena realloc 拒错 |
| AL-06 | slice region 逃逸 + Allocator 逃逸 | **同时检查**；双重拒错 | ✅ `run-al06-gate.sh` |

### 3.5 codegen：单态化，零间接

```text
// 源码（块内）
vec_u32_push(&v, 1);

// 生成（概念 IR；与手写等价）
vec_u32_push_arena(&v, 1, (Allocator){ .kind=ARENA, .arena=&__scope_arena_0 });
```

- **无** vtable；**无** 运行时 `kind` 分支（单态化后常量折叠）。
- Arena bump → 内联为指针加法 + 对齐 check（已有 `heap_arena64_alloc_c`）。
- PERF-007：`strace` 计数 `malloc=0` 为验收代理。

### 3.6 与 Zig 对比（Allocator 维度）

| | Zig | Shux v1 目标 |
|---|-----|--------------|
| 公共 API | `allocator: Allocator` 形参 | 同 |
| 块内调用链 | 每层传递 | **scope 注入** |
| Arena | `defer(a.deinit())` | `with_arena` + defer + 自动 binding |
| 容器 | `ArrayList(T).init(allocator)` | `vec_new()` 读 scope |
| 逃逸 | 程序员纪律 | **typeck 拒 Allocator/slice 逃逸** |
| 性能 | 零成本 | 零成本（同 IR） |

## 4. 编译期安全（MEM-A 层）— 安全压榨

### 4.1 安全子集：编译期必拒错误清单

| 错误类 | 机制 | 阶段 | 现状 |
|--------|------|------|------|
| slice UAF / 域逃逸 | `T[]<L>` + `region` | typeck | ✅ |
| double-move | `Linear(T)` | typeck | ✅ |
| 悬垂借用 | scope borrow | typeck | ✅ `run-scope-borrow-gate.sh` |
| 越界（可证明） | BCE | typeck→codegen | ✅ `run-bce-gate.sh` |
| 越界（不可证明） | 有定义检查/panic | codegen | 部分 |
| 空指针误用 | `Option<T>` / `T?` | typeck | 部分 |
| Allocator 域逃逸 | AL-04 | typeck | ✅ return / arena 容器 |
| 堆 use-after-free | owned + borrow 互斥 | typeck | ✅ owned 块尾 + early return deinit |
| 数据竞争 | Send/Sync / 无共享 mut | typeck | ⬜ 远期 |

**原则**：能在 typeck 拒的 **绝不留到运行时**；必须运行时的（不可证明越界）用 **有定义行为**，不用 UB。

### 4.2 精细 noalias — autovec 的安全阀门

**现状问题**：typeck 对所有 `*T` 形参标 `is_restrict` → 重叠指针被错误假定 disjoint → **错误 autovec 风险**。

**v1 规则**（MEM-A1 ✅，`run-noalias-gate.sh`）：

| 指针来源 | noalias | 说明 |
|----------|---------|------|
| 不同 `owned` 堆块 `.data` | ✅ | 若无 proven overlap |
| 同一 slice 基址 | ❌ | 同基不同 offset 仍可能重叠 |
| `region` 内只读 borrow vs 可变 | 只读集合 ↔ 可变 disjoint | 支撑 read + write 不同 slice |
| 两形参 `*T` | **仅当** typeck proof | 否则 **不** restrict |
| `unsafe` 块内 | 程序员负责 | 不 autovec |

codegen：C 路径 `T *restrict`；asm/LLVM 路径 `noalias` 元数据；**autovec pass 仅在 proof bit 置位时启用**。

### 4.3 BCE（边界检查消除）

| 模式 | 证明 | 动作 |
|------|------|------|
| `for i in 0..slice.len` | `i ∈ [0, len)` | 删除 `a[i]` guard |
| `while i < n` 且 `n == len` | 同上 | 删除 guard |
| 嵌套 loop | 逐层 range | 内层无分支 |

Debug：保留检查；Release：证明则删；**不满足则保留** — 不 silent wrong code。

### 4.4 scope borrow（✅ v1）

- 默认：`let r = &x` **不得**逃出 `{}` / `region`（《语法与类型设计》§8.3）。
- typeck：`return &x`、`save = &x`（块外变量）→ `scope borrow escape` 拒错。
- 与 region 分工：borrow 管 **栈/局部**；region 管 **slice 视图**。
- 性能：borrow 不逃逸 → 变量可全寄存器化 → 减少 spill → 支撑 autovec 寄存器压力预算。

## 5. 编译期释放（MEM-B 层）

### 5.1 defer — 零运行时栈（MEM-B0 ✅ C 路径；parser.sx pool + codegen.sx `emit_run_defers` 已落地）

算法见《控制流与语法构造清单》§8.3：

- Parser 收集 `defer_stack`；**禁止** `runtime_push_defer`。
- return/break/continue/块尾 → 逆序注入或 **goto cleanup**。
- 与 `with_arena` 标准模式：

```su
with_arena(cap) {
  defer arena64_deinit(&__scope_arena);
  ...
}
```

### 5.2 owned T — 堆容器 opt-in 自动 deinit

```su
{
  let v: owned Vec_u32 = vec_u32_new();
  ...
} // 编译期插入 vec_u32_deinit(&v)；Linear/region 规则不变
```

| 类型 | 释放 |
|------|------|
| `owned Vec/Map/Bytes` | 块尾 deinit；early return/break 走 goto cleanup |
| `Linear(T)` | move 消耗；**不用** owned |
| Arena 内 `Vec`（`al.kind=arena`） | deinit 为 no-op；Arena defer 一次 free |

### 5.3 为什么不默认全堆 auto-deinit

1. FFI 所有权必须可见。  
2. 错误路径释放顺序需可控。  
3. 嵌入式禁止隐式析构链。  
4. 自举编译器热路径：**Arena 为主**，不是堆 drop 链。

## 6. 性能压榨金字塔

从底到顶，每一层依赖上一层的事实；**跳过下层做顶层 = 错优化或假安全**。

```text
                    ┌─────────────────────┐
                    │ L5 loop autovec     │  dot/matmul 自动 SIMD
                    │ L4 显式 SIMD/SoA    │  Vec4f、#[repr(soa)]
                    ├─────────────────────┤
                    │ L3 BCE + noalias    │  无 guard、无 alias barrier
                    ├─────────────────────┤
                    │ L2 scope Allocator  │  0 malloc 热路径
                    │     + Arena bump  │
                    ├─────────────────────┤
                    │ L1 region/linear/   │  编译期拒 UAF/move
                    │     borrow          │
                    ├─────────────────────┤
                    │ L0 无 GC / 无隐藏分配│  可预测延迟
                    └─────────────────────┘
```

### 6.1 分配路径性能序（同一算法）

| 路径 | 相对成本 | 适用 |
|------|----------|------|
| 栈 / SROA | **1×** | 逃逸分析后不越块 |
| Arena bump | **~1×** + 一次 chunk syscall | parse/request/compile pass |
| 池化 freelist | **~2×** | 固定大小高频 |
| 堆 malloc/free | **~10–100×** | 长生命周期、跨线程 |

**编译器自动决策（远期 P3）**：

- 逃逸分析 → 栈 promotion。  
- `with_arena` 内所有 `#[alloc]` → 强制 bump。  
- CTFE 字符串/表 → `.rodata`，零堆。

### 6.2 向量化路径性能序

| 路径 | 何时用 |
|------|--------|
| 手写 `@shuffle` / intrinsic | 已知 shuffle 模式 |
| `Vec4f` 显式 | 宽度固定 kernel |
| loop autovec V1 | `for i in 0..len` + proof OK |
| 标量 + BCE | proof 不足时 **正确回退** |

**禁止**：proof 不足强开 autovec（宁可慢，不可错）。

## 7. 自动向量化 — 与内存的因果链

### 7.1 双轨策略（不变）

| 轨道 | 状态 | 依赖 |
|------|------|------|
| 显式 SIMD | ✅ STD-S3/S4 | `-target-cpu` |
| loop autovec | ✅ VEC-V1/V2/V3 | §4.2 noalias + §4.3 BCE |

### 7.2 端到端示例（目标态）

```su
function dot(a: f32[], b: f32[]) -> f32 {
  let n: i32 = a.length;
  let mut s: f32 = 0.0;
  for i in 0..n {
    s = s + a[i] * b[i];
  }
  return s;
}
```

**typeck 输出（概念）**：

```text
proof: i ∈ [0, n), n == a.len == b.len
noalias: a.data disjoint b.data  (若参数来自不同 owned/region)
→ autovec_eligible = true
```

**codegen 输出（x86 AVX2）**：

```text
vmovups / vmulps / vaddps  × (n/8)
vhadd / 水平归约 epilogue
```

若无 noalias：`autovec_eligible = false` → 标量 + BCE（若可）→ 仍 safe。

### 7.3 loop autovec 分期

| 阶段 | 范围 | 验收 |
|------|------|------|
| V1 | `for i in 0..len`，无 call，双指针 noalias | `run-autovec-loop-gate.sh` ✅ |
| V2 | while + reduction（f32 sum/dot） | `run-autovec-v2-gate.sh` ✅ |
| V3 | SoA + 跨平台 | `run-autovec-v3-gate.sh` ✅ |

## 8. 默认 vs opt-in — 决议表（v1）

| 能力 | 默认 | opt-in | 禁止 |
|------|------|--------|------|
| region / linear 检查 | ✅ | — | — |
| scope borrow | ✅ | — | — |
| BCE（Release） | ✅ | Debug 全检查 | — |
| loop autovec | ✅ 若 proof OK | `-C no-autovec` | proof 失败强开 |
| 精细 noalias | ✅ | — | 全指针 restrict |
| L0 堆 `vec_new` | ✅ 块外 | — | — |
| L1 `with_arena` + 注入 | — | ✅ | — |
| L2 API 显式 `al` | 边界必须 | — | — |
| defer | ✅ 语义 | — | — |
| owned T | — | ✅ | — |
| GC/RC | — | — | ❌ |

### 8.1 编译选项

| Flag | 含义 |
|------|------|
| `-C opt-level=2` | BCE + autovec（proof OK） |
| `-C no-autovec` | 仅显式 SIMD |
| `-C sanitize=address` | ASAN |
| `-C unsafe-allow` | bootstrap/FFI |

**不提供** `-C no-memory-safety`。

## 11. 风险与非目标

### 11.1 风险

| 风险 | 缓解 |
|------|------|
| 粗 restrict + autovec | P0 必须先改 noalias |
| scope 注入 + 单态化代码膨胀 | 仅 hot API 单态化；冷路径 heap |
| owned 与 Linear 冲突 | 类型级互斥 |
| Arena 满 | 编译期 cap 提示；运行时返回 null + 显式错误 |

### 11.2 非目标（v1）

- GC/RC 默认  
- 全程序逃逸分析（v1 仅 block 级）  
- 跨线程 Allocator 自动传递  
- polyhedral autovec

## 12. 总结

1. **编译时自动内存管理** = **A 安全 + B 释放 + C Allocator 作用域绑定 + D 布局/向量**；**零 GC**。  
2. **Allocator 可以且应该编译期自动化** — 在 `with_arena` / `region` 块内 threading，边界 API 仍显式；生成 IR **等同 Zig 手写**。  
3. **安全压榨到至极** = typeck 拒已知错 + noalias 阀门 + BCE；**性能压榨到至极** = Arena 0 malloc + noalias 驱动 autovec + 显式 SIMD 保底。  
4. **因果链**：作用域效应 → 别名证明 → 无 guard 循环 → autovec；任何一环缺失则 **标量回退**，不牺牲安全。  
5. **下一步**：~~P0~~ ✅ → ~~P1~~ ✅ → ~~P1.5~~ ✅ → ~~P2 autovec~~ ✅ → ~~P3 VEC-V3~~ ✅ → ~~P3 MEM-D1/D2/D3 CTFE 链~~ ✅ → **远期：全程序逃逸 / CTFE 字符串.rodata**。
