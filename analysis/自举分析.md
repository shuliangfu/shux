## 1. 完全自举的定义

「自举」在 Shux 仓库内**不是**单一概念，须按层次区分：

| 层次 | 名称 | 判据（摘要） |
|------|------|--------------|
| **语义自举** | Stage 0 烟测 | `make bootstrap-verify`：两代 `shux` 跑约定测试 outcome 一致 |
| **黄金自举** | 阶段 D | Stage1 再编同一 tree → Stage2；**SHA256(stage1)==SHA256(stage2)** |
| **编译器无 C** | 阶段 E | 默认 bootstrap **不链、不 cc -c** 编译器前端 C（E-soft：文件可保留） |
| **std 无 C** | 阶段 F | `std/` 下 **0 个手写 `.c`/`.h`**（F-ZC） |
| **完全自举** | **D + E + F** | 上述三者齐备；日常以单 `shux` 发布，允许 seed **仅冷启动** |
| **终局清场** | 阶段 G | 物理删除仓库内全部 `.c/.h`；Linux bootstrap **`-nostdlib` 硬绿**；`build.x` 替代 Makefile |

> **决议**：「全仓库 std 无 C」**不是可选项**；未完成 F 不得宣称完全自举。E-soft 保留 C 文件 ≠ 终局；终局见阶段 G。

## 2. 三阶段二进制闭环（Stage 0→1→2）

```text
Stage 0（寄生）  C/shux-c 编译全 .x 编译器  →  Stage 1 二进制 (shux_asm)
Stage 1（破茧）  Stage 1 再编同一套 .x 源码  →  Stage 2 二进制 (shux_asm2)
Stage 2（黄金）  SHA256(Stage 1) == SHA256(Stage 2) 且全链无 cc -c 编译器主体
终局（阶段 G）  删除仓库内手写 .c/.h —— 含编译器、LSP、core、std（全仓库无 C）
```

**金标准验收**（Linux x86_64 Docker）：

```bash
./tests/run-linux-a09-a11-gate.sh          # A-09～A-12 一键
SHUX_D03_FAIL=1 ./tests/run-d03-stage2-hash-gate.sh
```

## 3. 五关卡 × 三平台

| 关卡 | 内容 | Linux | macOS | Windows |
|------|------|-------|-------|---------|
| **一** | 编译器前端全 `.x`（lexer/parser/ast/typeck/codegen） | ✅ 默认链 `*_x.o` | ✅ B-strict | 🟡 B-hybrid |
| **二** | 后端 Codegen 自举 MVP（指针/struct/控制流/栈帧/cfg） | ✅ asm 主链 | ✅ asm 主链 | ⬜ |
| **三** | `_stubs.c` 退役 → `std/sys/*.x` 独立调 OS | ✅ v1 gate | ✅ v1 gate | 🟡 v1 门面 |
| **四** | 三阶段自举闭环 + 二进制哈希一致 | ✅ D v1 | ✅ 行为 parity | ⬜ |
| **五** | 全仓库 std/core 无 C（`.x` + FFI/asm） | ✅ F-ZC | ✅ F-ZC | ✅ F-ZC |

**平台 syscall 策略**（长期不变）：

- **Linux**：`std/sys/linux.x` — 内联 asm `syscall`；可 freestanding，拒绝 libc 脐带（F-no-libc）。
- **macOS**：`std/sys/macos.x` — `extern "C"` 绑 `libSystem.dylib`（规避 syscall 号漂移）。
- **Windows**：`std/sys/win32.x` — `extern "stdcall"` 绑 `kernel32.dll` / `ws2_32.dll` + **IOCP**。

## 6. 风险与原则

1. **不等 GitHub 全平台 CI**：自举冲刺期以 **macOS 宿主 + Docker Linux x86_64** 为金标准；Windows/Alpine ⏭ 不阻塞 G 主线。
2. **E-soft ≠ 终局**：软退役保证日常 bootstrap 无 C 前端；**阶段 G** 才物理删除并收紧 F-09 STRICT。
3. **F 与 F-no-libc 叠加**：F 清 C **源码**；F-no-libc 清 **`-lc` 链接依赖**——Linux 上二者齐备才是「真 no-lib」。
4. **BOOT 双轨**：mega7/parser 晋升已闭合（BOOT-021～028）；后续不阻塞 std 清场与 G 主线。
