# 与 C 互操作（FFI）规范

> 自举后编译器可能仍部分调用 C（系统调用封装、第三方 C 库）。本文档约定：从 .x 调用 C 的语法与语义、调用约定、可安全跨边界类型、以及 .x 暴露给 C 的方式。与《自举前-目标与缺口分析》及「内存契约」一致。

---

## 一、目标与范围

| 目标 | 说明 |
|------|------|
| **从 .x 调用 C** | 能声明并调用 C 函数（含系统调用、libc、第三方 C 库），并传递基本类型、指针、缓冲区（如 `*u8`、`u8[]`）。 |
| **从 C 调用 .x** | 能暴露 .x 函数给 C 使用（符号名、调用约定、布局与 ABI 一致）。 |
| **可预测的布局** | 与 C 互操作的类型（struct、slice、枚举等）在内存中的布局与对齐有明确约定，见《内存契约》与 ABI/布局文档。 |

**范围**：自举前以**规范文档 + 必要语法**为主；完整实现（如所有类型的 C 对应、可变参数等）可在自举后逐步补全。

---

## 二、调用约定

- **默认**：与 C 互操作时采用目标平台标准 C 调用约定：
  - **x86-64**：System V AMD64 ABI（Linux/macOS），Windows 为 MS x64。
  - **ARM64**：AAPCS64。
- **返回值**：基本类型与指针按 ABI 用寄存器返回；结构体按 ABI（如 16 字节内双寄存器，否则内存）。**Result&lt;T,E&gt; 寄存器化**时，T/E 利用双寄存器返回，见《自举前-目标与缺口分析》Result 寄存器化一节。
- **名称修饰（mangling）**：暴露给 C 的 .x 符号**不**进行 C++ 式 name mangling，保持与 C 声明一致（可由 `extern "C"` 或等价语法保证）。

---

## 三、FFI 语法（当前与预留）

### 3.1 从 .x 调用 C：extern 声明

- **自举前约定**：在 .x 中通过 **extern 函数声明** 声明 C 符号，无函数体；codegen 生成 C 的 `extern` 声明，链接时与 C 实现绑定。
- **预留语法示例**（以实际实现为准）：
  ```su
  // 声明 C 函数：无体，由链接器解析
  extern function putchar(c: i32): i32;
  extern function malloc(size: usize): *any;
  extern function free(ptr: *any);
  ```
- **当前实现**：若编译器尚未支持 `extern` 关键字，可暂时通过「在生成的 C 中手写或包含 C 头」的方式满足自举前调用 C 的需求；文档化该过渡方案，并计划在自举前或自举后补齐 `extern function` 语法。

### 3.2 从 C 调用 .x：导出符号名与约定

- 编译器生成 C 代码时，对需暴露给 C 的函数使用**稳定、可约定的符号名**（如 `shux_<module>_<name>` 或用户可配置的 prefix）。
- 若需与 C 头文件一一对应，可在文档或工具中约定：由 .x 侧声明 `export "C"` 或类似属性，生成未修饰的 C 符号名（与 C 头中的声明一致）。

## 四、可安全跨边界类型

以下类型在满足「内存契约」与 ABI 约定时，可安全用于 .x 与 C 之间传递：

| .x 类型 | C 侧对应 | 说明 |
|----------|----------|------|
| i32, u32, i64, u64, usize, isize | int32_t, uint32_t, int64_t, uint64_t, size_t, ptrdiff_t 等 | 与 ABI 整数一致 |
| bool | _Bool (C99) 或 int | 约定 0/1 |
| *T, *u8 | 对应指针类型 | 裸指针与 C 指针一致 |
| T[N]（数组） | 对应 C 数组 | 见 struct 布局与 Packed |
| struct（默认对齐或 Packed） | 对应 C struct | 见《内存契约》：默认对齐、Packed 显式 |
| T[]（切片） | 固定为 { ptr, len } 布局 | 见《内存契约》：Slice 固定 (ptr, len) |

**不承诺或需额外约定**：泛型单态化后的类型、带生命周期的引用（若将来引入）、未约定布局的枚举。

---

## 五、内存契约（与自举前必须清单第一步一致）

与 C 互操作与 ABI 的基础依赖以下约定，详细定义见《内存契约》文档（自举前必须清单第一步）：

- **struct 默认对齐**：与 C 一致的自然对齐；无 `packed` 时不留空洞的紧凑布局或按成员对齐依实现约定写清。
- **Packed 显式**：需与 C 的 `__attribute__((packed))` 或 `#pragma pack` 一致时，使用显式 `packed` 标记，避免隐式歧义。
- **Slice 固定 (ptr, len)**：切片在 ABI 上固定为「指针 + 长度」两字段，顺序与大小与 C 侧期望一致（如 8+8 字节 on 64-bit），便于零拷贝传递。

## 六、示例（示意）

### 6.1 .x 调用 C

```su
// 自举前语法以实际为准
extern function write(fd: i32, buf: *u8, count: usize): isize;

function my_print(msg: u8[]): isize {
  return write(1, msg.ptr, msg.len);  // 假定 slice 为 ptr+len 布局
}
```

### 6.2 C 调用 .x

C 侧声明与 .x 导出符号一致，例如：

```c
// C 头文件
long shux_std_io_print_i32(int x);
```

.x 中对应函数在 codegen 时生成名为 `shux_std_io_print_i32` 的 C 函数，调用约定与 C 一致。
