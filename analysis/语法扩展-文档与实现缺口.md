# 语法扩展：文档与实现缺口

本文档对照 `docs/01–07`、`docs/语法文档与实现对照.md` 与编译器实现（parser、typeck、codegen、ast），列出**文档已写明但实现尚未支持**的语法项，便于逐项补齐。

## 一、缺口清单（实现优先级）

| 序号 | 语法项 | 文档出处 | 实现状态 | 说明 |
|------|--------|----------|----------|------|
| 1 | **块内表达式语句 (expr;)** | 01-关键字.md：块为 `{ stmt; stmt; ... expr }`，表达式语句如 `f();`、`x = 1;` 需加分号 | ✅ 已实现 | ASTBlock 增加 expr_stmts/num_expr_stmts；parse_block 解析 (expr;)*；codegen/ast_free/typeck 已支持 |
| 2 | **void 返回类型** | 01/05：`extern function foo(...): void;`、返回类型须显式写出 | ✅ 已实现 | TOKEN_VOID、AST_TYPE_VOID；parse_type_name/typeck/codegen 已支持 void |
| 3 | **无参 return（return;）** | 01-关键字.md：`return expr;`、`return;` 均合法 | ✅ 已实现 | parser 允许 return 后无表达式；typeck 要求 return; 仅用于 void 函数；codegen 生成 return; |

## 二、实现要点

### 2.1 块内表达式语句 (expr;)

- **AST**：在 `ASTBlock` 中增加 `ASTExpr **expr_stmts`、`int num_expr_stmts`（及上限如 MAX_EXPR_STMTS）。
- **Parser**：在 labeled_stmts 循环之后、final_expr 之前：循环解析 “expr;”——若 `parse_expr` 成功且下一 token 为 `;`，则加入 expr_stmts 并 advance，否则将该 expr 视为 final_expr 并结束。
- **ast_block_free**：释放 `expr_stmts[i]` 及 `expr_stmts`。
- **Codegen**：在 labeled_stmts 与 final_expr 之间，对每条 expr_stmts 生成 `(void)(expr);`；collect_*_from_block / collect_var_names_from_block / dce_collect_from_block / collect_type_from_block 需遍历 expr_stmts。

### 2.2 void 类型与 return;

- **Token**：增加 `TOKEN_VOID`；lexer 中识别关键字 `void`。
- **AST**：增加 `AST_TYPE_VOID`。
- **Parser**：`parse_type_name` 中对 `TOKEN_VOID` 返回 void 类型；块内 “return;” 与 “return expr;” 均允许（return 后若紧跟 `;` 则 ret_expr 为 NULL）；表达式形式 `return` 同理（operand 可为 NULL）。
- **Typeck**：允许函数返回类型为 void；`return;`（operand 为 NULL）仅当返回类型为 void 时合法；`return expr;` 仅当返回类型非 void 时合法；main 仍须返回 i32 或 i64；对 void 函数不要求“显式 return”的 block_has_implicit_return 检查可放宽（void 时允许块尾无 return）。
- **Codegen**：返回类型为 void 时生成 C 的 `void` 与 `return;`；`c_type_to_buf`/`c_type_str` 对 AST_TYPE_VOID 输出 `"void"`。
