### C3=C4 根修（同 bug · mac L2 收口 · 待 Ubuntu 金标）

| 项 | 结论 |
|----|------|
| **C3 = C4** | T10 候选 C3（parser struct-lit）和 C4（unsafe-block）经二分确认是**同一个 bug** |
| **根因（最终）** | `parse_body_lets_into`（`compiler/src/parser/parser.x:3238`）**无 LBRACE handler**：bare `let p: Pair = { a: 0 };` 的 `{` 不匹配任何 init handler，let 被静默丢弃。后跟 `unsafe`/`while`/`for` → 整函数 P001。铁证：`let p: Pair = { a: 0 }; if (p.b != 1) {...}` 生成 C 里**没有 `p`**。`Pair { a: 0 }`（带前缀）走 IDENT→`parse_expr_into` 路径不坏 |
| **根修** | `compiler/src/parser/parser.x:3455` 新增 LBRACE handler：从 `{` 位置调 `parse_expr_into`（经已存在的 `parser_asm_primary_lbrace_looks_like_block_c` disambiguate struct-lit vs block-expr），与 IDENT 路径同样 lexer 状态收尾（copy + next + rewind）。PLATFORM: SHARED |
| **重生链路突破** | 之前误判本机无法重生 parser_gen.c。实测 `./shux-x -x -E -L .. -L src/lexer -L src/ast -E-extern src/parser/parser.x` **能重生**（同步约 3-4s），但重生产物含 parser.x 的 self-test `main()`（pinned seed 没有）。解法：重生后 strip 掉 `int32_t main(void){...}` 那 30 行，即得与 seed 同构的 library form → `parser_x.o` OK → g05 relink OK |
