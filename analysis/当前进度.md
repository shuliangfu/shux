## 下一波前排（按优先级排序）

### 1. C5 EXPR_ENUM_VARIANT fold（immediate next step）

**做什么**：cap-candidates-2026-07-19.md C5 子项「enum variant 常量折叠（match arm discriminant）」。enum variant 当前经 FIELD_ACCESS + `is_enum_variant=1` 表示（`Name.Variant` desugar），variant_index 由 typeck 填充。`const X: Color = Color::Red;` 应能在 typeck 期 fold 为 `const_folded_val = variant_index`（i32），让下游 `match X { ... }` 经 EXPR_MATCH handler 直接命中。

**根修路径（产生点单点）**：
1. [compiler/pipeline_glue.c](../compiler/pipeline_glue.c) `glue_typeck_fold_expr_ref` 加 EXPR_FIELD_ACCESS + `is_enum_variant=1` 分支：读 `field_access_is_enum_variant` + `field_access_offset`（variant_index 复用 offset 字段）→ stamp `const_folded_val = variant_index`，`const_folded_valid = 1`
2. `glue_is_const_expr_ref` 加同分支白名单
3. `pipeline_typeck_fold_expr_c` else-if 链加 EXPR_FIELD_ACCESS 分支
4. 新 lang-const case `c_enum_variant_const.x`（`enum Color { Red, Green, Blue } const X: Color = Color::Red; match X { .Red => 0; .Green => 1; _ => 2 }` want=0）+ baseline tsv 注册

**平台**：PLATFORM: SHARED（pipeline_glue.c 单点）

**验收**：mac L2 + Ubuntu L2；c_enum_variant_const exit=0；lang-const 15/15；bstrict 125 全绿

**升钉**：本子项也是 typeck CTFE 扩展非 ABI / 非 link 改动；不升钉
