# 阶段 1–2 支持的 .x 语法子集

本文档定义**阶段 1（AST + Lexer）与阶段 2（Parser）**要支持的最小 .x 语法子集，便于先打通「源码 → Token → AST」，再在阶段 3–4 接类型检查与代码生成。

---

## 一、目标

- **阶段 1**：Lexer 能对该子集产出正确 Token 流；AST 数据结构能表示该子集。
- **阶段 2**：Parser 能对该子集从 Token 流生成 AST；不要求类型检查与代码生成。

---

## 二、最小子集内容

### 2.1 程序结构

- **单文件**：仅一个顶层入口，无 `import`。
- **单一入口**：一个 **main 函数**，无参，返回类型为 `i32`（或先不写返回类型，由阶段 3 再约束）。

示例：

```su
function main() -> i32 {
    0
}
```

或（若阶段 1–2 暂不区分返回类型）：

```su
function main() {
    0
}
```

### 2.2 支持的语法元素

| 类别       | 内容 |
|------------|------|
| **关键字** | `function`、`main`、`i32`（或基础类型名）、`->`、`{`、`}`、`(`、`)` |
| **字面量** | 整数字面量（如 `0`、`42`） |
| **表达式** | 整数字面量；后续可加二元运算（如 `1 + 2`） |
| **语句**   | 表达式作为语句（函数体最后一式为返回值）；仅一条返回表达式即可 |
| **函数**   | 仅 `main`，无参，体为单一表达式 |

### 2.3 词法（Lexer 须识别的 Token）

- **关键字**：`function`
- **标识符**：`main`、以及后续扩展的变量名、类型名等
- **类型名**：`i32`（可视为关键字或标识符，由 Parser 区分）
- **字面量**：整数字面量（十进制）
- **符号**：`(`、`)`、`{`、`}`、`->`、`,`（预留）
- **空白与注释**：空格、换行、行注释 `//`（可选，阶段 1 可先忽略注释）

### 2.4 不包含（阶段 1–2 不实现）

- ~~`import`、模块、多文件~~（阶段 5 已支持顶层 `import path;`，path 为 `ident ( . ident )*`，如 `import("core.types");`）
- 变量声明（`let`）、赋值
- 其他函数、泛型、trait
- 字符串字面量、浮点、其他类型
- 控制流（if、for、while）

---

## 三、Hello World 的等价子集（阶段 4 用）

阶段 4 要达成「Hello World 可运行」，可在本子集上扩展：

- **方案 A**：main 体内允许一个**内置调用**，如 `print("Hello World")`；Lexer 识别 `print` 与字符串字面量 `"..."`，Parser 解析为「函数调用」节点。
- **方案 B**：main 仅返回 0，Codegen 对「已知为 main」的函数固定生成 `int main() { printf("Hello World\n"); return 0; }`，不依赖 .x 内写 print。

阶段 1–2 的 AST 只需能表示「main 返回一个字面量」即可；若采用方案 A，在阶段 2 末或阶段 3 增加「函数调用」与「字符串字面量」的 AST 节点与词法即可。

---

## 四、与开发时序的对应

- **阶段 0**：本文档即「语法子集」的确定（本文件）。
- **阶段 1**：AST 定义覆盖 2.1–2.2；Lexer 覆盖 2.3。
- **阶段 2**：Parser 覆盖 2.1–2.2，对合法子集产出 AST；错误恢复可最小化（报错带位置即可）。

后续阶段扩展子集时，在此文档或新文档中增补，并保持与 Lexer/Parser/AST 的同步更新。
