## 0. 一分钟必读（做错了全盘作废）

| # | 铁律 | 违反后的后果 |
|---|------|----------------|
| 1 | **工程轨绿 ≠ 产品轨绿** | 假「可自举」→ 下一波雪崩（14→15 号真例） |
| 2 | **只认本 SHA 链出的 `compiler/shux_asm`** | 旧 `shux_asm` / `shux_asm_stage1` / 残留 `shux_asm2` 全是假绿 |
| 3 | **谈自举 / 结案 / 放行 = L4 真冷 + 全量 bstrict** | 只删相关 `.o` 的 L2 **不能**写「真冷全绿」 |
| 4 | **金标机 = Ubuntu x86_64**（`ssh ubuntu-server`） | 只 mac 绿不得宣称产品放行；SHARED 改动须**双端** |
| 5 | **mac→Ubuntu 只 `git push` + `git pull`** | **禁止** scp 源码当同步 |
| 6 | **`git commit` 只在 macOS** | Ubuntu / Windows 上禁止 commit |
| 7 | **禁止 `cmd \| head` 当成功** | 必须看 **exit 码** + 运行结果 |
| 8 | **禁止 BEST_EFFORT / soft-skip 当绿** | `SHUX_W3_BSTRICT_BEST_EFFORT` 等只作调试，**不能写进放行** |

**口诀**：

```text
日常 = 本波 shux_asm + 本债探针 +（动产品面）产品矩阵（L2/L3 可）
结案 = L4 全擦全部 .o + 重链 shux/shux_asm/shux-c + 矩阵 + 全量 run-all-bstrict
放行自举 = 双端（至少 Ubuntu 金标 + SHARED 时 mac）L4 真冷全测已绿 + 本 SHA 日志
```

| 方向 | 正确 | 禁止 |

| 提交 | **仅 macOS** | 远端 `git commit` / 强推历史 |

```bash
# Ubuntu（验证前）

# Ubuntu → mac 只拉证据
# scp ubuntu-server:/tmp/ubuntu_true_bstrict_*.log /tmp/
```

## 2. 两条轨道（永远分清）

| 轨道 | 测什么 | 典型命令 / 绿点 | 能否单独说「可自举」 |
|------|--------|-----------------|----------------------|
| **工程轨** | hybrid surface、seed 冷启动、Stage2 子集 | `prove_module_selfhost` IDENTICAL、R2 H=0、return-value=42、`bootstrap-verify` | **否** |
| **产品轨** | 本波 `shux_asm` 编 **用户/测试程序** | 产品矩阵（§4）+ **`tests/run-all-bstrict.sh` exit 0** | **必要** |

- 工程轨可以写进度 KPI。  
- **产品轨红时禁止**写「可自举 / 本 SHA 放行 / 真冷全绿」。  
- Stage2 / `shux_asm2` 是 freestanding 子集，**禁止**冒充产品 bstrict 主机。

## 3. 构建级别 L0～L4（写进汇报，缺则作废）

| 级 | 含义 | 能否当放行 |
|----|------|------------|
| **L0** | 旧 `shux_asm` / `shux_asm_stage1` / 未 pull 二进制 | **禁止** |
| **L1** | 热 g05 relink，不删 `.o` | **禁止**当矩阵/结案 |
| **L2** | 只 `rm` **相关** `.o` → make/g05 | **日常探针最低**；**不能**写真冷 |
| **L3** | `bootstrap-driver-seed` + g05（std/core 可能残留旧 `.o`） | 开发端到端；谈自举前升 **L4** |
| **L4 真冷** | **全擦** `compiler|std|core` 下**全部** `*.o` + 删产品二进制 → pin seed → bootstrap → g05 → 矩阵 → bstrict | **结案 / 谈自举 / 双端放行默认标准** |

### 3.1 为何必须全擦（L4）

| 残留 | 会藏什么 |
|------|----------|
| 旧 `runtime_*_glue.o` / OS glue | ensure 从未跑；全擦后才 UNDEF |
| 旧 `std/**/*.o` / `core/**/*.o` | 链接语义与本 SHA 生成 C 不一致 |
| 旧 `parser_x.o` / `typeck_x.o` / `pipeline_x.o` | 前端债被旧目标盖住 |
| 旧 `shux` / `shux_asm` / `shux-c` | 验的是上一 SHA（14 号假绿） |

**禁止**把「只删了 parser/codegen 再 g05」或「树里还有 100+ 旧 `.o`」叫做真冷测。

## 4. 日常真测（平时怎么测）

> **何时用**：每个微 commit、单点探针、「继续下一步」日常债。  
> **级别**：L2 或 L3。  
> **禁止写**：「真冷全绿」「可自举」「本 tip 双端放行」。

### 4.1 日常真测最小流程（mac 或 Ubuntu 同构）

```bash
cd /path/to/shux   # mac 或 Ubuntu 仓库根
git rev-parse --short HEAD

# --- A. 只动了相关产物：L2 ---
# 例：改了 runtime_link_abi.from_x.c
cd compiler
# 按改动删相关 .o（示例，按实际文件改）
rm -f runtime_link_abi.o
# 若动 seed 产品面：
# cp -f seeds/codegen_gen.linux.x86_64.c codegen_gen.c
# cp -f seeds/parser_gen.linux.x86_64.c parser_gen.c
# cp -f seeds/typeck_gen.linux.x86_64.c typeck_gen.c
# make parser_x.o codegen_x.o   # 只用 Makefile，禁止裸 gcc -c mega 源

FULL=0 bash scripts/g05_prepare_and_relink.sh
# g05 若缺 mangled hybrid 符号：force make 对应 .o 再 FULL=0 g05 一次
# 例：make lsp_diag.o backend_enc_dispatch.o seed_link_compat.o 等

test -x shux_asm
export SHUX="$(pwd)/shux_asm"
ls -la shux shux_asm shux-c 2>/dev/null || true
cd ..
```

### 4.2 本债探针（按改动裁剪，必有 exit 码）

```bash
SHUX=./compiler/shux_asm

# 本波相关模块（示例：改 time 入链 → wipe + time 测）
# 改 link ensure → pure/hello 不拖未用 heavy std 的探针
# 改 typeck → tests/types/overload.x 等邻域

# 永远至少确认主机活：
$SHUX -o /tmp/rv tests/return-value/main.x > /tmp/rv.log 2>&1
echo rv_build:$?
/tmp/rv; echo rv_run:$?   # 必须 0 且 42
```

### 4.3 动产品面时：跑 §5 产品矩阵

凡改 **codegen / parser / typeck / std / runtime link / glue / seed pin** 之一：

- **必须**跑完整产品矩阵（§5）  
- 矩阵红 → 进度只报红；**禁止**只写「本模块 prove 绿」

### 4.4 日常真测成功标准

| 项 | 标准 |
|----|------|
| `SHUX` | 本波 g05 刚链出的 `./compiler/shux_asm` |
| 探针 | 本债相关用例 exit 符合预期 |
| 产品面 | §5 矩阵绿（若本波动产品） |
| 汇报 | 写清 **构建级别=L2 或 L3**、SHA、日志路径 |
| 结案话术 | **禁止**「真冷 / 可自举」 |

### 4.5 常见 L2 重建对照（按改动选）

| 改了什么 | 至少做什么 |
|----------|------------|
| `runtime_link_abi*.c` / ensure / fk0 | `rm` 对应 `.o` + g05；探针：pure + 相关 std 模块 + hello |
| `pipeline_glue*.c` / `ast_pool.c` | Ubuntu 重建 `pipeline_x.o` + g05 |
| `pipeline_glue_strict_minimal` seed | 重建 `build_asm/pipeline_glue_strict_minimal.o` + g05 |
| `seeds/*_gen.linux.x86_64.c` + 镜像 `.x` | **同 commit**；`cp` seed → make 对应 `*_x.o` + g05 |
| `std/**/*.x` | 仅该模块用例 + 邻域；**禁止**批量改 std 顶编译器债 |
| 仅进度/注释 | 可不重建；**不要**假装跑过矩阵 |

## 5. 产品矩阵（动产品面必跑）

> 在仓库根执行。`SHUX` 必须指向本波 `shux_asm`。  
> **禁止** `cmd | head` 判成功；**必须**打印 build/run exit。

```bash
cd /path/to/shux
export SHUX=./compiler/shux_asm
test -x "$SHUX"
echo "SHA=$(git rev-parse --short HEAD) SHUX_mtime=$(stat -f '%Sm' -t '%Y-%m-%d %H:%M' "$SHUX" 2>/dev/null || stat -c '%y' "$SHUX")"

# 1) 无 import 基线
$SHUX -o /tmp/rv tests/return-value/main.x > /tmp/rv.log 2>&1
echo rv_build:$?
/tmp/rv; echo rv_run:$?
# 期望：rv_build=0，rv_run=42

# 2) import + 调用（core）
$SHUX -o /tmp/opt tests/option/main.x > /tmp/opt.log 2>&1
echo opt_build:$?
# 期望：0；再运行 /tmp/opt 对照该测试意图

# 3) 多模块 import
$SHUX -o /tmp/si tests/stdlib-import/main.x > /tmp/si.log 2>&1
echo si_build:$?
# 期望：0

# 4) 用户 hello（stdout 语义）
$SHUX -o /tmp/hello examples/hello.x > /tmp/hello.log 2>&1
echo hello_build:$?
if [ -x /tmp/hello ]; then
  /tmp/hello > /tmp/hello.out
  echo hello_run:$?
  od -An -tx1 /tmp/hello.out | head -2
  # 期望含 0a（真换行）；禁止 5c 6e（字面 \n 未解码）
fi

# 5) 生成 C 抽检（-E 成功时）
$SHUX -E examples/hello.x > /tmp/he.c 2> /tmp/he.err
echo he:$?; wc -c /tmp/he.c   # 不能 0
grep -nE 'std_fmt_print\(|print_u8_ptr|Context _rc = 0|out->data|extern ssize_t write\(' /tmp/he.c | head || true
```

### 5.1 矩阵红灯（任一项 = 本波失败）

| 现象 | 含义 |
|------|------|
| `rv_run != 42` | 主机挂了，先修再谈一切 |
| option / si / hello **build ≠ 0** | 用户 import 路径红 |
| `pipeline error[XP003]`、`-E` 0 字节 | typeck/pipeline 硬失败，禁止当 soft |
| 生成 C：`struct … = 0` / 值类型 `out->` / bare `extern write` | codegen/stdlib/ABI 红 |
| 依赖 BEST_EFFORT / 旧 stage1 才绿 | **假绿** |
| 只改 `.x` 未改产品 seed（或反之）却说已修 | **双权威假绿** |

### 5.2 邻域矩阵（按债裁剪，防修坏别处）

| 改动类型 | 至少再绿 |
|----------|----------|
| overload / typeck 评分 | `tests/types/overload.x`、atomic 多宽整数、既有 PTR 重载（如 sort） |
| parser / unsafe region | 既有 unsafe 赋值探针 |
| codegen emit / init_globals | rv=42、option、hello；无「坏函数 let 升 top-level」 |
| link_abi / std.o 入链 | pure 不拖未用 heavy std；相关模块 wipe/ensure 探针 |
| SHARED ABI / mangle | **mac + Ubuntu** 都要矩阵（mac `-dead_strip` 会藏 UNDEF） |

## 6. 全量 bstrict（产品闸门套件）

约 **125** 个白名单脚本（含 `run-void-main-gate` + `run-comment-prefix`；历史钉盘日志可能仍写 123）。成功日志须类似：

```text
run-all-bstrict OK (125 scripts, …)
```

且 **进程 exit 0**。

### 6.1 正确跑法（验**已冷出/已 g05 的**产品二进制）

```bash
cd /path/to/shux
# 确认无假绿 env（调试 env 不得用于放行）
env | grep -E 'BEST_EFFORT|SKIP_BUILD|VIS=|W3_BSTRICT' || true

# 关键：先用本波 L4/L3 产出 shux_asm，再：
# SKIP_BUILD=1 → 只 cp shux_asm → shux，避免 bstrict 默认 bootstrap 盖掉刚冷出的二进制
# 默认会 cp 产品 shux_asm（不是 Stage2 的 shux_asm2）
export SHUX=./compiler/shux_asm
SHUX_BSTRICT_SKIP_BUILD=1 bash tests/run-all-bstrict.sh 2>&1 \
  | tee /tmp/bstrict_$(uname -s)_$(git rev-parse --short HEAD).log
echo bstrict_exit:$?
tail -20 /tmp/bstrict_$(uname -s)_$(git rev-parse --short HEAD).log
```

### 6.2 关键禁止

| 禁止 | 原因 |
|------|------|
| 不设 `SHUX_BSTRICT_SKIP_BUILD` 却以为在验「刚 L4 的 shux_asm」 | 脚本可能 `bootstrap-driver-bstrict` **重编盖掉** |
| `SHUX_BSTRICT_USE_ASM2=1` 当产品放行 | Stage2 freestanding 冒充产品 → SEGV/假红/假绿 |
| `SHUX_W3_BSTRICT_BEST_EFFORT=1` 当绿 | soft continue，**不是**严格闸门 |
| 日志是**别的 SHA** 的 tail | 只认 **本 SHA** 文件名与 `git rev-parse` 一致 |

### 6.3 全量 bstrict 何时必须跑

| 场景 | 是否必须 |
|------|----------|
| 日常微 commit L2 探针 | **否**（过重）；矩阵够即可 |
| 一波整体功能收口 | **是**（配 L4） |
| 改 glue / link ensure / seed pin / SHARED ABI | **是**（本波即可，可不等「叙事整波」） |
| 谈自举 / 升 tip L4 钉盘 / 双端结案 | **是 + 双端** |

## 7. L4 真冷全测（怎么测「真冷」）

> **定义**：`remaining_o=0` 起编 + 本波重链 `shux`/`shux_asm`/`shux-c` + §5 矩阵 + §6 全量 bstrict。  
> **不是**：只 `rm` 相关 `.o`、只 bootstrap 不擦 std、不跑 bstrict。

### 7.1 Ubuntu 金标（照抄）

```bash
# —— mac 已 push 本波 ——
ssh ubuntu-server
cd /home/shuliangfu/worker/shu/shux
git pull --ff-only origin self-hosting
SHA=$(git rev-parse --short HEAD)
echo "SHA=$SHA"

# 1) 全擦 .o
find compiler std core -name '*.o' -type f -delete
# 2) 全擦产品二进制（必须本波重链）
rm -f compiler/shux compiler/shux_asm compiler/shux-c compiler/bootstrap_shuxc \
      compiler/shux-x compiler/shux-seed-phase1 compiler/shux-no-c-frontend \
      compiler/bootstrap_shux compiler/shux_asm2 2>/dev/null || true
# 3) 确认干净
test "$(find compiler std core -name '*.o' 2>/dev/null | wc -l | tr -d ' ')" -eq 0
test ! -e compiler/shux_asm
echo "remaining_o=0 OK"

cd compiler
# 4) pin 产品 seed（Linux 金标源；mac 开发也 pin 同一语义）
cp -f seeds/codegen_gen.linux.x86_64.c codegen_gen.c
cp -f seeds/parser_gen.linux.x86_64.c parser_gen.c
# typeck_gen.c is host-local/gitignored; stale file survives .o wipe and rejects void main (-4).
cp -f seeds/typeck_gen.linux.x86_64.c typeck_gen.c

# 5) 冷启动 + 产品链（勿裸 gcc -c mega）
# 资源：优先 -j1/-j2；可 tee 日志
make bootstrap-driver-seed 2>&1 | tee /tmp/ubuntu_true_cold_${SHA}.log
FULL=0 bash scripts/g05_prepare_and_relink.sh 2>&1 | tee -a /tmp/ubuntu_true_cold_${SHA}.log

ls -la shux shux_asm shux-c bootstrap_shuxc
export SHUX="$(pwd)/shux_asm"
cd ..

# 6) 产品矩阵（§5）→ 落盘
{
  echo "=== matrix $SHA ==="
  $SHUX -o /tmp/rv tests/return-value/main.x; echo rv_build:$?
  /tmp/rv; echo rv_run:$?
  $SHUX -o /tmp/opt tests/option/main.x; echo opt_build:$?
  $SHUX -o /tmp/si tests/stdlib-import/main.x; echo si_build:$?
  $SHUX -o /tmp/hello examples/hello.x; echo hello_build:$?
  /tmp/hello > /tmp/hello.out; echo hello_run:$?
} 2>&1 | tee /tmp/ubuntu_l4_matrix_${SHA}.log

# 7) 全量 bstrict（勿让 bootstrap 盖掉刚冷出的二进制）
SHUX_BSTRICT_SKIP_BUILD=1 SHUX=./compiler/shux_asm \
  bash tests/run-all-bstrict.sh 2>&1 \
  | tee /tmp/ubuntu_true_bstrict_${SHA}.log
echo BSTRICT_RC:$?
grep -E 'run-all-bstrict OK|FAIL|error' /tmp/ubuntu_true_bstrict_${SHA}.log | tail -30
```

## 8. 双端测试（怎么测「双端」）

### 8.1 标准顺序（推荐）

```text
1. mac：改代码 → 日常真测（L2/L3 + 探针/矩阵）
2. mac：git commit（单主题）+ git push
3. Ubuntu：git pull --ff-only → 同级别验收
   - 日常债：L2/L3 + 矩阵
   - 结案 / SHARED / ensure：L4 + bstrict
4. 若 SHARED 或谈自举：mac 再跑 L4 + bstrict（可与 Ubuntu 串行，忌无界并行叠跑吃光内存）
5. 双端日志齐绿 → 才可写「双端 L4 放行」；钉盘 SHA 写入 自举进度.md
```

### 8.2 双端最低对照表

| 改动属性 | mac | Ubuntu | 备注 |
|----------|-----|--------|------|
| 仅文档 / 进度 | 可选 | 可选 | 不改二进制语义 |
| 单端探针债、非 SHARED | 开发机 L2 | **金标至少矩阵** | 谈放行仍要 Ubuntu |
| **PLATFORM: SHARED** | L2 矩阵 + **结案 L4** | **L4 + bstrict** | 禁 mac dead_strip 假绿 |
| link ensure / glue / seed pin | 本波倾向 L4 | **强制 L4** | 旧 `.o` 最易藏 |
| 谈自举 / tip L4 钉盘 | **L4 + 123 bstrict** | **L4 + 123 bstrict** | 缺一侧 → 不放行 |

### 8.3 双端假绿对照（历史）

| 现象 | 真因 | 正确验收 |
|------|------|----------|
| mac `run-net` 绿，Ubuntu `undefined net_accept_many_c` | SHARED mangle；mac dead_strip 藏 UNDEF | Ubuntu 全链为真值 |

## 9. 验证节奏（何时日常、何时真冷）

| 触发 | 跑什么 |
|------|--------|
| 每个微 commit / 「继续下一步」单债 | **日常真测**（L2/L3 + 探针；动产品面加矩阵） |
| **完成一个整体功能**（一波 Cap / 一批 R2 / M 闭环 / 矩阵红→绿结案） | **真冷全测** L4 + bstrict（Ubuntu；SHARED 双端）← **主闸门** |
| 改 glue / link ensure / seed pin / SHARED ABI | **本波即 L4**（可不等整波叙事） |
| 当天动过产品面、收工前还没 L4 | **日终兜底** L4+bstrict |
| 谈自举 / 升钉盘 / 「可自举」叙事 | **强制**本 SHA 双端 L4 真冷全测 |

**禁止**：

- 每个 bare-wrap commit 都全量 L4+123（过重）  
- **只**靠「晚上跑一次」、白天全程工程轨假绿（14 号结构）  
- 把 L2 写进「真冷全绿」

**什么叫整体功能**（触发主闸门）：

| 算 → 真冷 | 不算 → 日常即可 |
|-----------|-----------------|
| Cap **整波**收口 | 单 commit 包几个 bare call |
| 一条 R2 rest 真迁完 + 金标 | 只改进度文 / 注释 |
| 产品矩阵红→宣称绿 | 矩阵仍红时的中间步 |
| M2 替换链上 smoke 结案 | 本地试 `-E` mega 源 |
| 任意「可进下一阶段自举」 | 工程轨 thin N+1 |

## 10. 按场景速查（AI 接到任务时）

| 用户说 / 任务 | 你应该测 |
|---------------|----------|
| 「继续下一步」+ 单点产品债 | L2 相关 `.o` + g05 + 本债探针 + 矩阵；**双端至少 Ubuntu 同 L2**；**不**默认 tip L4 |
| 「这一波收口 / Cap 做完」 | **L4 + 矩阵 + 全量 bstrict**；SHARED → 双端 |
| 「可以自举了吗 / 放行」 | 查 tip 钉盘 SHA 的**双端 L4 日志**；无则**现在跑**，禁止用旧日志 |
| 「只改了文档」 | 可不测二进制；勿谎报矩阵绿 |
| 矩阵红 | 只报红；先归层（parser/typeck/codegen/link）根修；禁改测试期望糊绿 |
| g05 链失败缺符号 | force `make` 缺失 mangled `.o` 再 `FULL=0 g05`；**不要** naked gcc -c mega |
| Ubuntu 红、mac 绿 | **以 Ubuntu 为准**查 UNDEF / 全链；查是否 dead_strip 假绿 |

## 12. 汇报模板（强制）

日常真测：

```text
SHA：…
构建级别：L2 / L3（日常真测；非 tip L4）
平台：macOS=? / Ubuntu=?
SHUX：./compiler/shux_asm mtime=…
本债探针：…（exit）
产品矩阵：rv=42 / option=? / hello=? / si=?
bstrict：未跑 / 局部（写明）
下一波：…
```

真冷 / 双端结案：

```text
SHA：…
构建级别：L4 真冷（全擦 compiler|std|core .o）
.o 擦除：remaining_o=0？
二进制：shux / shux_asm / shux-c 本波 mtime？
平台：
  Ubuntu：cold 日志=… matrix=… bstrict=… EXIT=?
  macOS： cold 日志=… matrix=… bstrict=… EXIT=?
产品矩阵：rv=42 / option=? / hello=? / si=?
bstrict：125/125？ 须见 run-all-bstrict OK
钉盘：是否升 tip L4 钉盘（默认不升除非用户要求）
工程轨（可选）：prove/R2=…
下一波：…
```

**缺「构建级别」或 `SHUX=stage1` → 验收作废。**  
**谈自举却不是 L4 或 remaining_o≠0 → 验收作废。**

## 14. 给其它编辑器 AI 的默认行为

1. **接到自举/产品任务**：先读本文 §0～§3，再动代码。  
2. **改代码前**：确认测的是产品轨还是工程轨；写清将用 L2 还是 L4。  
3. **改完后**：按 §10 场景表跑测；日志落 `/tmp/*` 带 SHA。  
4. **禁止**自创「全仓库 make test 绿就过」；本仓库放行锚点是 **矩阵 + bstrict + 正确二进制**。  
5. **红则只报红**；回产生点根修（见 `AGENTS.md` 根源治理）；禁止 soft-skip / 改期望。  
6. **用户未要求 tip L4** 时：日常债用 L2/L3；结案话术前再提议或执行 L4。  
7. **commit 仅 mac**；Ubuntu 只 pull + 测；远端失败用 scp 日志回 mac。  

**最后一句**：  
**绿必须是「本 SHA · 正确构建级别 · 正确主机 · 正确轨道」的绿；否则等于没测。**
