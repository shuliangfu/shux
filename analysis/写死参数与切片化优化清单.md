# 写死参数与切片化优化清单

本文档列出 std 中「写死」的参数/段数/上限，以及可改为「指针+长度/段数」切片式 API 的项，便于与 Zig/Rust 对齐并减少魔数。

---

## 一、已切片化（指针+段数）的 API

| 模块 | API | 说明 |
|------|-----|------|
| std.io | `io_read_batch_buf` / `io_write_batch_buf` (C) | (fd, bufs, n), n 1..16 |
| std.io.driver | `submit_read_batch_buf` / `submit_write_batch_buf` | (handle, bufs: *Buffer, n, timeout_ms) |
| std.net | `stream_read_batch_buf` / `stream_write_batch_buf` | (stream, buffers: *Buffer, n, timeout_ms) |
| std.io.mod | `read_batch_fd_buf` / `write_batch_fd_buf` | (fd, buffers: *Buffer, n), n 1..16 |
| std.io.driver | `register_fixed_buffers_buf(bufs: *Buffer, nr: u32)` | nr 1..8；C 侧 io_register_buffers_buf |
| std.io.mod | `register_fixed_buffers_buf(bufs: *Buffer, nr: u32)` | 同上，对外 API |
| std.net | `net_batch_max(): i32` | 返回 64，与 C IO_NET_BATCH_MAX 一致，去魔数 |
| std.fs | `fs_readv_buf` / `fs_writev_buf` | (fd, bufs: *FsIovecBuf, n)，C 侧 fs_readv_buf_c/fs_writev_buf_c，n 1..16 |
| std.net | `udp_recv_many_buf` / `udp_send_many_buf` | (sock, bufs: *Buffer, n, …) / (sock, addrs, ports, bufs, n)，C 侧 recvmmsg/sendmmsg，n 1..8 |

---

## 二、仍写死、可切片化的项

### 2.1 读写批量（readv/writev 风格）

| 位置 | 当前 | 建议 |
|------|------|------|
| **std/io/mod.x** | `read_batch_fd` / `write_batch_fd`：4 段 (p0,l0)..(p3,l3) + n | ✅ 已增加 `read_batch_fd_buf` / `write_batch_fd_buf(fd, buffers, n)`。 |
| **std/fs/mod.x** | 同上，`read_batch_fd` / `write_batch_fd` 同上 | 用户 import("std.io") 即可用 read_batch_fd_buf/write_batch_fd_buf；fs 已 import("std.io")。 |
| **std/fs** | `fs_readv2`(p0,l0,p1,l1)、`fs_readv4`(p0..p3)、`fs_writev2`/`fs_writev4` | ✅ 已增加 `fs_readv_buf(fd, bufs: *FsIovecBuf, n)` / `fs_writev_buf`，C 侧 `fs_readv_buf_c`/`fs_writev_buf_c`，n 1..16。 |

### 2.2 固定 buffer 池注册（io_uring fixed buffers）

| 位置 | 当前 | 建议 |
|------|------|------|
| **std/io/driver.x** | `register_fixed_buffers(p0,l0, p1,l1, p2,l2, p3,l3, nr)`，nr 1..4 | ✅ 已增加 `register_fixed_buffers_buf(bufs: *Buffer, nr: u32)`，nr 1..8。 |
| **std/io/core.x** | `io_register_buffers_4` 4 对参数 | driver 直接 extern C，无需 core 改。 |
| **std/io/io.c** | `io_register_buffers` 8 对 (p0..p7) | ✅ 已增加 `io_register_buffers_buf(shu_batch_buf_t *bufs, int nr)`。 |

### 2.3 网络批量 accept/connect

| 位置 | 当前 | 建议 |
|------|------|------|
| **std/net/mod.x** | `accept_many` / `connect_many` 内部 `fd_buf: i32[64]`，`n_req = min(out.length, 64)` | ✅ 已增加 `net_batch_max(): i32` 返回 64，与 C IO_NET_BATCH_MAX 一致；内部仍用 64 字面量以避免 parser 对 if 内 cast 的限制。 |
| **std/net/net.c** | `IO_NET_BATCH_MAX` 64 | 已按 n 由调用方传入，无需改。 |

### 2.4 UDP 批量收/发

| 位置 | 当前 | 建议 |
|------|------|------|
| **std/net** | `udp_recv_many`(p0,l0, p1,l1, n 1..2)、`udp_send_many`(a0,port0,p0,l0, a1,port1,p1,l1, n 1..2) | ✅ 已增加 `udp_recv_many_buf(sock, bufs: *Buffer, n, timeout_ms, out_sizes, out_addrs, out_ports)` 与 `udp_send_many_buf(sock, addrs, ports, bufs, n)`；C 侧 recvmmsg/sendmmsg，n 1..8。 |

### 2.5 其他（保持或仅文档化）

| 位置 | 说明 |
|------|------|
| **std/io** | `wait_readable(fds: *i32, n: i32, ...)` 已是「指针+数量」，无需改。 |
| **std/mem, std/string, std/vec** | copy/set/compare、append_slice、from_slice 等已是 (ptr, n) 或切片语义，无需改。 |

---

## 三、实现优先级

1. **高**：`read_batch_fd_buf` / `write_batch_fd_buf`（std/io + std/fs）— 仅 .x 包装，无 C 改动。
2. **高**：`register_fixed_buffers_buf(bufs, nr)` — driver + core + io.c 增加，统一 4/8 块为指针+数量。
3. **中**：`fs_readv_buf` / `fs_writev_buf` — fs.c 新增 + mod.x 包装，与 io 的 batch_buf 语义一致。
4. **中**：std/net 将 64 提为 `NET_BATCH_MAX` 常量。
5. **低**：UDP many_buf — ✅ 已实现：net_udp_recv_many_buf_c / net_udp_send_many_buf_c（n 1..8），std/net mod.x 暴露 udp_recv_many_buf / udp_send_many_buf。

---

## 四、常量汇总（去魔数）

| 常量名 | 值 | 位置 |
|--------|-----|------|
| IO_READV_BUF_MAX | 16 | std/io/io.c（已存在） |
| IO_NET_BATCH_MAX | 64 | std/io/io.c（net 用）, std/net 可暴露为 NET_BATCH_MAX |
| IO_FIXED_MAX | 8 | std/io/io.c（fixed buffer 池块数） |

.x 侧若需使用，可在对应 mod 中写注释说明与 C 常量一致，或通过常量函数/只读变量暴露（视语言能力而定）。
