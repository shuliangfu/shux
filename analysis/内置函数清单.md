# Shux 内置函数清单（分析文档）

本文档列出**所有计划的内置函数与内建能力**，按类别与阶段整理，便于实现时**不遗漏**、不重复。开发内置功能前可先在此表内勾选并补充签名与实现状态。

---

## 一、概念区分

| 类型 | 说明 | 示例 |
|------|------|------|
| **编译器内建** | 由 codegen/driver 直接生成，不在 .sx 中显式调用 | 当前 main 固定生成 `printf("Hello World\n");`、main 的 return 值生成 C 的 `return expr;` |
| **语言内建（intrinsic）** | 编译器识别的特殊名字，在 .sx 中可调用，生成时映射到 C/IR 或内联实现 | 未来：`size_of::<T>()`、`assert(cond)`、`unreachable()` |
| **标准库必须 API** | 通过 `import("core.xxx")` / `import("std.xxx")` 提供，语义上「必须实现」才能自举 | `core.mem.size_of`、`std.process.exit`、`std.io.print` 等 |

本文档将「语言内建」与「标准库必须 API」中**与内置能力重叠**的部分统一列为**内置函数**，并注明：由**编译器直接生成**还是由 **core/std 模块实现**。

---

## 一（续）、自举前必要 vs 自举后再实现

**自举**：用 .sx 写编译器，由当前 C 版 shux 编译，产出的新 shux 能编译自身。为实现自举，需要「写编译器 + 跑测试」所必需的最小内置与库；其余可**自举后再实现**，避免首轮范围过大。

| 分类 | 说明 | 实现顺序 |
|------|------|----------|
| **自举前必要** | 写编译器源码、驱动、测试时**必须用到**的内置与 core/std API；缺了则无法完成自举。 | **先实现** |
| **自举后再实现** | 自举阶段可不依赖，或仅少量场景需要；自举完成后再补全，便于优化与扩展。 | 自举成功后按需实现 |

下文各表中用列 **「自举」** 标注每项归属：**必要** = 自举前必要；**后** = 自举后再实现。

### 2.2 阶段 5–6：core 与 std 最小子集（建议优先实现的内置）

以下为 **core** / **std** 中与「内置能力」直接相关的函数；实现时可选择：**编译器内建**（解析到名字后直接生成 C/IR）或 **core/std 的 .sx 实现**（需先有函数调用解析与 import）。**「自举」列：必要 = 自举前必要，后 = 自举后再实现。**

| 所属 | 名称 | 建议签名/行为 | 实现方式建议 | 自举 | 状态 |
|------|------|----------------|--------------|------|------|
| core.types / core.mem | `size_of` | `size_of::<T>() -> i32` 或 `size_of(T) -> i32` | 编译器内建 或 core.mem 实现后 import | **必要** | ⬜ |
| core.types / core.mem | `align_of` | `align_of::<T>() -> i32` 或 `align_of(T) -> i32` | 同上 | **必要** | ⬜ |
| core.debug | `assert` | `assert(cond: bool)`，假时 abort/panic | 编译器内建 或 core.debug | **必要** | ⬜ |
| core.debug | `assert_eq` | `assert_eq(a, b)`，不等时 abort | core.debug | **必要** | ⬜ |
| core.builtin | `unreachable` | `unreachable()`，标记不可达，可生成 `__builtin_unreachable()` | 编译器内建 | **必要** | ⬜ |
| core.mem | `copy` / `copy_nonoverlapping` | 内存块复制，对应 memcpy | core.mem | **必要** | ⬜ |
| std.process | `exit` | `exit(code: i32) -> never` | std 实现，或编译器识别后生成 `exit(code)` | **必要** | ⬜ |
| std.process | `args` | 命令行参数（需类型如 `string[]` 或迭代器） | std 实现 | **必要** | ⬜ |
| std.io（或内置） | `print` / `println` | `print(s: str)` 或 `println(s: str)` | 编译器内建 或 std.io；需先有字符串类型 | **必要** | ⬜ |
| std.io / std.fs | 标准输入/读行、文件 open/read | 读 stdin、读源文件 | std 实现 | **必要** | ⬜ |
| core.alloc / std.heap | `alloc` / `dealloc` 或 `Box` | 堆分配（编译器/运行时数据结构） | core 或 std 实现 | **必要** | ⬜ |
| core.fmt（最小） | 格式化输出 | `format!("{}", x)` 或简单拼接，供错误信息/诊断 | core.fmt 最小子集 | **必要** | ⬜ |
| std.runtime | `panic` | `panic(msg)`，终止并打印 | std 实现 或 编译器内建 | **必要** | ⬜ |
| core.builtin / core.mem | `clone` 等 | 按 core.mem 设计（若与 copy 分开） | core.mem | 后 | ⬜ |

## 三、按「实现位置」分类（便于分工）

| 实现位置 | 函数/能力 | 说明 |
|----------|-----------|------|
| **codegen 直接生成** | Hello World printf、main return、未来可能的 `unreachable()`、`assert(cond)` | 不依赖 .sx 标准库，编译器看到固定名字或 AST 节点即生成 C |
| **编译器解析 + 内建表** | `size_of`、`align_of`、`assert`、`exit`（若选为内建） | Parser/Typeck 识别为内建后，codegen 查表生成对应 C/IR |
| **core.xxx / std.xxx 的 .sx** | `core.mem.size_of`、`core.debug.assert`、`std.process.exit`、`std.io.print` | 需先有：函数声明、import、多模块 codegen；实现写在 .sx 中，可能内部再调 C 或内建 |

建议：**先定一份「最小内建集合」**（如 `assert`、`exit`、`size_of`、`print` 四类），再决定每项是「纯 codegen 内建」还是「core/std 提供」。避免遗漏的方式：**本表每实现一项即把状态从 ⬜ 改为 ✅，并注明实现方式**。

## 四、最小内建集合与自举前实现顺序

**自举前**只实现「二（续）、自举前必要汇总」表中的 11 项；首轮可按下述顺序（与语法/类型能力扩展一致）：

1. **`print(s: str)` / `println(s: str)`** — 替代写死 Hello World；依赖字符串字面量、函数调用 AST。
2. **`exit(code: i32)`** — 程序退出码；依赖函数调用。
3. **`assert(cond: bool)` / `assert_eq(a, b)`** — 测试与不变式；依赖函数调用、布尔或 i32。
4. **`size_of` / `align_of`** — 类型布局；依赖泛型或类型参数（可随泛型一起做）。
5. **`unreachable()`** — 不可达分支；编译器内建即可。
6. **`copy` / `copy_nonoverlapping`** — core.mem，对应 memcpy。
7. **`args`** — 命令行参数；依赖字符串/切片或迭代器。
8. **标准输入、文件 open/read** — std.io / std.fs。
9. **`alloc` / `dealloc` 或 `Box`** — 堆分配。
10. **core.fmt 最小** — 错误信息/诊断用格式化（可先简单拼接）。
11. **`panic(msg)`** — 致命错误终止并打印。

**自举后**再实现 2.3 / 2.4 / 2.5 及 2.2 中标「后」的项，避免首轮范围过大。
