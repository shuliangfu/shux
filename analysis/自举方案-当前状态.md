## ARM64 ABI 问题分析

### 根因

parser.sx 中使用 `LexerResult` 和 `OneFuncResult` 的结构体按值返回/赋值在 ARM64 上存在 ABI 不稳定性。

| 模式 | 问题 |
|------|------|
| `r = lexer_next(lex, source)` | 返回 `LexerResult` 按值，部分字段可能未写回 |
| `return OneFuncResult { ... }` | 大结构体（~400字节）按值返回不可靠 |
| `res = parse_one_function(...)` | `OneFuncResult` 赋值可能丢失字段 |

### 当前状态

parser.sx 已经**部分**改用 _into API：
- `lexer_next_into(&r, lex, source)` ✅ 已用
- `lex_from_next_into(out, r)` ✅ 已有辅助函数
- `set_onefunc_fail(out, next_lex)` ✅ 已有
- `copy_onefunc_into(dst, src)` ✅ 已有

但还有**需注意的兼容返回值**（非主路径，优先改用 *_into）：`diag_*`/`body_skip*`/`skip_*` 等仍为 `LexerResult`，文档仅作提醒以便审计。

- `lex_from_onefunc_next_into(out, res)` ✅ 接 `*OneFuncResult`，避免整颗 OneFuncResult **按值传入**（ARM64 大结构 ABI）；与 `lex_from_next_into` 只读 `Lexer` 子域不同，后者仍常接临时的 `LexerResult`。

### `-sx` 入口预处理与缓冲

- **`shux_sx`/driver `-sx`**：入口与各 dep 预处理须走 **`preprocess_preprocess_sx_buf`**（与 `preprocess.sx` 一致）；宿主 `PIPELINE_CTX_BUF_SIZE` **512KiB**（可与 `loaded_buf`/`preprocess_buf`/`preprocess_sx_buf` 固定数组对齐），避免仅靠 C `preprocess.o` 与流水线分叉。

## 逐模块迁移计划

### Phase 1: parser 独立链入

**做法：**
1. shux-c 编译 parser.sx → parser_gen.c
2. cc 编译 parser_gen.c → parser_sx.o
3. 链接时用 parser_sx.o 替代 parser.o

**关键符号桥接：**

| C 原版符号 | .sx 版符号 | 桥接方式 |
|-----------|-----------|---------|
| `parse()` | `parser_parse_into()` | 包装函数 |
| `parser_slice_from_buf()` | 生成代码中自带 | 直接调用 |

**验收：** `./shux -E examples/hello.sx` 生成正确 C 代码

### Phase 2: typeck 独立链入

**做法：**
1. 补全 typeck.sx 中缺失的逻辑（当前大量 extern 到 C）
2. shux-c 编译 typeck.sx → typeck_gen.c
3. cc 编译 → typeck_sx.o

**关键迁移：**

| C 函数 | 行数 | 迁入 typeck.sx |
|--------|------|---------------|
| `typeck_module()` | ~100 | 入 typeck.sx |
| `typeck_one_function()` | ~200 | 入 typeck.sx |
| `typeck_check_expr_impl()` | ~800 | 入 typeck.sx |
| struct layout 计算 | ~100 | 入 typeck.sx |
| CTFE 常量折叠 | ~150 | 入 typeck.sx |

### Phase 3: codegen 独立链入（最大难点）

**当前 codegen.sx 依赖 ~20 个 extern C 函数：**

| extern | 用途 | 解决方案 |
|--------|------|---------|
| `driver_get_current_dep_prefix_for_codegen` | 文件名前缀 | 从 ctx 读取 |
| `codegen_sx_import_path_to_c_prefix` | 导入路径→C标识符 | 在 codegen.sx 内实现 |
| `codegen_sx_path_is_std_io_core` | 判断标准库 | 字符串比较 |
| `driver_force_param_*` | 参数类型覆盖 | ctx 配置 |
| `driver_should_skip_emit_func*` | 跳过生成 | ctx 配置 |
| `driver_is_submit_batch_buf_call` | 批处理判断 | 规则内化 |
| `driver_std_io_fixed_fd_emit_impl` | 固定FD生成 | 模板内化 |
| `driver_call_num_args_override` | 参数数量 | ctx 配置 |

**迁移策略：** 把这些 extern 变成 .sx 内部逻辑 + ctx 字段。

### Phase 4: 运行时最小化

迁出后 runtime.c 可大幅缩减：

| 保留 | 可删 |
|------|------|
| `main_entry`（子命令分发） | `run_compiler_c`（~180行） |
| `driver_get_argv_i`（argv 原语） | `invoke_cc`（~280行） |
| `driver_exec_cmd`（shell 执行） | `read_file` |
| 静态 arena/module 缓冲 | `resolve_import_file_path_multi` |
| | `load_one_import` + 递归 |
| | 各种 `get_*_o_path` |
