# Shux 代码全效化方案 v0.3

> **目标**：写了定义了就要有效——未使用的绑定 / 私有函数 / 不可达代码在编译期被发现或裁掉；最终产物（可执行物）无多余私有死代码。
>
> **硬约束（相对 v0.2 的转向）**：
> 1. **不增加新语法**——不引入 `#[must_use]` / `pure` / `#[side_effect]` / `#[no_dead_code]` / `#[allow_dead_code]` 等标注。学习成本为零：默认行为即全效化。
> 2. **复用既有权威**——可达性只走 `codegen_wpo_reach_*` / `codegen_compute_used`；未使用诊断走 typeck + TOOL-002 lint。禁止第二套 call graph / 平行 `STRICT_CODE_USE` 宇宙。
> 3. **不阻塞自举 E-track**；开发可回退，CI/release 逐步收紧。

---

## 0. 设计原则

| 原则 | 说明 |
|------|------|
| **默认即有效** | 用户不学新属性；编译器默认做诊断 + 裁剪。特殊逃逸只用**已有**机制（见 §2.2） |
| **不增语法** | 全效化能力落在 typeck / codegen / 构建 / gate，不落在新语言表面 |
| **诊断 ≠ 消除** | 诊断给开发者（「你写了没用」）；消除给产物（「二进制不胖」）。共享同一套 roots |
| **复用既有权威** | WPO/DCE/L6 已存在；只补缺口，不重造 |
| **证明驱动** | 每条裁减有可达性证据；export / FFI / `#[used]` 等不可猜删 |
| **分层闭合** | 局部绑定 → 模块私有符号 → 跨模块 emit → 链接 / 构建白名单 |
| **可回退** | 环境变量可关诊断或跳过 DCE，不挡调试与冷启动 |

## 1. 已落地能力（接轨现网，禁止当「待建」）

### 1.1 编译器侧

| 能力 | 锚点 | 说明 |
|------|------|------|
| 函数可达性 DCE | `codegen_compute_used` | 从 entry/main 起算 used 函数 / mono |
| 类型可达性 | `codegen_compute_used_types` | 被引用的 struct/enum 等 |
| WPO 跨模块 | `codegen_wpo_reach_compute` + `SHUX_WPO_DCE=1` | dead export 剔除 |
| Call graph dump | `SHUX_WPO_DUMP_CALLGRAPH=<path>` | JSON 可达图（验收主数据源） |
| 入口 root | `codegen_entry_root_func` | main / entry |
| 未使用绑定 lint | `SHUX_UNUSED_HINT=1` → L6 `unused binding` | let/const/import；现为 **opt-in + info** |
| 防误删（已有属性） | `#[used]` / `#[no_mangle]` / `*volatile T` | 内核/FFI/MMIO；**不是**为本方案新增 |
| 库模块约束 | 无 `main` 时 `dce_ctx_arg = NULL` | 库 `-E` 不可把整库当 dead |
| 回归 | `tests/run-wpo-dce-emit.sh`、`run-perf-wpo-dce-text.sh`、if-block reach 等 | 含「分支内 call 误删」防护 |

### 1.2 仍薄弱 / 待补齐

| 缺口 | 表现 | 处理方向（仍无新语法） |
|------|------|------------------------|
| 未使用诊断默认关 | L6 需 `SHUX_UNUSED_HINT=1`，且不阻断 | **默认开 warn**；CI 可升 error（见 §4） |
| 未用私有函数只裁不报 | DCE 不 emit，开发者常无感知 | check 路径对 **private + 不可达** 报 unused |
| 未用字段 / 私有类型 | 基本无诊断 | 后续用同一 reach 集合扩展（可选） |
| 链接层 sections | `ld -r` 等未系统开 GC | release 路径 `-ffunction-sections` + `--gc-sections` |
| seed / 双权威残留 | PREFER_X_O 后 seed 符号仍可能进最终链接 | **构建白名单 gate**（§5.2） |
| 文档/基线 | 曾误用 `nm T` vs `nm U` 做 unused | **禁止**该算法；改用 callgraph（§5.1） |

### 1.3 死代码分类（处理手段不同）

| 类 | 含义 | 默认处理 |
|----|------|----------|
| **A. 未使用局部绑定** | `let x = …` / 未用 import | typeck **警告**（默认）；`_` / `_name` 明确丢弃 |
| **B. 未引用私有函数/方法** | 模块内无调用方 | **警告/error（模式定）** + codegen **不 emit** |
| **C. 不可达 pub/export** | 可执行链接图中未达 | **不报死代码**（库 API）；产物侧可 DCE（exe 图） |
| **D. 死分支** | `if false`、return 后语句等 | 折叠 / 不生成（优化，不要求新语法） |
| **E. 静默丢弃调用结果** | `f();` 丢非 void 返回值 | **启发式**（§2.3）：无副作用推断则 warn；**不**引入 must_use/pure 关键字 |
| **F. 构建/seed 残留** | 双权威、白名单外 seed 符号 | 构建审计，与源码 lint 分开 |

**伪死代码（不可当 dead）**：`#[no_mangle]` / `#[used]` / extern 导出 / dynlib 符号 / 测试 root / 库的 pub API。

## 2. 语义与规则（零新语法）

### 2.1 Roots 与可见性（全部基于现有概念）

```
roots = {
  main / entry（codegen_entry_root_func）,
  所有 #[no_mangle] 函数,          // 已有
  所有 #[used] 函数,               // 已有：asm/中断等静态看不到的引用
  库模式下的 pub / 导出符号,       // 库产物：export 即 root
  测试入口（test harness 时）,
  构建声明的 ABI 导出表（可选配置，非语法）,
}

reachable = BFS(roots, follow=callees)   // 实现 = 现有 WPO reach
dead_private = { 非 root 且非 pub 的函数 } − reachable
```

| 场景 | 处理 |
|------|------|
| 可执行物 | 从 main 可达；不可达 **private** → 诊断 + 不 emit |
| 库模块（无 main） | **全部 export/pub 为 root**；仅裁/报模块内 private 不可达 |
| `extern "C"` 声明（无 body） | leaf，不展开 callees |
| 条件编译 | 按当前 build 配置展开后再做 reach |
| 泛型单态 | 每个实例独立节点（沿用 used_mono） |
| 函数指针 / 间接调用 | 保守：可能指向的符号视为可达（宁漏删勿误删） |

### 2.2 逃逸与「故意保留」——只用已有手段

| 需求 | 做法 | 不做什么 |
|------|------|----------|
| 仅被 asm/中断引用 | 已有 `#[used]` | 不新增 `#[allow_dead_code]` |
| 导出给 C/插件 | 已有 `#[no_mangle]` / 现有 export | 不新增模块级 `#[no_dead_code]` |
| 故意不用绑定 | `_` / `_name` 或 `let _ = expr`（现有习惯） | 不新增 allow 属性 |
| 临时调试保留私有 fn | 开发模式 warn 不 fail；或先删 | 不引入 allow 语法 |
| 关掉全效化 | 环境变量（§7.4） | 不改语言 |

### 2.3 返回值丢弃（无 pure / must_use 语法）

原则：**编译器推断，用户不标注。**

```
语句位调用 call，返回类型非 void，且结果未绑定：
  若 call 目标可证明「无副作用」（既有 side-effect 启发式，
      含：不写全局、非 extern C、非 volatile 读、不调 impure 等）：
    → 警告：返回值被丢弃（可能是死计算）
  否则（可能有 I/O / 写内存 / 未知）：
    → 不报（避免误伤 log/init）
```

与内核路径对齐：`*volatile T` 读视为有副作用，不得被块内 DCE 抹掉（已有 K2a）。

**明确不做**：`pure function` 关键字、`#[must_use]`、`#[side_effect]`。若未来 std 需要更强契约，优先 **库约定 + 文档**，而不是新表面语法。

### 2.4 跨模块图：扩展现有结构，不另起 GraphNode 方言

权威结构：`CodegenWpoReach`（`codegen.h`）+ `codegen_compute_used` 的 used 集合。

全效化只要求：

1. typeck/check **读取**同一 reach 结果，对 `dead_private` 发诊断；
2. emit 继续用 `is_func_used` 回调跳过不可达；
3. dump 继续用 `SHUX_WPO_DUMP_CALLGRAPH`（或等价 JSON），**不**再发明 `.deps` 文本方言（若构建需要依赖清单，由同一 JSON 导出）。

## 3. 编译器增强（在现有 pass 上补缺口）

### 3.1 诊断轨（typeck / `shux check`）

| 规则（建议纳入 TOOL-002） | 触发 | 默认层级 |
|---------------------------|------|----------|
| L6 unused binding（已有） | 未使用 let/const/import | **默认 warn**（从 info/opt-in 升级） |
| L7 unused private function | private + 不在 reachable | warn（CI profile 可 error） |
| L8（可选）unused private type | 类型从未出现在 used types | info → 后升 warn |
| L9（可选）discarded pure-ish call | §2.3 启发式 | warn |

实现要点：

- check 与 `-E`/`-o` **共用** `codegen_wpo_reach_compute`（或 typeck 阶段等价 reach，结果一致）；
- 禁止 check 一套算法、emit 另一套。

### 3.2 消除轨（codegen，已有 + 强化）

```
reachable = WPO / compute_used(...)
for func in module:
  if not reachable and not must_keep(func):  // must_keep = used/no_mangle/export…
    skip_code_generation(func)
```

| 优化 | 条件 | 备注 |
|------|------|------|
| 函数级 DCE | 已有 | 单文件 + WPO 跨模块 |
| 死分支 | 常量条件 / return 后 | 强化既有基础，无新语法 |
| 类型不 emit | `codegen_compute_used_types` | 继续收紧 |
| ASM 路径 | `SHUX_ASM_WPO_DCE` 等既有开关 | 与 C `-E` 路径语义对齐 |

### 3.3 依赖 / callgraph 输出

统一：

```bash
SHUX_WPO_DUMP_CALLGRAPH=/tmp/cg.json ./compiler/shux check -L . path.x
# 或编译路径同等 dump
```

用途：自举 diff、audit gate、增量构建「谁依赖谁」。不再规划平行 `.deps` 格式，除非 JSON 导出一层薄包装给 make。

## 4. 构建与模式矩阵（默认开、可关）

### 4.1 模式（用户无新语法可学）

| 模式 | 未使用绑定 | 未用 private fn | 产物 DCE | 适用 |
|------|------------|-----------------|----------|------|
| **默认 / 开发** | warn | warn | **开**（安全裁剪） | 日常 |
| **`shux check`** | warn（profile 可 error） | warn/error | 不做链接 | CI 源码卫生 |
| **release / 可执行** | 可升 error | error 或仅裁 | 严格 + 链接 GC | 发布 |
| **冷启动 seed** | 宽松 | 宽松 | 不严裁 seed 链 | 保证能编出 shux-c |
| **库编译（无 main）** | 同绑定规则 | 仅 private | export 全 root | std / 叶子库 |

原则：**默认 warn + 默认 DCE 裁剪**；「严格失败」用 CI profile / 环境变量，而不是新关键字。

### 4.2 各构建路径

| 路径 | 策略 |
|------|------|
| `Makefile` / PREFER_X_O | emit 走现有 DCE；合并时逐步 sections；**seed 符号白名单** |
| `shux-build.sh` | release：`--gc-sections`（及 function/data sections）；dev 可不加 GC |
| `build_tool` | 原生接同一套 env；不平行实现 strip 逻辑 |
| `-E + cc -c` | 需要时 `-ffunction-sections -fdata-sections`，最终 exe 链接再 GC |

### 4.3 冷启动 vs 自举后

| 阶段 | 策略 |
|------|------|
| 冷启动（seed） | 宽松：不因 unused 失败；DCE 以能链上为准 |
| 自举后（.x 为主） | 默认诊断 + DCE；CI 收紧 private unused |
| gen1 == gen2 | 两侧 **同一** DCE/诊断开关，保证 bit-identical |

## 5. 验收体系

### 5.1 静态验收（正确算法）

**禁止**（v0.2 错误示例）：

```bash
# 错误：nm 的 U 是「未定义外部引用」，不是「被用到的本文件符号」
# comm -23 defined_T used_U  → 假阳性爆炸
```

**正确**：

```bash
# tests/code-effectiveness/audit.sh（待建，接现有 dump）
# 1) 编译/check 时写出 callgraph
SHUX_WPO_DUMP_CALLGRAPH=/tmp/cg.json ./compiler/shux check -L . "$entry"

# 2) 解析 JSON：reachable=false 且非 export/no_mangle/used → unused_private
# 3) 期望：unused_private == 0（严格 profile）；或 ≤ 基线且不回升

# 4) 体积基线（辅助）
# 沿用 tests/run-perf-wpo-dce-text.sh / tests/baseline/wpo-dce-text.tsv 方向
```

可执行物补充：对比 `-E` 输出是否仍含已知 dead 符号名（已有 `run-wpo-dce-emit.sh` 模式）。

### 5.2 构建 / 双权威验收

```text
PREFER_X_O=1 最终链接符号
  − .x 可达集合
  − 显式 seed 白名单
  = 违规残留 → gate 失败
```

与源码 unused **分列指标**，避免把合法 seed 桥误判为业务死代码。

### 5.3 动态验收（辅助，不作唯一 gate）

| 工具 | 用途 | 说明 |
|------|------|------|
| callgrind / dtrace / perf | 覆盖与热点 | 冷路径合法存在，**不能**要求 100% 指令覆盖 |
| gen1 == gen2 | 自举一致 | 可附带 callgraph / .text 基线 diff |

### 5.4 Gate 设计

```bash
# tests/run-full-effectiveness-gate.sh（待建）
# 默认：诊断 warn 不阻断开发；CI profile：
#   unused_private == 0（指定入口 crate）
#   wpo dce emit 回归绿
#   seed 白名单审计绿
./tests/run-wpo-dce-emit.sh
./tests/code-effectiveness/audit.sh   # callgraph 基，非错误 nm 差集
```

指标口径：

| 指标 | 严格期望 |
|------|----------|
| `unused_private` | 0（指定模块集） |
| `unused binding`（产品路径） | 0 或仅允许 `_` 丢弃 |
| 「所有全局 T 符号 unused=0」 | **不做**（误伤 export/FFI） |

## 6. 分阶段路线图（先接现网，后收紧）

| 阶段 | 目标 | 关键交付 | 新语法？ | 与自举关系 |
|------|------|----------|----------|------------|
| **P0** | 文档与基线对齐 | 本方案 v0.3；callgraph 基线脚本；修正「待建」认知 | 无 | 并行，不挡 E-track |
| **P1** | 默认诊断 | L6 默认 warn；L7 unused private；接 TOOL-002 profile | 无 | 并行 |
| **P2** | 消除与图一致 | check/emit 共用 reach；补 method/分支等 reach 缺口 | 无 | 依赖 typeck 稳定模块 |
| **P3** | 构建产物 | function-sections + release GC；seed 白名单 gate | 无 | 接 PREFER_X_O |
| **P4** | CI 严格 | 指定 crate `unused_private==0`；体积不回升 | 无 | 自举后期 |
| **P5** | ASM/后端对齐 | ASM 路径 DCE 与 C 路径语义一致；死分支强化 | 无 | ASM 成熟后 |

**不做的 Phase**：「语言特性：must_use / pure / allow_dead_code」整章删除，避免学习负担与双权威标注。

## 7. 风险、代价与权衡

### 7.1 代价

| 维度 | 代价 | 缓解 |
|------|------|------|
| 编译时间 | 全图 reach 增量 | 开发已算 used 则复用；重分析可限 release |
| 误报 unused | 间接调用 / 宏式生成 | 保守可达；`#[used]`；开发 warn 不 fail |
| 误删 | 库当 exe 裁、if 块内 call | 库 export=root；已有 if-block 回归 |
| 二进制体积 | 预期下降 | 以 wpo-dce-text 基线衡量 |

### 7.2 特殊场景

| 场景 | 处理 |
|------|------|
| FFI / 插件 | `#[no_mangle]` / 导出表 = root |
| dynlib `dlsym` | 导出符号保留；只裁内部 private |
| seed fallback | 冷启动宽松 + 白名单；自举后收紧白名单外符号 |
| MMIO / volatile | 读保留，不 DCE（已有） |

### 7.3 与自举

| 冲突 | 决策 |
|------|------|
| typeck 未全绿 | P1 可先对已绿模块开；不全仓强制 error |
| seed 桩「未引用」 | seed 模式不 fail；白名单单列 |
| 不阻塞 E-track | 全效化并行轨；主线仍 Track L/W |

### 7.4 回退（环境变量，非语法）

```bash
# 诊断
SHUX_UNUSED_HINT=0              # 关闭未使用绑定提示（过渡期；目标默认开后改为显式关）
SHUX_LINT_CI_FAIL_ON=error      # 仅 error 失败（默认档）

# 消除
SHUX_WPO_DCE=0                  # 关闭跨模块 dead export 剔除
SHUX_ASM_WPO_DCE=0              # ASM 路径
SHUX_NO_DCE=1                   # 完全跳过 DCE（调试用，名称可与实现统一）
```

不引入 `STRICT_CODE_USE` 新宇宙；严格度 = **lint profile + 是否 fail-on-warn + 是否 release GC**。

## 9. 一句话执行口径

> **不学新语法。** 默认：未使用绑定/私有不可达 → 警告；codegen 不生成不可达私有代码；release 链接再剪。  
> **实现只扩现有 WPO/DCE/L6。** 验收看 `unused_private` 与 callgraph，不看错误的 `nm` 差集，不要求「全符号 unused=0」。
>
> 下一步可执行：P0 基线脚本（`SHUX_WPO_DUMP_CALLGRAPH`）→ P1 默认 L6/L7 → P3 seed 白名单；全程不挡自举主线。
