# 自举语言子集

> 本文档列出当前 shux 支持的语言子集，并约定：**自举期间仅小步扩展，新语法/新类型在自举验证通过后再加**，以避免两代编译器行为不一致。

---

## 一、约定

- **自举语言子集**：指当前 shux 已实现并稳定支持的语法与类型集合，见下文各节。
- **自举期间**：从开始用 .sx 重写编译器前端/中端/后端，到完成「用 .sx 编译 .sx 并验证通过」的整段时间。
- **稳定性原则**：自举过程中**少动语法与类型**；仅允许小步扩展（如修复明显遗漏、与现有文档一致的小补丁）。**新语法、新类型、新关键字**应在自举验证通过后再加入，并在文档中同步更新本子集说明。

---

## 二、关键字

当前支持的关键字（与 `compiler/include/token.h` 一致）：

| 类别     | 关键字 |
|----------|--------|
| 函数与块 | `function`、`return`、`let`、`const` |
| 条件与循环 | `if`、`else`、`while`、`loop`、`for`、`break`、`continue` |
| 控制与错误 | `panic`、`defer`、`match`、`goto` |
| 类型与接口 | `struct`、`packed`、`enum`、`trait`、`impl`、`self` |
| 模块与 FFI | `import`、`extern` |
| 模式     | `_`（match 通配） |

---

## 三、类型

### 3.1 内建类型

- **整数**：`i32`、`i64`、`u8`、`u32`、`u64`、`usize`、`isize`
- **布尔**：`bool`
- **浮点**：`f32`、`f64`（若已接入 parser/typeck/codegen）
- **向量**：`i32x4`、`i32x8`、`u32x4`、`u32x8`（SIMD，文档 §10；实现上可能以 struct 模拟）

### 3.2 复合与用户类型

- **具名类型**：用户定义的 `struct`、`enum` 名称
- **指针**：`*T`（裸指针）
- **数组**：`T[N]`（固定长度，N 为编译期常量）
- **切片**：`T[]`（语义为 `{ ptr, len }`，与文档 §6.3 一致）

---

## 四、表达式

- **字面量**：整数字面量、浮点字面量、`true`/`false`
- **变量/常量引用**：标识符
- **算术**：`+`、`-`、`*`、`/`、`%`；一元 `-`
- **位运算**：`&`、`|`、`^`、`<<`、`>>`；一元 `~`
- **比较**：`==`、`!=`、`<`、`<=`、`>`、`>=`
- **逻辑**：`&&`、`||`；一元 `!`
- **条件**：`if cond { then } else { else }`（条件表达式）；`cond ? then : else`（三元）
- **赋值**：`left = right`（如 for 的 step）
- **控制流表达式**：`break`、`continue`、`return expr`、`panic`/`panic(expr)`
- **匹配**：`match expr { lit \| _ => expr ; ... }`（整数字面量与 `_`，以及枚举变体 `Name.Variant`）
- **结构体与数组**：字段访问 `base.field`、结构体字面量 `TypeName { field: expr, ... }`、数组字面量 `[e1, e2, ...]`、下标 `base[index]`
- **调用**：函数调用 `callee(args)`；方法调用 `base.method(args)`（trait/impl 解析后）

---

## 五、语句与块

- **块内**：`const name [: Type] = expr`；`let name [: Type] = expr`
- **循环**：`while cond { block }`；`loop { block }`（等价 `while true`）；`for (init; cond; step) { block }`（C 风格，init/cond/step 均可省略）
- **延迟**：`defer { block }`（块退出时逆序执行）
- **标签与跳转**：`label:`、`goto label`、`return expr`（作为语句）
- **块**：由 const/let、循环、defer、标签/return 与**最终表达式**组成；最终表达式为块的值

---

## 六、顶层定义

- **导入**：`import path;`（如 `import("core.types")`；解析先试 `xxx.sx` 再试 `xxx/mod.sx`）
- **结构体**：`struct Name [<T,…>] { field: Type; … }`；可选 `packed`
- **枚举**：`enum Name { Variant1, Variant2, … }`（无负载枚举，§7）
- **Trait**：`trait Name { method(...) -> Ret; … }`（阶段 7.2）
- **实现**：`impl Trait for Type { function method(...) { … } … }`
- **函数**：`function name [<T,…>] (params) -> Ret { block }`；多函数，入口为 `main`
- **外部声明**：`extern "C" function name(params) -> Ret;`（无体，由链接器解析 C 符号）

---

## 七、多文件与泛型

- **多文件**：通过 `import path` 引用其他 .sx 模块；模块由 parser 解析后合并/可见（见 compiler 的 import 解析与 typeck 的跨模块解析）。
- **泛型**：当前 AST 支持泛型参数（`generic_param_names`、`num_generic_params`）与单态化实例（`mono_instances`）；具体支持程度以 typeck/codegen 实现为准（阶段 7.1/7.2）。

---

## 八、与「子集稳定」的对应

- 上文各节以**当前已实现并用于测试/示例**的语法与类型为准；若某特性在 parser 中有占位但 typeck/codegen 未完全支持，应在自举子集中视为「部分支持」或按实际可编译范围描述。
- **自举期间**：仅允许在上述子集范围内的**小步扩展**（如 bug 修复、与既有文档一致的小补丁）。**新语法/新类型/新关键字**须在自举验证通过后，再纳入本子集并更新本文档。
- 若与 `compiler/docs/`、`analysis/` 中其他文档有出入，以本文档的「当前子集」描述为准；冲突处可在自举后统一修订。
