## 一、读本文前先对齐：四个「自举键」

项目存在**四个不同严格度**的「可以自举了」判据，勿混用：

| 按键 | 名称 | 判据（摘要） | 现状（2026-06-28） |
|------|------|--------------|-------------------|
| **键 A** | 语义自举 | Stage0→Stage1→Stage2；两代行为一致 | ✅ `make bootstrap-verify` |
| **键 B** | 完全自举 v1 | **D + E + F**：黄金哈希 + 运行时无 C 前端 + std 无 C | ✅ Linux x86_64 |
| **键 C** | 放心写 .sx 编译器 | asm `-o` 直链无 gcc 回退；fresh seed 可重建；编译器硬依赖 std 无 C 桩 | 🟡 主战场 |
| **键 D** | 终局自举 | **G 清场**：物理零 C/H、`-nostdlib` bootstrap、`build.sx` 替代 Makefile | 🟡 ~30% |

**本文「必须项」分三档**：

- **M（Must）**：不完成则**不能**按对应按键推进（键 B/C/D 各自有子集）。
- **S（Should）**：不阻塞编译，但自举期写数万行 .sx 会**极慢、极难排错**。
- **D（Defer）**：明确**自举后再做**；接口已预留或不影响闭环。

### 3.3 L9：尾部填充（Trailing Padding）与 Arena 对齐陷阱

**问题**：结构体默认按 ABI 对齐；若 Bump 分配器只做 `ptr += size` 而不 `align_up`，下一对象可能落在**非 8 字节边界**。对含 `usize`/指针字段的普通 struct 写入时：x86_64 性能劣化；**ARM64 可直接 SIGBUS**。

**与 L2 的关系**：

- **`packed` 显式类型**：允许非对齐布局，**仅允许在 `unsafe` 块内**解引用/写入；Arena **不得**把 packed 对象当作普通对齐节点复用。
- **非 packed AST/IR 节点**：编译器内部凡经 Arena/page bump 吐出的 `*T`，**必须**在分配时 `align_up(off, max(8, align_of(T)))`；宁可尾部 padding 浪费若干字节，也不产出奇数/非对齐指针进入安全子集。

**验收**：

1. `std/heap/page_mmap.sx`：`page_mmap_heap_alloc` 调用 `mem.align_up`（已满足，须保持）。
2. 编译器 AST/token Arena（`grow_vec` / `with_arena` 路径）接入同一对齐策略（**键 C 前须审计**）。
3. 负例：`tests/memory-contract/arena_align.sx` + `run-memory-contract-arena-align-gate.sh`（优先 `shux-c` typeck+`-o`；asm 直链归 C6）。

### 4.4 S9：标准库 API 命名规范与改名同步

> **单一真相源**：[`std/标准库api命名规范.md`](../std/标准库api命名规范.md)（含 §一～§三 规则与清单、§六 Tier 分级、§七 高频模块三轮精简对照）。

#### 4.4.1 自举期必须遵守（原则层 — 现在就要做）

| 规则 | 要求 |
|------|------|
| 绑定 import | 用户代码 `const io = import("std.io"); io.read(...)`，**禁止**在函数名中重复模块前缀（`io_read`） |
| 去标量类型 token | 函数名不得含 `_i32` `_u8` 等；靠重载 / 容器单态化区分 |
| 去参数个数后缀 | 禁止 `format_2` / `select_recv_2`；合并为同名重载 |
| 禁止新增债务 | **自举期新增公开 API 须先对照规范**；不得新增违反上述规则的 export |
| Tier-X 不 export | `*_smoke`、`lock_diag_*`、C 层 bench 桩等**不得**写入公开 `mod.sx`（见规范 §六） |

**验收（原则）**：code review / gate 人工审计；新增 std 模块 PR 须引用规范 §一～§二。

#### 4.4.2 自举后再做（落地层 — 不阻塞键 C）

| 项 | 说明 | 档位 |
|----|------|------|
| 简化名批量 export | 规范 §三 清单「简化名称」列写入各 `std/*/mod.sx` | D |
| 旧名兼容 alias | 规范 §五：旧 `_*_i32` / 长名保留一版 `@deprecated` alias 后删除 | D |
| 大模块拆子 path | `std.http.client`、`std.sync.mutex` 等（规范 §二 子路径绑定表） | D |
| 编译器重载完备 | 支撑 `read` / `print` / `heap.alloc` 等同名合并 | D |

#### 4.4.3 修改 API 名称后必须同步（**任何改名均适用**）

重命名或新增简化名 alias 时，**同一 PR / 同一批次**须同步以下引用面，避免 gate 绿但文档/链接/符号漂移：

| # | 同步面 | 路径 / 动作 |
|---|--------|-------------|
| 1 | **规范对照表** | [`std/标准库api命名规范.md`](../std/标准库api命名规范.md) §三 对应行「当前名称 / 简化名称 / 绑定调用」 |
| 2 | **std 源码** | `std/*/mod.sx` export；实现文件 `function` 名；模块内 `import` 调用 |
| 3 | **编译器 / runtime** | `compiler/src/`、`compiler/pipeline_glue.c`、`import_alias.c` 过渡桩 |
| 4 | **测试与 gate** | `tests/**/*.sx`、`tests/**/*.sh`、baseline `tests/baseline/*.tsv` |
| 5 | **C 链接 / 符号** | `import_alias.c`、`tests/bench/*.c` 中 `-Dold=new` 映射；核对 **V7 符号可见性**（§7.2） |
| 6 | **analysis 文档** | 引用旧 API 名的 analysis（如 `高并发IO与多平台.md`、`std-change-rfc-*.md`） |
| 7 | **用户文档** | `docs/`、`README.md`、模块文件头「典型用法」注释 |
| 8 | **自举清单与进度** | 本文 §4.1 S* 验收列；[`自举进度.md`](自举进度.md) §5 变更记录 |

**推荐流程**：

```text
1. 先改 std/标准库api命名规范.md（简化名 + 绑定调用）
2. 改 std 实现 + mod.sx export（新名为主，旧名 alias 一版）
3. ripgrep 旧名全仓 → 逐项改 tests / compiler / docs / analysis
4. 跑相关 gate（bootstrap-min / symbol-visibility / 模块专项 gate）
5. 更新自举进度.md 变更记录
```

**禁止**：只改 `.sx` 实现不改规范表；只改规范不跑 gate；公开 export 与 `import_alias.c` 短名不一致导致 **V7 跨代符号劫持**。

### 5.2 诊断格式（C2 目标样例）

类型不匹配场景必须包含四要素：

```text
Error: Type mismatch at main.sx:45:12
   | let x: i32 = get_data();
   |              ^^^^^^^^^^ Expected i32, found Option<i32>
```

**自举常用路径**尚须扩展到：

- import 失败（模块未找到、循环依赖）
- 泛型单态化失败 / trait 未实现
- 参数个数/类型不匹配
- 跨模块 `return` 类型解析

### 5.3 C5 深化：Regalloc Spilling × 16B Result ABI 复合冲突

**背景**：L3 锁定 SysV amd64 下 16B `Result<T,E>` 经 **rax + rdx**（或等价 homing）返回。C5 使用 **Linear Scan**；`typeck.sx` 等巨型函数必然触发 **Spilling**（变量溢出到栈槽）。

**静默损坏模式**：Spill 目标是临时 **16B Result** 或 struct16 复合值时，若 codegen 将其当作 **8B 整型** 写入栈槽，或 spill 槽未 **16B 对齐**、恢复时只 reload rax 而丢失 rdx，则运行期 **高 8 字节静默丢失** — gate 仍可能绿，但 Stage2 哈希或语义 subtly 漂移。

**现有实现线索**（须纳入回归，不可假设已完备）：

- `pipeline_glue.c`：`glue_spill_struct16_call_arg_to_lea_elf_c`、栈帧 spill 表 `[sp,#slot*16]`、`compute_frame_size +512` scratch（`run-result.sh` 修复链）。
- `backend_call_dispatch.c`：struct16 CALL 实参 spill 路径。

**必须验收（键 C 前）**：

1. **高密度 spill 烟测**：构造「局部变量数 > 可用 GPR、嵌套 CALL、中间 Result/struct16 临时量」的合成函数（可放 `tests/codegen/regalloc_result_spill.sx`），`-o` asm 直链，运行期校验 Result 高低字均正确。
2. **Spill 槽对齐**：16B 对象 spill 槽偏移为 **16 的倍数**（或与 `alignof` 一致）；reload 须 **成对** 恢复 rax/rdx（或 mem16 等价）。
3. **与 §9.1 联动**：`run-result.sh` 绿 **不等于** 大函数 spill 绿；gold gate 须包含 regalloc+Result 复合用例。

**gate**：`tests/run-comp-regalloc-result-spill-gate.sh`（已接入 bootstrap-min / comp-regalloc-gate；`-o exit=225` ✅ asm stage1）。

### 8.1 G-03-rt：伪 Freestanding 下的 compiler-rt / 隐式 libc 符号

**问题**：即使前端 100% `.sx`、链接行去掉 `-lc`，codegen 仍可能为下列运算**隐式发射**对外符号调用：

| 运算 | 典型隐式符号（平台相关） | 触发场景 |
|------|--------------------------|----------|
| **i64 / i64、i64 % i64** | `__divdi3`、`__moddi3`、`__udivdi3`… | typeck 常量折叠失败时的运行期除法 |
| **大 struct 赋值/拷贝** | `memcpy`、`memmove` | struct 大于寄存器返回阈值 |
| **i128 / 软浮点** | `__multi3`、`__addtf3`… | 若语言子集引入宽整数/软浮点 |

**`-nostdlib` 链接**会立即 **undefined reference**；更糟的是链入错误版本的 libc stub → **静默行为不一致**。

**硬性约束（键 D 必须）**：

1. **core 自给自足**：`core/mem` 提供 **`memcpy`/`memset`/`memmove` 语义**（当前 `.sx` 循环实现 ✅）；`-nostdlib` 链接须 **显式链入** `core.mem` 对象，且 codegen **不得**再 emit `call memcpy@plt` 除非经 `import core.mem` 解析到同一实现。
2. **i64 除法**：要么 codegen **内联展开**为 x86 `div/idiv` 序列，要么在 `core/` 或 `runtime/*.s` 提供 **`shux_i64_divmod` 等最小 compiler-rt 子集**并写入链接白名单。
3. **大 struct 拷贝**：优先 **逐字段 / 内联 rep mov**；禁止依赖 libgcc/compiler-rt 默认符号。
4. **链接审计**：`SHUX_BOOTSTRAP_NOSTDLIB=1` 产物执行 `nm -u \| grep -E 'memcpy|memset|__div|__multi'` 应为**空**或**仅指向仓库内 `.o`**。

**验收**：

```bash
SHUX_NOLIBC_N07_V5_FAIL=1 ./tests/run-nolibc-n07-v5-gate.sh
# 附加（建议写入 gate）：
# ldd bootstrap_shuxc 2>&1 | grep -q 'not a dynamic executable' || ldd ... | grep -vc libc
# nm -u ./compiler/shux_asm | grep -E '^(.*\s)?(memcpy|__divdi3|__multi3)$' && exit 1
```

**与 G-03 任务表对齐**：[`phase-g-full-selfhost-tasks-v1.md`](phase-g-full-selfhost-tasks-v1.md) G-03-04 libm 替代同理扩展为 **compiler-rt 全清单**。
