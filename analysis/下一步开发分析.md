## 三、阶段 10.1：修 -sx -E 多文件（最高优先级）

### 3.1 问题描述

- **现象**：`shuc_su -sx -E tests/multi-file/main.sx` 在部分环境（含 CI）下输出不完整：
  - 期望：生成的 C 包含 `bar()` 定义及 main 对 bar 的调用，编译运行得 42。
  - 实际：有时仅得到片段（如 "42   ()"），或缺少 bar，导致 run-su-multi-file 判定失败并在 CI 中 SKIP。
- **影响**：-sx -E 多文件是「纯 .sx 流水线编译多文件」的关键路径，自举与完全 .sx 驱动依赖此路径。

### 3.2 根因方向（待定位）

- pipeline 中「先编 dep（foo），再编 main」的**顺序**与**写缓冲/合并**方式；
- codegen 或 driver 在**多模块输出合并**时丢失或错序（dep 的 C 未正确写入 stdout 或与 main 的 C 合并）；
- 与单文件 -sx -E 的差异：多文件时存在 dep_modules/dep_arenas 与 main 的 typeck/codegen 顺序、out_buf 复用与 flush 时机。

### 3.3 任务拆解

| 序号 | 任务 | 说明 | 完成标志 |
|------|------|------|----------|
| 10.1.1 | ✅ 复现与定位 | 在本地/CI 复现「缺 bar 或 42 ()」；确定是 codegen 未生成 bar，还是写 stdout 时丢失/覆盖 | ✅ 根因：main.sx 的 driver_run_sx_emit_sx 仅单文件无 import；shuc_su 走 C 的 run_compiler_sx_path 已有完整 deps+main |
| 10.1.2 | ✅ 修 pipeline 多文件输出顺序 | 保证 dep 的 C 先完整写出，再写 main 的 C；必要时显式 flush 或统一缓冲策略 | ✅ -sx -E 时 main.sx 的 entry 改为调 C 的 driver_run_sx_emit_c（set_path/set_lib/set_n + driver_run_sx_emit_c），多文件走 C 路径 |
| 10.1.3 | ✅ 修 codegen/合并逻辑（若需） | 若为 codegen 只生成了 main 未生成 dep，则修 typeck/codegen 对 dep 的调用与输出 | ✅ 不需要；C 路径已正确写 dep 再 main |
| 10.1.4 | ✅ 取消 CI 对 run-su-multi-file 的 SKIP | 在 run-all.sh 或 run-su-multi-file.sh 中去掉「CI 下失败则 SKIP」的逻辑，改为失败即失败 | ✅ 已去掉 run-su-multi-file.sh 中两处 CI SKIP |

### 3.4 验收标准（阶段 10.1 完成）

- 在仓库根执行：`./tests/run-su-multi-file.sh` **无 SKIP** 且退出码 0。
- 在 CI 中 run-all 包含 run-su-multi-file，且该脚本通过（不依赖 SKIP 保绿）。

### 4.3 验收标准（阶段 10.2 完成）

- CI 中 run-sx-pipeline 不再因「避免挂」而无条件 SKIP；或存在必跑的 -sx -E 单文件 smoke 且通过。

### 4.4 平台差异（待定位）与诊断建议

- **现象**：Linux / Windows / Debian 上 `shuc_su -sx -E` 的 stdout 仅得到 `"0"` 或 `"   0"`，macOS / Alpine 正常。
- **已做**：`driver_run_sx_emit_c` 中每次 `fwrite` 后已加 `fflush(stdout)`，避免重定向下只看到最后一段。
- **后续定位建议**：
  1. ✅ 已在 `driver_run_sx_emit_c` 的 main 段 `fwrite` 前增加：若 `out_buf.len < 20` 则向 stderr 打 `out_buf.len` 及前最多 16 字节（hex），便于 CI 或本地复现时确认是 len 错误还是内容只写了数字。
  2. ✅ 已在 runtime.c 中为 `struct codegen_CodegenOutBuf` 增加 C11 `_Static_assert(offsetof(..., len) == SU_CODEGEN_OUTBUF_CAP)`，保证各平台 `data[262144]` 与 `len` 布局一致，排除 ABI/对齐问题。
  3. ✅ run-sx-pipeline.sh、run-su-multi-file.sh 已将 -sx -E 的 stderr 写入临时文件，失败时打印「--- stderr ---」及内容，CI 日志中可见诊断。
