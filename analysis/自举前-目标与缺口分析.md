## 二、Target B 阻塞项（按优先级）

### 2.1 大模块 asm 单编崩溃（P0）

| 模块 | 现象 | 已做缓解 |
|------|------|----------|
| `typeck.x` | segfault(139) / Bus error | `ulimit -s`、pthread 大栈、`check_expr_*_impl` 拆分、`PipelineDepCtx.typeck_check_depth` |
| `codegen.x` | segfault | `codegen_prefix_mirror` 改走 glue，避免 struct 内 `u8[64]` 下标 typeck 失败 |
| `backend.x` / `parser.x` / `pipeline.x` | segfault | 仍待 asm codegen 栈/表达式种类覆盖 |

**验收**：`SHUX=./shux ./scripts/check_asm_o_quality.sh build_asm` → `.asm_text_quality=1`（24/24）。

### 2.2 跨模块 typeck（P0，部分已解）

- **现象**：`import asm.types` 后 `return append_asm_line(...)` → `expected i32, found ?`。
- **根因**：库模块 dep 预跑偶发未写 `func.return_type_ref`；`get_dep_return_type` 曾误用 `ctx.ndep==0` 早退。
- **修复**：`typeck.x` 中 dep 槽兜底、`append_asm_line` 等 i32 助手按名回填、`RETURN`+`EXPR_CALL` 写回 `resolved_type_ref`。
- **剩余**：`lsp.x`、`x86_64_enc.x` 等仍有个别 return 不匹配；`arm64` asm 不支持 `ExprKind=46`（INDEX）。

### 2.3 符号链接（P1）

- `backend.o` 与 `arch_*.o` 间 `arch_arm64_*` vs `_arch_arm64_*` 命名需与 [SELFHOST.md](../compiler/docs/SELFHOST.md) §4.1 对齐。
- **验收**：`SHUX_ASM_EXPERIMENTAL_SKIP_GEN=1 ./scripts/build_shux_asm.sh` 试链无 undefined（Darwin 允许回退 gen_driver）。
