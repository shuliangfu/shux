# 内存契约（自举前必须清单第一步）

> 与 C 互操作与 ABI 的基础。约定：struct 默认对齐与显式 Packed、Slice 固定布局 (ptr, len)。自举前必须锁定，供 FFI、std.io.driver、std.net 及后续 ABI/布局文档引用。

---

## 一、目标与范围

| 目标 | 说明 |
|------|------|
| **与 C 一致** | 用于 FFI 的 struct/slice 在内存中的布局与 C 或目标平台 ABI 可对应，便于跨边界传递与调试。 |
| **显式优先** | 默认行为明确；需要「无填充」布局时使用显式 **packed**，避免隐式歧义。 |
| **Slice 稳定** | 切片类型在 ABI 上固定为「指针 + 长度」两字段，顺序与大小稳定。 |

---

## 二、Struct 布局

### 2.1 默认对齐（自然对齐）

- **规则**：与 C 一致的自然对齐（每个字段按自身对齐要求放置，结构体整体按最大字段对齐）。
- **实现**：由 typeck 的 `compute_struct_layouts` 计算各字段偏移、总大小、结构体对齐；codegen 生成 C struct 时字段顺序与 .x 一致，由 C 编译器按同一规则布局。
- **隐式 padding**：若存在字段前或结构体末尾的隐式填充，且未使用 **allow(padding)**，则编译报错；用户须显式添加 padding 字段或写 `allow(padding) struct Name { ... }` 以允许隐式填充。

### 2.2 Packed 显式

- **语义**：与 C 的 `__attribute__((packed))` 等价——不插入任何字段间或末尾填充，字段按声明顺序紧密排列，结构体对齐为 1。
- **语法**：在 struct 名前加 **packed** 关键字，例如：`struct Foo packed { x: u8; y: i32; }`。
- **用途**：与 C 的 packed 结构、网络/协议二进制布局、或需精确控制字节序与对齐的 FFI 场景。
- **约束**：packed 结构体可能产生未对齐访问；仅在与 C 互操作或协议约定明确时使用。

---

## 三、Slice 固定 (ptr, len)

- **ABI**：切片类型 `T[]` 在内存中固定为两个字段、顺序不变：
  1. **第一字段**：指向元素类型的指针（`*T`），与 C 的 `T*` 或 `void*` 对应。
  2. **第二字段**：长度（`usize`），与 C 的 `size_t` 对应。
- **大小与对齐**：在 64 位平台上为 16 字节（8 + 8），对齐 8；32 位平台上为 8 字节（4 + 4），对齐 4。
- **实现**：codegen 生成 `struct shux_slice_<elem> { elem* data; size_t len; }`（或等价命名）；对外约定「第一成员为指针、第二成员为长度」，与本文档 (ptr, len) 一致；与《高并发 I/O 与多平台》中 Buffer、ReadOnlySlice/WriteOnlySlice 的零拷贝传递兼容。
