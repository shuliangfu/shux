# 向量类型与车道数分析

> 内建向量类型支持 **4、8、16 车道**（i32x4、i32x8、i32x16、u32x4、u32x8、u32x16）。**32 车道留作占位**（不实现，仅文档预留）；**64 车道不提供**。本文档说明与常见 CPU SIMD 宽度的对应关系及设计结论。

---

## 一、当前实现概览

| 类型     | 车道数 | 说明 |
|----------|--------|------|
| i32x4 / u32x4 | 4 | 4 个 i32/u32，共 128 位 |
| i32x8 / u32x8 | 8 | 8 个 i32/u32，共 256 位 |
| i32x16 / u32x16 | 16 | 16 个 i32/u32，共 512 位（对应 AVX-512） |

- **实现方式**：codegen 生成 C 侧 `typedef struct { elem v[lanes]; } i32x4_t` 等，二元运算为**逐车道标量展开**（循环生成 `.v[0] op .v[1] ...`），尚未映射到真实 SIMD intrinsic（如 SSE/AVX/NEON）。
- **涉及位置**：lexer（TOKEN_I32X4 等）、parser（parse_type_primary）、typeck（向量字面量车道数校验）、codegen（vector_c_type_name、ensure_vector_typedef、codegen_vector_binop）。

---

## 二、与常见 CPU SIMD 宽度的对应

| 车道数 | 位宽 (i32) | 常见硬件 |
|--------|------------|----------|
| 4      | 128-bit    | **SSE/SSE2/SSE4**（x86）、**NEON**（ARM），最普及 |
| 8      | 256-bit    | **AVX/AVX2**（x86）、部分 **SVE**（ARM） |
| 16     | 512-bit    | **AVX-512**（x86 服务器/部分桌面）、**SVE 512-bit**（ARM） |
| 32     | 1024-bit   | 无主流 CPU 固定 1024-bit；**留作占位**，暂不实现 |

结论：

- **4、8、16**：已实现，覆盖 128/256/512-bit，与 SSE、AVX2、AVX-512 及 NEON/SVE 对齐。
- **32**：**仅占位**。无主流 CPU 一维 32×i32 固定宽度；若将来有 1024-bit 向量 ISA 或类型对称需求，再实现 i32x32/u32x32。
- **64**：不提供。64×i32 = 2048-bit 无对应主流 CPU 向量；需求可用 `[64]i32` 或 4×i32x16 表达。

---

## 三、小结与建议

| 车道数 | 状态 | 说明 |
|--------|------|------|
| 4、8、16 | 已实现 | 覆盖 128/256/512-bit，对应 SSE/NEON、AVX2、AVX-512 |
| 32       | 留作占位 | 不实现；文档预留，后续按需再加 |
| 64       | 不提供 | 无主流硬件对应；不加入语言 |

---

## 四、与自举前性能基建的对应

- 《自举前性能与语义基建》§5.2 要求：自举前具备**内建函数 → 后端 intrinsic** 的映射框架，便于标准库（如 core.simd）生成目标平台最优 SIMD。
- 向量类型 4/8/16 与上述框架**正交**：类型决定「标量展开或将来 intrinsic 的宽度」；具体映射到 SSE/AVX/AVX-512/NEON 由内建或标准库在自举后实现。16 车道有利于后续 **i32x16 → __m512i** 等映射不返工。

---

*本文档为向量车道数设计结论：16 已完善，32 留作占位，64 不提供。*
