# Shux 安全特性演进矩阵（性能与安全双赢版）

> **语言**：Shux（`.sx`）  
> **定位**：参考 C 的执行模型，默认安全子集 + 显式 `unsafe` 边界；**无 GC**，安全语义在 **编译期（typeck）** 证明，运行期零额外税。  
> **关联**：[analysis/安全与性能.md](./analysis/安全与性能.md)、[analysis/语法与类型设计-高性能与内存安全.md](./analysis/语法与类型设计-高性能与内存安全.md)、[analysis/编译时自动内存管理和自动向量化.md](./analysis/编译时自动内存管理和自动向量化.md)

---

## 0. 一句话

**安全是骨架（typeck + 门禁），性能是肌肉（codegen 在证明成立后压榨）。** 可证明的安全不是性能的敌人，而是 BCE、noalias、autovec、arena 等极致优化的**许可证**。

目标不是数学意义上的「绝对安全」（通用 OS + FFI + 人写代码做不到），而是：

> **安全子集内**：无泄漏、不因内存错误崩溃、访问有定义；  
> **子集外**：行为文档化、边界可 grep、变更可审计。

---

## 1. 优先级总览

| 级别 | 名称 | 核心特征 | 合拢窗口 | 典型收益 |
|------|------|----------|----------|----------|
| **S0** | 已焊进门禁 | 拒错规则 + gate 硬失败，自举前不可放水 | **现在** | 防 Stage1/Stage2 行为漂移、SHA256 对不齐 |
| **S** | 必须（零成本） | 编译期消灭 ~70% 内存/逻辑隐患，运行期 0 开销 | Stage 2 合拢 → **阶段 G 结束前** | 安全与 SIMD/BCE/arena 同链 |
| **A** | 强烈建议 | 与外界（C 库、syscall、不可信输入）的物理隔离墙 | **脱离 libc（-nostdlib）后** | 可 grep 审计、供应链可控 |
| **B** | 未来进化 | 分布式、AI 插件、用户脚本等跨代能力 | **v1.0.0-selfhost 之后** | 编译期沙箱、能力凭证 |

```text
威胁输入 ──→ S0 拒错门禁 ──→ S 级编译期证明 ──→ codegen 优化（BCE/noalias/autovec/arena）
                                    │
                                    └── 证不出 → 保守回退（保留检查），禁止 silent wrong code
不可信边界 ──→ A 级 unsafe/FFI/纵深防御 ──→ B 级 Capability / Wasm 沙箱
```

## 2. 威胁模型（写坏了会怎样）

| 层级 | 典型缺陷 | 后果 | Shux 对策 |
|------|----------|------|-----------|
| 应用 `.sx` | 越界、UAF、整数溢出、逻辑漏洞 | 崩溃 → RCE / 信息泄露 | 待在安全子集；不可信输入走有界解析 |
| 编译器自身 | 超长 token、布局算术溢出、CTFE 截断 | 供应链攻击、生成带毒二进制 | Arena 有界解析、溢出检查、gate 回归 |
| 系统边界 | 错误 syscall、无界 C API | 内核交互风险、经典 C 漏洞回归 | `std/sys/` 收拢 + FFI 安全包装 |

**铁律**：在安全子集外或 `unsafe {}` 内写错，效果等同 C——进程可被攻破；高权限运行则宿主系统亦可能被利用。

## 3. 安全 ↔ 性能因果链

| 安全机制 | 编译器得到的信息 | 性能收益 | 证不出时 |
|----------|------------------|----------|----------|
| **MEM-A3** scope borrow / 域 | 指针不悬垂、slice 不逃逸 | 更可靠的 alias 分析 | 硬拒错（不进 codegen） |
| **MEM-A1** noalias | 两指针不重叠 | autovec、指令重排 | 标量回退 |
| **MEM-A2** BCE | 下标 ∈ `[0, len)` | 删除循环内边界分支 | 保留运行时检查 |
| **MEM-C1** `with_arena` | 分配域 = 块生命周期 | 热路径 `malloc=0` | 回退 heap + 显式 free |
| **Linear(T)** move | 无 double-free / 无别名 | 更激进销毁与布局 | typeck 拒二次使用 |
| **Option / Result** | 可空与错误显式 | 寄存器 ABI、DCE 可靠 | 禁止 silent ignore |
| **region L {}** | slice 借用域 | 零指令嵌套块 | escape → 编译错误 |

**冲突只出现在两种情形**：typeck **证不出**（默认保守回退），或程序员 **显式 `unsafe` 断言**（opt-in，可审计）。

## 5. S 级：必须（编译期零成本消灭 ~70% 隐患）

**合拢窗口**：Stage 2 自举彻底合拢后、**阶段 G 结束前**，在编译器前端（Typeck）与 codegen 侧固化。  
**共同特征**：把安全压力压给编译器；运行期开销归零（0-cost）。

### 5.1 切片 `T[]` + region 借用域

| | |
|---|---|
| **消灭的威胁** | 缓冲区越界、slice UAF、悬垂引用 |
| **落地规范** | 切片语义为 `{ ptr, len }`；`region L { }` 内 stamp 为 `T[]<L>`，块外赋值/return/call 若域不匹配或 escape → **typeck 硬拒** |
| **性能收益** | region 块 codegen 等价嵌套 `{ }`，**零指令**；为 BCE 与 read_ptr 零拷贝提供域证明 |
| **实现** | ✅ v1 定版 — [type-region-v1-rfc.md](./analysis/type-region-v1-rfc.md) |
| **门禁** | `run-typeck-region.sh`、`run-zc3-gate.sh` |

### 5.2 scope borrow + noalias（MEM-A1 / MEM-A3）

| | |
|---|---|
| **消灭的威胁** | 悬垂指针、非法别名、数据竞争前提 |
| **落地规范** | 借用不得长于被借对象生命周期；alias 分析 **证不出就回退**，禁止为性能猜 noalias |
| **性能收益** | noalias 成立 → loop autovec（VEC-V1）、load/store 重排 |
| **实现** | ✅ 拒错 + gate；noalias → autovec **部分闭合** |
| **关联** | [自举前性能与语义基建.md](./analysis/自举前性能与语义基建.md) |

### 5.3 Option[T] / Result[T, E]

| | |
|---|---|
| **消灭的威胁** | 空指针误用、错误被 silent ignore、异常控制流不透明 |
| **落地规范** | **不学 Go 的 `if err != nil` 运行时逃逸链**（堆分配 + 分支预测失败）。做成强类型枚举 + `match` / `if let`；codegen **寄存器 ABI 返回**（非内存临时结构体拷贝）。错误分层见 EXC-001：`Result_i32` / Layer C 侧车 |
| **性能收益** | 汇编层退化为寄存器 + 条件跳转（JZ/JNZ）；配合 DCE 消除 dead error 路径 |
| **实现** | ✅ 单态化族（`Option_i32` 等）；泛型 `Option<T>` / `Result<T,E>` 推进中 — LANG-009/010 |
| **关联** | [core-option-result-unify-v1.md](./analysis/core-option-result-unify-v1.md)、[exc-result-error-v1-rfc.md](./analysis/exc-result-error-v1-rfc.md) |

### 5.4 BCE 驱动的动态边界检查（MEM-A2）

| | |
|---|---|
| **消灭的威胁** | 盲目每下标检查（慢） vs 盲目删检查（漏洞）的两难 |
| **落地规范** | Typeck + 值域分析：循环边界、常量下标、len 关系可证 → **物理删除**检查指令；外部动态输入流上保留保底检查 + `panic`（缩小为 DoS 而非内存破坏） |
| **性能收益** | 循环内零分支 → SIMD autovec 前提；与 [编译时自动内存管理和自动向量化.md](./analysis/编译时自动内存管理和自动向量化.md) 同链 |
| **实现** | ⚪ BCE v1 部分闭合；证不出默认保留检查 |
| **铁律** | **禁止** silent wrong code |

### 5.5 隐式单态化 `with_arena` + defer / owned（MEM-C1 / MEM-B0）

| | |
|---|---|
| **消灭的威胁** | 泄漏、UAF、double-free、热路径隐藏 malloc |
| **落地规范** | **不引入 Rust 式生命周期运行时符号**。Typeck 根据块生命周期拓扑：① `with_arena(cap) { }` 绑定 Allocator；② `scope_allocator()` 块内展开为栈槽；③ `defer` / `owned T` 块尾逆序释放。标准库热路径（`std/json`、parser）隐式单态化注入 Arena |
| **性能收益** | 块内 bump + 零 per-op free；bench 已验证 `with_arena_vec_push` 零 malloc |
| **实现** | ✅ P1 v1 闭合 — `run-with-arena-vec-gate.sh` |
| **关联** | [std-heap-allocator-v1.md](./analysis/std-heap-allocator-v1.md) |

### 5.6 Linear(T) 所有权转移

| | |
|---|---|
| **消灭的威胁** | double-free、use-after-move |
| **落地规范** | 每个 `Linear(T)` 绑定最多消费一次；禁止取址；与 region **分工**：linear 管所有权转移，region 管借用域 |
| **性能收益** | 与内层 `T` 同 ABI、同布局；零 RC / drop 表 |
| **实现** | ✅ v1 定版 — [type-linear-v1-rfc.md](./analysis/type-linear-v1-rfc.md) |
| **限制** | v1 不支持 slice/array 字段级 linear；线程共享证明见 A 级 |

### 5.7 编译期整数溢出守卫

| | |
|---|---|
| **消灭的威胁** | 长度/偏移算术溢出 → 堆溢出、错误代码生成 |
| **落地规范** | **Debug/Test**：默认编译溢出 panic（拦截逻辑漏洞）；**Release**：安全子集内有符号溢出仍为 UB 除非显式 wrapping API；提供 `+%=` 等显式回绕操作符；编译器/layout/CTFE 路径强制溢出检查 |
| **性能收益** | Release 热路径可标注或 profile 引导平铺为硬件回绕；冷路径保留检查 |
| **实现** | ✅ 编译器 CTFE/i64 gate；用户态 wrapping API 持续推进 |
| **关联** | [UB与未定义行为.md](./analysis/UB与未定义行为.md) |

### 5.8 显式初始化 + UB 收窄

| | |
|---|---|
| **消灭的威胁** | 未初始化读、除零、有符号溢出 UB |
| **落地规范** | 安全子集内「能定义的都定义」；不能接受的行为 typeck 拒错或运行时有定义地 `panic` |
| **性能收益** | 语义明确 → 常量折叠、DCE 更可靠 |
| **实现** | ✅ 部分闭合 — `run-ub.sh` |

## 6. A 级：强烈建议（隔离外部剧毒生态）

**职责**：Shux 彻底脱离 libc（`-nostdlib`）后，划定与外界污染源（C 动态库、syscall、网络包）的隔离墙。

### 6.1 `unsafe { }` 代码块

| | |
|---|---|
| **定位** | 仓库物理零 C 后的**唯一后门** |
| **落地规范** | 仅 `unsafe {}` 内允许：裸指针算术、未证明边界的解引用、特定 FFI、直接 Asm Syscall。业务层目标 **零 unsafe**；底层收拢 `std/sys/`、`std/ffi/` |
| **审计** | 新增 unsafe 面必须更新 `tests/baseline/safe-unsafe-api.tsv`，跑 `run-safe-unsafe-audit-gate.sh`；AI/脚本可 grep 全仓库饱和审计 |
| **实现** | ✅ U1–U3 定版 — [lang-unsafe-v1-rfc.md](./analysis/lang-unsafe-v1-rfc.md) |

### 6.2 FFI 边界零污染隔离（Safe Wrappers）

| | |
|---|---|
| **落地规范** | 经 `std/dynlib` 链接外部 C 库时，裸指针在 FFI 胶水层**强制**转换为带长度 `T[]`；禁止裸指针渗透主业务逻辑。禁止无界 C API（`strcpy`、`sprintf` 等） |
| **验收** | `run-safe-ffi-contract-gate.sh`（SAFE-004） |
| **关联** | [type-ffi-bridge-v1.md](./analysis/type-ffi-bridge-v1.md) |

### 6.3 不可信输入处理规范

| | |
|---|---|
| **落地规范** | 解析器：长度上限 + 有界 Arena；所有索引：**证明 → 检查 → panic** 三选一，禁止「这次不会越界」的假设 |
| **典型场景** | lexer/parser、HTTP body、JSON、用户路径名 |
| **关联** | [doc-memory-safety-error-v1.md](./analysis/doc-memory-safety-error-v1.md) |

### 6.4 基于 Linear(T) + sync 的线程共享限制（演进中）

| | |
|---|---|
| **目标** | 编译期静态证明：Arena 树/内存块别名计数为 1 且未跨线程 → channel 路径可消除多余原子锁，免疫 data race |
| **现实** | v1 Linear 尚未闭合 slice/struct 字段语义；当前并发共享可变须走 `std.sync` / 原子 |
| **路线** | S 级 linear/region 闭合 → A 级线程 send 证明 → TSAN gate（SAFE-006，实验线） |

### 6.5 编译器与工具链自身硬化

| | |
|---|---|
| **要求** | 解析有界、布局算术防溢出、CTFE 不截断、ASAN/LSAN 夜跑（SAFE-005） |
| **原因** | 恶意 `.sx` 不能打穿 `shux`；比用户代码 UB 更危险的是**静默生成错误代码** |
| **清单** | [tests/docs/compiler-sx-security-checklist.md](./tests/docs/compiler-sx-security-checklist.md) |

### 6.6 纵深防御（进程层，不替代 typeck）

| 机制 | 作用 |
|------|------|
| NX / DEP | 数据页不可执行 |
| ASLR + PIE | 提高 ROP 成本 |
| 栈保护 / 金丝雀 | 检测栈溢出 |
| 最小权限 | 服务不以 root 运行 |

## 7. B 级：未来进化（v1.0.0-selfhost 之后）

面向分布式量化、AI 插件生态、用户自定义脚本的跨代特性；**不阻塞**自举与 v1 发布。

### 7.1 编译期能力系统（Capability-Based Security）

| | |
|---|---|
| **思路** | 取代粗暴 OS 权限划分；标准库入口引入能力凭证 |
| **示例** | 第三方 AI 插件未声明 `std/fs.ReadCapability` → 编译器拒绝编译涉及文件读取的代码 |
| **收益** | 编译阶段完成沙箱化；零运行时权限检查税（相对 seccomp 每 syscall 拦截） |

### 7.2 Wasm / Region 轻量级沙箱

| | |
|---|---|
| **场景** | 「织梦/墨绘」等平台运行大量用户或 AI 提交的自定义脚本 |
| **选项** | 内建轻量 Wasm 运行时，或基于 region 域隔离的虚拟沙箱 |
| **目标** | 防止恶意脚本读取进程内敏感数据（如量化私钥）；与 Capability 系统组合 |

### 7.3 其他远期项

| 项 | 说明 |
|----|------|
| 侧信道 / 微架构 | 超出语言内存模型；需硬件/运行时协同 |
| 全仓库业务层 unsafe 清零 | 阶段 F 目标；底层仍集中在 `std/sys` |
| Windows/Alpine 安全变体 | 自举闭环后 — A-08/A-13 |

## 9. 我们 deliberately 不选的路

| 方案 | 为何不选 |
|------|----------|
| **默认 GC** | 与「性能比肩 C、轻量级、无 STW」目标冲突 |
| **运行时每下标检查** | 作为最终态太慢；仅作 BCE 证不出时的保底 |
| **Rust 式复杂生命周期符号** | 学习曲线与维护成本与「比 C 简单」目标冲突；用 region + arena + defer 替代 |
| **靠 UB 换优化** | 破坏内存安全与自举确定性；与项目宗旨相悖 |
| **Go 式 error 接口 + 堆逃逸** | 热路径性能毒瘤；用 enum + 寄存器 ABI |

## 10. 开发者速查

### 写业务 `.sx`

- [ ] 能否不写 `unsafe`？
- [ ] 外部输入是否有长度/范围校验？
- [ ] 是否用 `T[]` 而非裸指针传缓冲区？
- [ ] 错误路径是否处理（Result / match，不 silent continue）？
- [ ] 热路径解析是否包在 `with_arena` 内？
- [ ] 并发是否走 `std.sync`，而非共享可变裸数据？

### 改编译器 / std

- [ ] 内存大小/偏移算术是否防溢出？
- [ ] 新增 `unsafe` / `extern` 是否更新 SAFE-002 审计？
- [ ] 优化 pass 证失败是否回退？
- [ ] 是否跑过 §4 对应 gate？
