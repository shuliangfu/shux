# 条件编译

Shux 提供两套互补的条件编译机制：**行预处理 `#if`** 与 **属性 `#[cfg(...)]`**。二者共用同一套条件求值（`target_os`、`target_arch`、`-D` 符号等），但作用阶段与适用场景不同。

> **使用边界**：条件编译主要用于 **标准库与编译器内部**按平台选实现；应用层业务代码应尽量少用，且不保证语法细节长期冻结。实现细节见 [`analysis/条件编译.md`](../analysis/条件编译.md)。

---

## 一、何时用哪种

| 场景 | 推荐 | 原因 |
|------|------|------|
| 顶层 `const` / `function` / `struct` / `extern` 按平台保留或丢弃 | **`#[cfg(...)]`** | parse/typeck 前剪枝，行号稳定；可修饰单条声明 |
| 顶层 `import` 分平台（如 `std.fs` 选 `fs_posix` / `fs_win32`） | **`#[cfg(...)]`** | 未选中平台的模块不会被解析 |
| 函数内单条语句或短分支 | **`#[cfg(...)]` + 语句/块** | 与周围代码同缩进，易读 |
| 整文件大段平台代码（数十行以上） | **`#if` … `#endif`** | 预处理阶段整段跳过，避免大量重复 `#[cfg]` |
| 同一文件内互斥的多个 `main` / 整函数替换 | **`#if` / `#elseif` / `#else`** | 一次只保留一个分支 |

**经验法则**

1. **默认优先 `#[cfg]`** — 标准库 `std/net`、`std/fs`、`std/sys` 均以属性为主。
2. **长段平台差异用 `#if` 包文件级块** — 减少属性行噪音。
3. **不要混用两套语法表达同一件事** — 同一逻辑选一种风格写到底。

## 二、`#[cfg(...)]` 属性

### 2.1 语法

- 形式：`#[cfg(COND)]`，`COND` 与 `#if` 条件表达式对齐。
- 常见条件：
  - `target_os = "linux"` / `"macos"` / `"windows"`
  - `target_arch = "x86_64"` / `"aarch64"`
  - `not(COND)`，如 `not(target_os = "windows")`
  - 纯标识符（表示 `-D SYMBOL` 或编译器注入的 `SHUX_OS_*` 等）

### 2.2 顶层声明

```sx
/** 按 OS 选择平台实现模块（见 std/fs/mod.sx）。 */
#[cfg(target_os = "linux")]
const fs_platform = import("std.fs.posix");
#[cfg(target_os = "macos")]
const fs_platform = import("std.fs.posix");
#[cfg(target_os = "windows")]
const fs_platform = import("std.fs.win32");

/** 同名函数，仅保留与 host 匹配的一个（见 tests/lexer/cfg_attribute_skip.sx）。 */
#[cfg(target_os = "linux")]
function helper_os(): i32 { return 7; }
#[cfg(target_os = "macos")]
function helper_os(): i32 { return 5; }
```

未满足条件的项在 **parse 之前被移除**，typeck 看不到被剪枝的代码。

### 2.3 函数体内

**块形式**（多语句）：

```sx
function net_ipv6_poll_writable(fd: i32, timeout_ms: u32): i32 {
  #[cfg(not(target_os = "windows"))]
  {
    let pfd: PollFd = PollFd { fd: fd; events: POLLOUT; revents: 0; };
    let n: i32 = poll(&pfd, 1 as u64, timeout_ms as i32);
    if (n <= 0 || (pfd.revents & (POLLERR | POLLHUP)) != 0) { return -1; }
    return 0;
  }
  #[cfg(target_os = "windows")]
  {
    return 0;
  }
}
```

**单语句形式**（一行）：

```sx
  #[cfg(target_os = "windows")]
  if (net_ipv6_ensure_wsa() != 0) { return -1; }
```

### 2.4 与 `-target` 联动

交叉编译时传入 `-target ...linux...` 等 triple 时，`target_os` / `target_arch` 按 **目标平台** 求值，而非仅 host。烟测：`tests/run-cfg-target-triple-gate.sh`。

## 三、行预处理 `#if`

在 **词法分析之前**执行；被跳过的行以空行占位，**保持行号**便于报错定位。

### 3.1 指令

| 指令 | 说明 |
|------|------|
| `#if COND` | 条件成立则保留直到 `#else` / `#elseif` / `#endif` |
| `#elseif COND` | 前述分支均未保留时，再判断本条件 |
| `#else` | 前述分支均未保留时保留本块 |
| `#endif` | 结束当前 `#if` 块 |

支持嵌套。详见 [01-关键字 — 预处理](./01-关键字.md#预处理条件编译)。

### 3.2 条件表达式

除 `-D SYMBOL` 外，支持与 `#[cfg]` 对齐的表达式：

```sx
#if target_os == "linux"
function main(): i32 { return 41; }
#elseif target_os == "macos"
function main(): i32 { return 42; }
#else
function main(): i32 { return 43; }
#endif
```

亦支持 `target_os = "linux"`、`!=` 等（与 `cfg_eval` 共用）。

### 3.3 自动注入符号（无 `-D` 时）

编译器在 POSIX 上可通过 `uname()` 注入（详见 `analysis/条件编译.md`）：

- `SHUX_OS_LINUX`、`SHUX_OS_DARWIN` 等
- `SHUX_ARCH_X86_64`、`SHUX_ARCH_ARM64` 等

示例：

```text
#if SHUX_OS_LINUX
  // Linux 专用
#endif
```

## 四、标准库中的典型模式

### 4.1 门面 + 平台子模块（`std.sys/mod.sx`）

```sx
#[cfg(target_os = "linux")]
const linux = import("std.sys.linux");
#[cfg(target_os = "macos")]
const macos = import("std.sys.macos");
```

上层只 `import("std.sys")`，不手写 `#ifdef`。

### 4.2 平台常量勿重复 `#[cfg]` 同名顶层项

多个 `#[cfg(target_os = "linux")]` / `#[cfg(target_os = "macos")]` **同名 `const`** 会在 typeck 阶段冲突。应改为：

- 各平台独立命名，或
- **平台函数返回常量**（如 `net_dns_ai_addconfig_c()`）。

### 4.3 顶层 struct 与 let 顺序

- 所有 **`struct` 定义须在任何顶层 `let` 之前**。
- 顶层 **`let x: Struct = Struct { ... }` 暂不支持**；struct 初值请在函数内构造。

## 五、对照速查

| | `#if` | `#[cfg]` |
|---|--------|----------|
| 执行阶段 | 词法前预处理 | parse 前剪枝 |
| 作用对象 | 任意行文本 | 紧随其后的单条声明或 `{` 块 |
| 行号 | 跳过行留空行 | 移除项不占行 |
| 适合长度 | 长段、整文件 | 单声明、函数内短分支 |
| 测试 | `tests/preprocess/` | `tests/run-cfg-attribute-skip-gate.sh` |
