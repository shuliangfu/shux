# 内置与标准库

## 概述

- **core**：无 OS 依赖的核心模块，用户通过 `const m = import("core.xxx");` 引用。
- **std**：依赖 OS 的标准库，通过 `import("std.xxx")` 绑定或解构引用。
- 标准库扁平化：顶层为 `core/` 与 `std/`，无单独 `stdlib/` 目录；按需链接，不增大体积。
- **Phase 2 路线图**：根目录 `NEXT.md`；模块状态与 `core/README.md`、`std/README.md` 由对应 sync gate 校验（CORE-015 / STBL-002 / DOC-007）。

## 内建函数（builtin）

- `core.builtin` 提供位运算内建（`clz_u32` / `ctz_u32` / `popcount_u32` / `bswap_u32` / `rotl_u32` / `rotr_u32`）与 `copy`；跨模块调用映射到 `shux_builtin_*`（CORE-009/018）。
- 编译器识别 `core_mem_*` 等符号时可映射到 C `__builtin_*` 或专用实现；内建列表以 `compiler/src/codegen/codegen.c` 与专项 gate 为准。

## 使用方式

- 在入口或库文件顶部：`const types = import("core.types");`、`const io = import("std.io");` 等（详见 [05-函数与模块](./05-函数与模块.md)）。
- 编译时通过 `-L` 指定库根目录（如项目根），使 `core.xxx`、`std.xxx` 能解析到对应 `.x` 模块。
- 权威解析规则与物理布局见 `analysis/stbl-import-std-layout-v1.md`（STBL-004）；与 TOOL-007/008 包清单联动。
