# 函数与模块

## 自定义函数定义

除入口 `main` 和通过 `import` 使用的库函数外，用户在本模块内用以下方式定义**自定义函数**。

- **语法**：`function name(params): return_type { body }`
- **参数**：`name: Type`，多个用逗号分隔；无参为 `()`。
- **返回类型**：必须显式写出，如 `(): i32`、`(x: i32): bool`。
- **body**：块（block），含 const/let、循环、defer、标签/goto/return 以及**最终表达式**；也可用 `return expr;` 提前返回。有返回类型的函数须用 `return expr;` 显式返回值（不能仅靠块尾表达式）。

示例：

```su
function add(a: i32, b: i32): i32 {
  return a + b;
}
```

## extern 函数声明（FFI）

- **语法**：`extern function name(params): return_type;`（无函数体，以分号结尾）。
- **用途**：声明由 C 或其它外部编译单元提供的函数，由**链接器**解析符号；调用方式与普通函数一致。
- **不需要单独的「导入 C」**。使用 C 函数有两种方式：
  1. **本文件声明**：在调用该 C 函数的 `.x` 文件内直接写 `extern function name(...);`，即可在同一文件中调用。
  2. **通过 .x 模块**：将 C 函数的 extern 声明放在某个 `.x` 模块中（如 `std.fs`、`std.c` 等），其它文件通过绑定 import 导入该模块，即可获得这些声明并在本模块中调用；链接时再链接提供该符号的 C 库（如 `-lc` 或对应 `.a`/`.so`）。
- 常见于与 C 库互操作（如 `extern function printf(fmt: *u8, ...): i32;`），具体 ABI 与可变参数等以实现为准。

## main 与入口

- 含名为 `main` 的函数的模块为**可执行程序**。
- **main** 返回类型可为：
  - **`i32` / `i64`**：进程退出码；用 `return expr;` 显式给出（禁止仅靠隐式最终表达式）。
  - **`void`**（Zig 风格）：正常跑完等价 **exit 0**；可用 `return;` 提前成功退出。需要非零退出码时用 `i32`/`i64` main 或 `std.process` 退出 API。
- 须有函数体。无 main 的模块可作为**库**被其他模块 import。

**入口示例（推荐 Hello World 写法）：**

```su
const fmt = import("std.fmt");

function main(): void {
  fmt.println("Hello World");
}
```

等价于 C 进程入口 `int main` 成功返回 0。需要按错误码退出时：

```su
function main(): i32 {
  return 1;
}
```

## 泛型函数

- 语法：`function name<T, U>(params): return_type { ... }`
- 调用时可显式指定类型实参：`name<i32>(args)`；若可推断也可省略。
- main 不能为泛型。

## 模块与 import

- 每个 `.x` 文件是一个**模块**。
- **import 只能导入 .x 模块**，不能 `import xxx.c`。
- 路径写在字符串里，语法参考 Zig 的 `@import("path")`，写为 **`import("path")`**。

### 路径形式

| 形式 | 示例 | 说明 |
|------|------|------|
| 逻辑模块名 | `"std.io"`、`"core.mem"` | 在 `-L` 根下解析为 `std/io/mod.x` 等 |
| 相对文件路径 | `"../lib/helper.x"` | 相对**当前 .x 文件所在目录** |
| 绝对文件路径 | `"/abs/path/mod.x"` | 直接打开（注意可移植性） |

### import 写法（与 Zig `@import` 一致）

仅支持**绑定 import**：`const mod = import("path");`，通过 `mod.symbol()` 访问函数/常量；跨模块 struct/enum 类型与字面量均用**绑定名限定**（如 `process.SpawnIo`、`driver.Buffer { ... }`）。

### 模块导出（export）

- 用关键字 **`export`** 标记本模块对外 API（函数 / 方法 / struct / enum / 顶层 const 等）。
- **默认（strict）**：未写 `export` = 模块私有，其他 `.x` 不可 `m.name`。
- 回退：`XLANG_VISIBILITY=compat`（历史全开）/ `warn`（告警仍放行）/ `strict`（默认，仅 export 可跨模块）。
- **`xlang check` / LSP L7**：未 `export` 且本模块内不可达的函数 → **`warning`**（`unused private function` / code `L7`）；**不报 error、不阻断编译与运行**（默认 check exit 0；`XLANG_LINT_CI_FAIL_ON=warn` 才因 warning 失败）。LSP 诊断会话同步上报，编辑器在定义处波浪线。`XLANG_UNUSED_PRIVATE=0` 关闭。
- 与 `#[export_name]` / `#[no_mangle]` **正交**（后者是链接符号，不是模块导出表）。
- Lint：[`analysis/tool-lint-rules-v1.md`](../analysis/tool-lint-rules-v1.md) L7；回归 `tests/run-export.sh`。

| 场景 | 写法 |
|------|------|
| 函数、常量 | `const io = import("std.io");` → `io.print_str(...)`（strict 下 `print_str` 须 `export`） |
| 跨模块 struct/enum 类型 | `let io: process.SpawnIo = ...` |
| 跨模块 struct 字面量 | `process.SpawnIo { stdin_fd: -1, ... }` |
| 避免绑定名冲突 | 使用不同绑定名，如 `const io2 = import("std.io");` |
| 声明导出 | `export function print_str(...): void { ... }` |

**绑定示例：**

```su
const process = import("std.process");

function main(): i32 {
  return process.getpid();
}
```

**函数 + 跨模块类型示例：**

```su
const io = import("std.io");
const driver = import("std.io.driver");

function main(): i32 {
  let msg: u8[12] = [72, 101, 108, 108, 111, 32, 87, 111, 114, 108, 100, 10];
  io.print_str(&msg[0], 12);
  let buf: driver.Buffer = driver.Buffer { ptr: 0, len: 0, handle: 0 };
  return 0;
}
```

**相对路径示例（本地多文件）：**

```su
const helper = import("../common/helper.x");

function main(): i32 {
  return helper.run();
}
```

**若项目里自己写了 xxx.c**：不能也不需要在 .x 里 `import xxx.c`。正确做法是：（1）在某个 .x 里用 `extern function ...;` 声明 xxx.c 提供的 C 函数；（2）在需要调用的 .x 里用绑定 import 导入；（3）编译时用 C 编译器编译 xxx.c，用 xlang 编译 .x，最后**链接**在一起。

## 编译与链接

- 使用 `xlang` 编译：`xlang [ -L <lib> ] [ -target <triple> ] [ -O 0|2|s ] <file.x> [ -o <out> ]`
- `-L` 指定库/模块搜索目录；`-O` 指定优化级别（0/2/s）；`-target` 指定目标三元组。
- 多文件/多模块时，由 driver 解析 import 并一起参与编译与链接。
