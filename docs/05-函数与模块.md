# 函数与模块

## 自定义函数定义

除入口 `main` 和通过 `import` 使用的库函数外，用户在本模块内用以下方式定义**自定义函数**。

- **语法**：`function name(params): return_type { body }`
- **参数**：`name: Type`，多个用逗号分隔；无参为 `()`。
- **返回类型**：必须显式写出，如 `(): i32`、`(x: i32): bool`。
- **body**：块（block），含 const/let、循环、defer、标签/goto/return 以及**最终表达式**；也可用 `return expr;` 提前返回。有返回类型的函数须用 `return expr;` 显式返回值（不能仅靠块尾表达式）。

示例：

```su
function add(a: i32, b: i32): i32 {
  return a + b;
}
```

## extern 函数声明（FFI）

- **语法**：`extern function name(params): return_type;`（无函数体，以分号结尾）。
- **用途**：声明由 C 或其它外部编译单元提供的函数，由**链接器**解析符号；调用方式与普通函数一致。
- **不需要单独的「导入 C」**。使用 C 函数有两种方式：
  1. **本文件声明**：在调用该 C 函数的 `.su` 文件内直接写 `extern function name(...);`，即可在同一文件中调用。
  2. **通过 .su 模块**：将 C 函数的 extern 声明放在某个 `.su` 模块中（如 `std.fs`、`std.c` 等），其它文件通过解构或绑定 import 导入该模块，即可获得这些声明并在本模块中调用；链接时再链接提供该符号的 C 库（如 `-lc` 或对应 `.a`/`.so`）。
- 常见于与 C 库互操作（如 `extern function printf(fmt: *u8, ...): i32;`），具体 ABI 与可变参数等以实现为准。

## main 与入口

- 含名为 `main` 的函数的模块为**可执行程序**。
- **main** 须返回 `i32` 或 `i64`，且须有函数体；返回值通过显式 `return expr;` 给出（禁止仅靠隐式最终表达式）。
- 无 main 的模块可作为**库**被其他模块 import。

## 泛型函数

- 语法：`function name<T, U>(params): return_type { ... }`
- 调用时可显式指定类型实参：`name<i32>(args)`；若可推断也可省略。
- main 不能为泛型。

## 模块与 import

- 每个 `.su` 文件是一个**模块**。
- **import 只能导入 .su 模块**，不能 `import xxx.c`。
- 路径写在字符串里，语法参考 Zig 的 `@import("path")`，写为 **`import("path")`**。

### 路径形式

| 形式 | 示例 | 说明 |
|------|------|------|
| 逻辑模块名 | `"std.io"`、`"core.mem"` | 在 `-L` 根下解析为 `std/io/mod.su` 等 |
| 相对文件路径 | `"../lib/helper.su"` | 相对**当前 .su 文件所在目录** |
| 绝对文件路径 | `"/abs/path/mod.su"` | 直接打开（注意可移植性） |

### 仅支持两种 import 写法

| 情况 | 写法 |
|------|------|
| 用到模块内大部分 / 全部符号，且无跨模块 struct/enum 类型注解 | **绑定**：`const token = import("std.token");` → `token.method()` |
| 只用到部分符号，或需要该模块的类型 | **解构**：`const { m1, m2, Buffer } = import("std.io.driver");` → 类型与函数直接用裸名 `Buffer`、`m1()` |
| 解构符号与本地已有名冲突 | **解构 + as**：`const { print_str as io_out } = import("std.io");` → 本地用 `io_out()` |

> **类型规则**：编译器暂不支持 `mod.Type` 限定；解构导入的类型与函数须写裸名（或 as 后的别名），不能写 `driver.Buffer`。

**绑定示例：**

```su
const process = import("std.process");

function main(): i32 {
  return process.getpid();
}
```

**解构示例（函数）：**

```su
const { print_str } = import("std.io");

function main(): i32 {
  let msg: [12]u8 = [72, 101, 108, 108, 111, 32, 87, 111, 114, 108, 100, 10];
  let n: i32 = print_str(msg, 12);
  return if (n >= 0) { 0 } else { 1 };
}
```

**解构 + as 别名示例（避免同名冲突）：**

```su
const { print_str as io_out } = import("std.io");
const { Buffer as IoBuf } = import("std.io.driver");

function main(): i32 {
  let msg: [12]u8 = [72, 101, 108, 108, 111, 32, 87, 111, 114, 108, 100, 10];
  io_out(&msg[0], 12);
  let buf: IoBuf = IoBuf { ptr: 0, len: 0, handle: 0 };
  return 0;
}
```

**解构 + 类型示例：**

```su
const { Buffer } = import("std.io.driver");

function main(): i32 {
  let buf: Buffer = Buffer { ptr: 0, len: 0, handle: 0 };
  return 0;
}
```

**相对路径示例（本地多文件）：**

```su
const helper = import("../common/helper.su");

function main(): i32 {
  return helper.run();
}
```

**若项目里自己写了 xxx.c**：不能也不需要在 .su 里 `import xxx.c`。正确做法是：（1）在某个 .su 里用 `extern function ...;` 声明 xxx.c 提供的 C 函数；（2）在需要调用的 .su 里用解构或绑定 import 导入；（3）编译时用 C 编译器编译 xxx.c，用 shu 编译 .su，最后**链接**在一起。

## 编译与链接

- 使用 `shu` 编译：`shu [ -L <lib> ] [ -target <triple> ] [ -O 0|2|s ] <file.su> [ -o <out> ]`
- `-L` 指定库/模块搜索目录；`-O` 指定优化级别（0/2/s）；`-target` 指定目标三元组。
- 多文件/多模块时，由 driver 解析 import 并一起参与编译与链接。
