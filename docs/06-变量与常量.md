# 变量与常量

## let（变量）

- 语法：`let name: Type = init_expr;` 或 **`let name: Type;`**（省略 `=`，零初始化）
- **类型注解**：必须写，无类型推断省略形式。
- **初始值**：
  - 带 `=`：按右侧表达式初始化（与上下文类型 coerce 规则一致）。
  - **省略 `=`**：按声明类型**零初始化**（栈槽清零），语义等价于 `let buf: u8[N] = []`、`let n: i32 = 0` 等；适用于标量、指针、定长数组、结构体、SIMD 向量、联合类型等所有 `let` 可声明类型。
  - **`const` 仍须** `= init_expr`（编译期常量表达式）。
- **作用域**：块内；从声明处到块结束。
- **可变性**：当前语义下 let 绑定可被重新赋值（如在 for 的 step 中 `i = i + 1`），具体以实现为准。

示例：

```su
let x: i32 = 1;
let y: i32 = x + 2;
let buf: u8[64];          // 等价 let buf: u8[64] = []（全零）
let z: i32x4;             // 向量零初始化
let p: Point;             // 结构体零初始化
let total: isize = 0;   // 字面量 0 按 isize 收窄
let p: *u8 = 0;         // 指针语境下的 null
```

## 上下文类型

`let` / `const` 声明右侧的整型字面量、`0` 空指针等，按左侧**类型注解**做编译期 coerce（与赋值、`return`、实参、struct 字段规则一致）。须写类型注解；无「省略类型的 let 推断」。

`const` 与 `let` 在字面量 coerce 上行为一致，例如 `const MAX: u32 = 8` 无需 `8 as u32`。

## const（常量）

- 语法：`const name: Type = init_expr;` 或 `const name = init_expr;`（类型可选，由常量表达式推断）。
- **初始值**：必须为**常量表达式**（编译期可求值，如字面量、其它 const、简单运算）。
- **作用域**：块内或顶层；从声明处到块/模块结束。
- **不可变**：const 绑定不可重新赋值，生成 C 时对应 `const`。

示例：

```su
const N: i32 = 10;
let x: i32 = 1 + 2;
let y: i32 = x + N;
```

## 块与顺序

- 块内可有多条 `const` 与 `let`，按书写顺序出现；使用前须已声明。
- const 与 let 后可跟循环、defer、标签/goto/return，最后可为**最终表达式**（块的值）；若函数需要返回值，通常用 `return expr;` 显式返回。
