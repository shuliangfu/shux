# 语法规范

本文档约定 Shu 源码的**书写格式与分号、缩进等规范**，便于统一风格、避免语法歧义。与 [01-关键字](./01-关键字.md) 等篇目配合使用。

---

## 一、缩进

- **缩进单位**：统一使用 **2 个空格**，不使用 Tab。
- **作用**：块（函数体、`if`/`while`/`for` 体、`defer` 块等）内代码相对块首行缩进一级；嵌套块逐级增加 2 空格。

示例：

```su
function main(): i32 {
  let x: i32 = 1;
  if (x > 0) {
    let y: i32 = 2;
    return y;
  }
  return 0;
}
```

---

## 二、分号

### 2.1 总则：表达式以分号结尾，除非以 `}` 结尾

- **所有表达式/语句都必须以分号结尾**（否则会与下一句粘连，解析报错）。
- **唯一例外**：若该表达式**以 `}` 结尾**，则**禁止**写分号；写 `};` 会报错「semicolon not allowed after '}'」。单行、多行皆然。

示例：

| 写法 | 是否以 `}` 结尾 | 分号 |
|------|------------------|------|
| `return var;` | 否 | **必须**写分号 |
| `return {}`、`return Lexer { ... }` | 是 | **禁止**写分号（单行、多行都一样） |
| `let x = 0;` | 否 | 必须写分号 |
| `let x = S {}` | 是 | 禁止写分号 |

### 2.2 必须加分号的情况（不以 `}` 结尾时）

| 类别 | 示例 |
|------|------|
| 声明 | `let x: i32 = 0;`、`const N: i32 = 10;`、`const io = import("std.io");` |
| 控制流（语句形式） | `return expr;`、`return;`、`goto label;`、`panic;`、`panic(msg);`（若 `return expr` 以 `}` 结尾则不加分号，见上） |
| 表达式语句 | 块内表达式且**不是**块最后一项时须加分号：`f();`、`x = 1;` |
| **类型定义** | **struct 字段之间、trait 方法声明之间用分号 `;`**。详见 [02-类型定义](./02-类型定义.md)。枚举变体之间用逗号。 |

### 2.3 块与最终表达式

- **块（block）** 形式为：`{ stmt; stmt; ... expr }`。
- 块内**最后一项**可以是**最终表达式**（无分号），其值作为块的值。
- 若最后一项是语句且**不以 `}` 结尾**，则须带分号；**以 `}` 结尾**则不能带分号。

### 2.4 match 分支

- 每个分支形如 `pat => expr;`，分支之间用分号分隔。
- 整数字面量模式可合并：`1 | 2 | 3 => expr;`（与 Rust 多模式语法类似）。
- match 整体为表达式。

### 2.5 大括号 `}` 后禁止分号

- 无论是块（函数体、if/else、match、for、defer、struct/enum/trait/impl 定义）还是结构体字面量，**`}` 后面一律不得紧跟分号 `;`**，写 `};` 会报错。
- 因此：`return Foo { }`、多行 `return Foo { a: 1,\n  b: 2 }` 等都以 `}` 结尾，**都不能**写分号。

---

## 三、空格与换行

- **运算符两侧**：建议加空格，如 `a + b`、`x = 1`、`i < 10`。
- **逗号后**：建议加空格，如 `f(a, b)`、`let x: i32 = 1;` 中类型与初值之间按语法已有冒号与空格。
- **冒号**：类型注解 `name: Type` 中冒号后加空格。
- **行宽**：建议单行不超过约 80～100 字符；过长时可换行缩进，保持可读性（具体以项目约定为准）。

---

## 四、命名

| 类别 | 建议 | 示例 |
|------|------|------|
| 函数 | 小写 + 下划线（snake_case） | `get_value`、`main` |
| 变量 / 常量（let / const） | 小写 + 下划线（snake_case） | `user_name`、`MAX_LEN` |
| 类型（结构体、枚举、trait） | 大驼峰（PascalCase） | `Buffer`、`TokenKind` |
| 模块路径（import） | 小写 + 点分隔 | `core.types`、`std.io` |
| 标签（label） | 与变量类似，snake_case | `cleanup:`、`done:` |

关键字与内建类型名（如 `function`、`i32`、`bool`）为保留，不可用作标识符。

---

## 五、块内顺序（推荐）

块内书写顺序建议为（与解析顺序一致，便于阅读）：

1. `const` 声明
2. `let` 声明
3. `while` / `loop` / `for` 循环
4. `defer { ... }` 块
5. 可选：`label:`、`goto label;`、`return expr;` / `return;`
6. 可选：若干**表达式语句**（`expr;`）
7. 可选：**最终表达式**（无分号，作为块的值）

函数体若需返回值，建议使用显式 `return expr;`，避免仅依赖块尾表达式。

---

## 六、注释与预处理

- **注释**：行注释 `//`、块注释 `/* ... */`、井号行注释 `# ...`（见 [01-关键字](./01-关键字.md)）。注释与代码之间建议留空格。
- **预处理**：`#if` / `#elseif` / `#else` / `#endif` 单独成行，与正文缩进一致即可。完整说明与 `#[cfg(...)]` 对照见 [09-条件编译](./09-条件编译.md)。

---

## 七、与实现的对应关系

- 分号规则与编译器 `parser` 一致：声明、return/goto/panic、表达式语句后必须 `;`；块尾最终表达式无分号；`}` 后可选分号。
- 缩进不改变语义，但 2 空格为文档与示例的约定，推荐统一使用。
