# 类型定义

> **状态（2026-07-24）**  
> - **已实现 / 主表正式**：整数族（含 `i8`/`i16`/`u16`）、浮点、`bool`、指针宽、SIMD（i32/u32 向量）、复合类型。  
> - **无语言级 `string`/`str`**：文本见 §「文本与字节（当前）」；**语言级 string/str 与插值 → 自举完成后实现**。  
> - 时机与缺口总表：`analysis/基础类型缺口与自举时机分析.md`。

## 内建类型

### 整数

| 类型 | 说明 | size / align（64 位目标） |
|------|------|---------------------------|
| `i8` | 8 位有符号整数 | 1 / 1 |
| `i16` | 16 位有符号整数 | 2 / 2 |
| `i32` | 32 位有符号整数（常用默认整型） | 4 / 4 |
| `i64` | 64 位有符号整数 | 8 / 8 |
| `u8` | 8 位无符号整数（**字节**；文本缓冲元素） | 1 / 1 |
| `u16` | 16 位无符号整数 | 2 / 2 |
| `u32` | 32 位无符号整数 | 4 / 4 |
| `u64` | 64 位无符号整数 | 8 / 8 |
| `isize` | 与指针同宽的有符号整数 | 8 / 8 |
| `usize` | 与指针同宽的无符号整数（长度、下标、指针运算） | 8 / 8 |

说明：

- `i8` / `i16` / `u16` 在 typeck 中为 **NAMED 整型**（与 first-class `i32`/`u32` 等并列参与整数拓宽、`size_of`/`align_of`）。
- 整数拓宽（let / assign / return / call 等）在安全方向允许（如 `i8`→`i32`、`u16`→`u32`）；**反向收窄须显式 `as`**。
- `core.types` 提供 `size_of_i16` / `size_of_u16` / `align_of_*` 及泛型 `size_of<T>` / `align_of<T>`（见 [07-内置与标准库](./07-内置与标准库.md)）。

### 布尔与浮点

| 类型 | 说明 | size / align（64 位目标） |
|------|------|---------------------------|
| `bool` | 布尔；**不**从整数隐式转换 | 1 / 1（布局以 ABI 为准；逻辑宽度 1） |
| `f32` | 32 位浮点 | 4 / 4 |
| `f64` | 64 位浮点 | 8 / 8 |

- 浮点与整数互转须 **`as`**（部分路径仍在 Cap residual 补齐中，以探针/矩阵为准）。
- `f32`→`f64` 拓宽在赋值/返回等路径支持；反向收窄须 `as`。

### 指针宽度（与上表重复列出便于检索）

| 类型 | 说明 |
|------|------|
| `usize` | 长度、容量、数组下标、与指针同宽的无符号量 |
| `isize` | 有符号指针宽整数（偏移、部分系统 API） |

### SIMD 向量

| 类型 | 说明 |
|------|------|
| `i32x4` / `i32x8` / `i32x16` | 4/8/16 车道 `i32` 向量（128 / 256 / 512-bit） |
| `u32x4` / `u32x8` / `u32x16` | 4/8/16 车道 `u32` 向量 |

> **缺口（自举后 / 按需）**：更多车道与元素类型（如 `i8x16`、`f32x4`、`u8x16` 等）见 §「规划中的类型（自举后）」。

### 函数返回与「无值」

| 形式 | 说明 |
|------|------|
| `: void` | 函数无有效返回值（已支持） |
| 表达式类型 `void` 一等公民 | **未**作为可写在 `let x: void` 的完整一等类型推广 |

`never` / `!`、unit `()` 见 §「规划中的类型（自举后）」。

## 文本与字节（当前 · 无语言级 string）

**当前没有**关键字类型 `string` / `str`。不要写：

```xlang
// 尚未支持（自举完成后规划）
// let s: string = "Hello World";
```

### 字面量与缓冲

| 写法 | 类型语义 | 用途 |
|------|----------|------|
| `"hello"` | 默认 **`*u8`**（C 互操作）；typeck 亦可匹配 **`u8[]`** 形参（如 `print`/`println` 单参） | FFI、格式化、路径字节 |
| `u8[N]` | 定长字节数组 | 栈缓冲 |
| `u8[]` | 字节切片（指针 + 长度） | 视图、API 入参 |
| `*u8` + 显式长度 | 裸缓冲 | 底层 IO / 系统调用 |

示例（当前推荐）：

```xlang
function main(): i32 {
  let msg: *u8 = "Hello World";
  // 或栈缓冲 + 长度；或 import std.string / core.str 使用视图与拥有串 API
  return 0;
}
```

### 标准库 / core（库类型，不是内建关键字）

| 模块 | 类型 / API | 语义 |
|------|------------|------|
| `core.str` | `BytesView` 等 | 无 OS 依赖的只读字节视图 `(ptr, len)` |
| `std.string` | `String`、`StrView` | 库级**拥有串**与**视图**；堆与生命周期由 API 约定 |
| `std.fmt` / `std.io` | `print` / `format` 等 | 多以 `*u8` + 长度或 `u8[]` 入参 |

详见 [07-内置与标准库](./07-内置与标准库.md)。

### 规划（**自举完成后实现**）

| 规划项 | 目标形态（示意） | 说明 |
|--------|------------------|------|
| 语言级 `string` | `let s: string = "Hello World";` | 拥有文本类型；布局/析构/与 `std.string.String` 关系在 RFC 中定 |
| 语言级 `str` | 参数/视图 `s: str` | 非拥有视图；宜与 `StrView` / `u8[]` 收敛，避免三套权威 |
| 字符串插值 | `"x={n}"` 类 | 依赖 fmt + 临时缓冲策略；与 `string`/`str` 同阶段或紧随 |
| `*u8 + *u8` 运行时拼接 | 运算符糖 | 可选；当前用 concat API；**自举后**再定是否做运算符 |

时机权威：`analysis/基础类型缺口与自举时机分析.md`。  
**自举期禁止**实现语言级 `string`/`str` 或半残 keyword 别名冒充拥有语义。

## 复合类型

### 指针

- 裸指针：`*T`，如 `*i32`、`*u8`。
- **volatile 指针**：`*volatile T`，确保 MMIO 读写不被编译器消除（用于内核硬件寄存器访问）。
  ```xlang
  let mmio: *volatile u32 = 0xB8000 as *volatile u32;
  unsafe { *mmio = 0x0741; }  // 写不被 DCE
  ```

### 数组

- 固定长数组：**`T[N]`**（C 风格），如 `u8[10]`、`i32[512]`；`N` 为编译期常量。
- **多维**：**`T[N][M]`**（及更深），与 C 一致：外层 `N`、内层 `M`。例如 `i32[2][3]` 与前缀写法 **`[2][3]i32`** 同义；下标 `a[i][j]` 中 `i` 走外层、`j` 走内层。wave356 起后缀连写多维在 layout 解析中完整登记（此前只吃一层 `[N]` 导致 `T[N][M]` 解析失败）。

### 切片

- 切片：**`T[]`**，如 `u8[]`、`i32[]`；语义为「指针 + 长度」；长度成员只写 **`.length`**（**禁止**简写 `.len`）。固定数组 **`T[N]`** / 向量同理，编译期 N 亦经 **`.length`** 读取。
- 域标签：`u8[]<label>`（M-3 region slice）。

### 结构体

- 定义：`struct Name { field: Type; ... }`
  - **C ABI 布局**：`#[repr(C)] struct Name { ... }`，按 C ABI 布局（允许隐式 padding），用于硬件描述符、FFI 接口。
  - **字段分隔**：定义体字段之间可用 **分号 `;` 或逗号 `,`**（wave355 起与字面量一致）。例如：`struct Point { x: i32; y: i32 }` 与 `struct Point { x: i32, y: i32 }` 均合法。历史仅 `;` 时误用 `,` 会留下不完整 layout（只登记首字段）——已根修。
  - **空结构体**：`struct Empty {}` 合法（零字段，ZST，`sizeof` 与 host C 同为 0）。host-C 发完整 `struct Tag { };`（wave365；历史 `nf<=0` 跳过定义 → incomplete type BLD001）。字面量 `Empty {}`。嵌套字段（如 `struct Box { e: Empty, n: i32 }`）layout metrics 接受 `fsize==0`（wave366；历史 `fsize<=0` → T001）。freestanding asm 对 `Empty {}` STRUCT_LIT（`nf==0`）发零字段基址（wave366；历史 `nf<=0` → CG002）。**末字段**字面量 `A { x: 42, e: Empty {} }`（wave367；wave364 空体消歧曾把外层 `}` 当 bare block → P001）。**全空字段嵌套 ZST**（`struct Nest { e: Empty }` 作另一结构体字段）亦合法（wave368；`is_empty` 递归「全字段皆空」）。**中部/末位** empty-of-empty 嵌套字段 freestanding layout（`Box { a, n: Nest{e:Empty{}}, b }`）与 host 同（wave369；历史 ensure 误 append 字段表 + ZST store 写指针 → 乱偏移）。
- **无填充布局**：`struct packed Name { field: Type; ... }`，与 C `__attribute__((packed))` 一致，无隐式 padding、对齐为 1。
- 支持泛型：`struct Name<T> { ... }`
- 字面量：`Name { field: expr, ... }`（字段之间用 **逗号或分号** 均可）
- 字段访问：`value.field`（语言侧**只有** `.`，没有 `->`；指针成员在生成 C 时由 codegen 决定 `.` 或 `->`）
- **位域**：字段后加 `: N` 指定位宽（C 风格位域，用于紧凑硬件寄存器）：
  ```xlang
  struct PageFlags { present: u32 : 1; writable: u32 : 1; addr: u32 : 20; }
  ```
  > 位域字段宽度 N 为正整数，codegen 发射 C 位域 `uint32_t field : N;`。

### 枚举

- 定义：`enum Name { VariantA, VariantB, ... }`（无负载枚举）；**变体之间用逗号 `,` 分隔**（与 struct 字段用分号不同）。
- 变体引用：`Name.VariantA`（类型名 + 变体名）
- 在 `match` 中可对整型或枚举值按变体分支。
- > **缺口（后期）**：带负载代数数据类型（如 `Option<T>` 语言级 enum）以 `core.option` / `core.result` **库结构体**为主路径；语言级 ADT 增强不在自举硬闸门内。

### Trait 与 impl

- Trait 定义：`trait TraitName { function method(...): RetType; ... }`；**方法声明之间用分号 `;` 分隔**（与 struct 字段一致，不要用逗号）。
- 实现：`impl TraitName for TypeName { function method(self: TypeName, ...) { ... } }`
- 方法调用：`value.method(args)`，由 typeck 解析到对应 impl 中的函数。

## 类型注解

- 变量/常量：`let x: i32 = 0;`、`let b: i8 = 1;`、`let w: u16 = 0;`、`let buf: u8[10];`（省略 `=` 为零初始化）、`const N: i32 = 10;`
- 函数参数与返回值：`function f(x: i32, y: bool): i64 { ... }`、`function g(): void { ... }`
- 泛型参数：`function f<T>(x: T): T { ... }`、`struct S<T> { ... }`
- 多类型 API：统一使用**函数重载**表达。

## 函数重载（Overload）

同一模块内可定义**多个同名函数**，形参列表（类型与个数）不同：

```su
function pick(x: i32): i32 { return x + 1; }
function pick(x: i64): i32 { return x as i32 + 2; }

function main(): i32 {
  pick(10);           // → i32 重载
  pick(20 as i64);    // → i64 重载
  return 0;
}
```

调用规则：

| 规则 | 说明 |
|------|------|
| 分派依据 | 实参已 typecheck 后的类型，与候选形参做 assignability 匹配 |
| 优选 | **精确类型匹配**优先于隐式拓宽（如 `u32` 实参优先选 `u32` 形参，而非 `i32`） |
| 歧义 | 多个候选同分 → 报错 `ambiguous call to overloaded function` |
| 未匹配 | 报错 `no matching overload for 'name'` |
| `main` | 仍须唯一，不可与其他 `main` 重载 |
| 跨模块 | `import` 后 `mod.func(...)` 同样按重载解析；codegen 对多 overload 做 C 符号 mangling（如 `format_2_i32_u32`） |

标准库示例（`std.fmt`）：

```su
const fmt = import("std.fmt");
fmt.format(buf, cap, 10, 20);                   // i32, i32
fmt.format(buf, cap, 1 as usize, 2 as usize);  // usize, usize
```

无 `format_i32_i32` 等兼容别名；同名即 API。

## 规划中的类型（自举后实现）

下列**不是**当前语言内建类型。文档在此登记，避免与「已实现」混淆。  
**实现时机：自举主闸门关闭之后**（或标注为更后期）；自举期不实现、不假绿。

| 优先级 | 类型 / 能力 | 目标 | 当前替代 | 时机 |
|--------|-------------|------|----------|------|
| **P0** | `string`（拥有） | `let s: string = "…"` | `std.string.String` / 手动缓冲 | **自举后** |
| **P0** | `str`（视图） | 非拥有文本视图类型 | `StrView` / `BytesView` / `u8[]` | **自举后**（宜与 string 同 RFC） |
| **P0** | 字符串插值 | `"a{x}"` 类 | `std.fmt` 等 | **自举后**（随 string 或紧随） |
| **P1** | `char` / `rune` | Unicode 标量值类型 | `u8` 字节或 `u32` + `std.unicode` | **自举后**（宜随 string） |
| **P2** | `*u8 + *u8` 等拼接运算符 | 运行时字符串拼接糖 | concat / arena API | **自举后**（可选） |
| **P3** | `void` 一等类型 | `let`/泛型中的 void | 函数返回 `: void` | 可选 / 后期 |
| **P3** | `never` / `!` | 发散函数（panic、无限循环） | 无精确类型标记 | 后期 |
| **P3** | `()` unit | 空元组表达式类型 | `: void` / 不写值 | 可选 |
| **P4** | `f16` / `f128` | 半精度 / 扩展精度 | `f32` / `f64` | 很后期 |
| **P4** | 更多 SIMD | `i8x16`、`f32x4`、`u8x16` 等 | 现有 `i32x*` / `u32x*` | 按性能热点 |
| **P4** | `u128` 等 | 宽整数 | 多字 + 库 | 按需 |

### 自举期文本策略（强制）

1. 使用 `*u8` / `u8[]` / `u8[N]` 与 `core.str` / `std.string`。  
2. **不要**在产品代码或测试中依赖尚未存在的 `string`/`str` 关键字。  
3. 不新增「`string` = `u8[]`」半残别名冒充拥有语义（避免日后真 `string` 双权威）。
