### 1.6 语义特性

- **分号可选**：block 尾表达式 / return / let 后分号可省略（与 parser.x `parser_rewind_lex_for_following_stmt` 一致）
- **struct 字段分隔符**：逗号 / 分号 / 仅换行均可
- **注释**：`//`（行）、`/* */`（块，含 `/** */`）、`#`（hash）

### 3.2 P1：查询文件增强

#### 3.1.1 locals.scm（作用域高亮）

```scheme
; 变量声明 vs 函数名 vs 类型名差异化着色
(function_declaration name: (identifier) @local.function)
(let_declaration name: (identifier) @local.variable)
(struct_declaration name: (type_name) @local.type)
```

**价值**：编辑器能区分局部变量、函数名、类型名，提供更精准的高亮。

#### 3.1.2 indents.scm 细化

当前缩进规则较基础，需补充：
- `match_arm` 的 `=>` 后缩进
- `struct_literal` 字段换行缩进
- `if_expression` 多行条件缩进
- 链式方法调用 `.method()` 换行缩进

#### 3.1.3 injections.scm（嵌入式语言）

如果 XLANG 字符串支持插值（如 `f"val={x}"`），添加注入查询。

### 3.3 P2：编辑器集成

## 四、grammar 维护原则

遵循 AGENTS.md「禁止双权威」原则：

1. **grammar.js 必须与 lexer.x / parser.x 同步演进**
   - 新增关键字时，先查 `lexer.x` 的 `try_keyword`（字节码匹配）
   - 新增语法结构时，先查 `parser.x` 的解析逻辑

2. **不猜测语法**
   - 所有 grammar 规则必须有真实 .x 源码佐证
   - 不为了"完整性"添加 XLANG 不支持的语法

3. **conflict 声明必须有注释**
   - 说明歧义场景
   - 说明为何 conflict 是必要的

4. **修改后必须跑全量测试**
   - `npx tree-sitter generate && npx tree-sitter test`
   - `find compiler/src std examples -name "*.x" | xargs -I{} npx tree-sitter parse --quiet {}`
   - 通过率不得下降
