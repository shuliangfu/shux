# Shux VSCode 扩展 — 打包与安装说明

本文档记录扩展的**编译、打包顺序**以及**在 node 不在 PATH 时的处理方式**，便于本机或 CI 复现。

---

## 一、操作顺序（推荐）

按以下顺序执行，在扩展目录下进行：

1. **安装依赖**（首次或 package.json 变更后）
   ```bash
   cd editors/vscode
   npm install
   # 或
   yarn
   ```

2. **编译 TypeScript**
   ```bash
   npm run compile
   # 或
   yarn compile
   ```
   产出在 `out/extension.js`、`out/extension.js.map`。

3. **打包 VSIX**（不再自动执行 compile，需先完成第 2 步）
   ```bash
   npx vsce package
   ```
   生成 `vscode-shux-0.1.0.vsix`（版本号以 package.json 为准）。

4. **安装**
   - 方式一：在 VSCode/Trae 中：**扩展** → **⋯** → **从 VSIX 安装...** → 选择生成的 `.vsix` 文件。
   - 方式二（命令行，推荐）：
     ```bash
     # 首次安装
     trae-cn --install-extension vscode-shux-0.1.0.vsix

     # 已安装过，覆盖更新（加 --force）
     trae-cn --install-extension vscode-shux-0.1.0.vsix --force
     ```

5. **重新加载窗口**
   - `Cmd+Shift+P` → `Developer: Reload Window`

---

## 二、一键打包+安装

编译 + 打包 + 安装一步完成：

```bash
cd editors/vscode
npm run compile && npx vsce package && trae-cn --install-extension vscode-shux-0.1.0.vsix --force
```

安装后 `Cmd+Shift+P` → `Developer: Reload Window` 重新加载。

---

## 三、图标不生效排查

重新安装后图标未更新，按以下顺序排查：

1. **完全退出编辑器**（`Cmd+Q`，不是关窗口）再重新打开——reload window 不会刷新图标资源缓存。

2. **清除图标缓存**（完全退出后执行）：
   ```bash
   rm -rf ~/Library/Application\ Support/Trae\ CN/Code\ Cache
   rm -rf ~/Library/Application\ Support/Trae\ CN/Cache
   ```
   然后重新打开 Trae。

3. **检查文件图标主题**：`Cmd+Shift+P` → `File Icon Theme` → 如果选了 Seti / Material 等主题，可能覆盖语言级图标，切到 `None` 测试。

4. **验证安装目录图标是否更新**：
   ```bash
   md5 ~/.trae-cn/extensions/shux.vscode-shux-0.1.0/icons/x-128.png
   md5 editors/vscode/icons/x-128.png
   ```
   两个 MD5 一致说明文件已正确安装，问题在缓存；不一致说明安装失败，重新执行步骤 1。
