## 0. 本轮结论（2026-07-15 晚）

### 已根治并本地验证（mac L2，`-backend c` / g05）

**产品矩阵（mac，本波 shux）**：rv=42 / hello OK / mem OK / string OK / base64 OK / json OK / csv OK

### 权威原则（本轮固化）

1. **有预编 `std/*.o` 的模块**：用户程序 **只链 `.o`，不 co-emit 函数体**（`pipeline_codegen_std_dep_link_only`）。
2. **库模块自编 `.o`**：入口 pin 同包时 **仍 emit**（否则 csv.x 体丢失）。
3. **invoke_cc**：`http`/`json`/`csv`/`net`/… **按生成 C 真实引用**按需链入；禁止无条件硬链。
4. **hello ≠ net.o**：未引用的 std 不得入链（产品 C 后端已验）。

### 0.1 未用 std 不链接 — 全面检查（2026-07-16）

#### 产品轨 C 后端（Darwin 默认 `-backend c`）— **已按需**

| 机制 | 位置 | 作用 |
|------|------|------|
| `use_line` 扫描 | `link_abi_generated_c_contains_substr_use_line` | 忽略 `extern`/`#define`/`struct`/`typedef`/placeholder，只认真实调用行 |
| 模块 need_* | `shux_invoke_cc_impl` ~3855+ | 各 std 仅 `need_X` 时 `push_existing` |
| net 针脚 | `net_api[]` 具体 API | **禁止**裸 `std_net_`（preamble 假阳性） |
| context | 仅 entry API | 不把 co-emit 体内的 `is_cancelled` 当 need |
| dead_strip / gc | Darwin `-Wl,-dead_strip`；Linux `--gc-sections` | 未引用 co-emit 体不拖依赖 |
| heap | 产品路径 `heap_o=NULL` + on_demand | 非恒链 |
| `ensure_std_net_o_auto_tls` | 仅 `SHUX_NET_TLS` 非空才 work | 默认不碰 net |

**实测（本波 `compiler/shux_asm`，mac）**：

| 程序 | 体积 | `std_net*` / heavy std 符号 | 生成 C 含 `std_net_` |
|------|------|------------------------------|---------------------|
| return-value | ~16K | 0 / 0 | — |
| hello | ~33K | 0 / 0 | **0** |
| mem | ~16K | 0 / 0 | — |
| net test | ~34K | **有** | 有 |

hello `main` 仅 `std_fmt_print_u8_ptr_i32`；print 体 co-emit 在用户 C，**不链 `net.o`/`json.o`/`http.o`/`context.o`**。

#### ASM 后端 residual（Linux 金标 / `-backend asm`）

| 路径 | 行为 |
|------|------|
| `shux_asm_ld_append_std_objs` | **仍 bulk-push** `labi_std_plan`（~58 步：string/http/json/csv/thread/… 只要 `.o` 存在就入 argv） |
| `shux_asm_ld_append_on_demand_user_objs` | net/set/map/heap 等 **按 U 符号**按需 |
| **net.o** | **不在 bulk plan**，只走 on_demand（`link_abi_user_o_needs_std_net`） |

→ ASM 上 hello **也不会链 net.o**，但仍可能 bulk 链入已存在的 http/json/csv 等（与「未用勿链」原则不符）。  
根治方向：bulk 仅留 io_stubs/panic 等最小 runtime；其余一律 on_demand（须扩 `labi_od_simple_group` / needs 表，再跑 Ubuntu asm 矩阵）。**未在本轮改**，避免 bstrict 半绿。

#### 仍注意

| 项 | 说明 |
|----|------|
| Ubuntu 全量 bstrict | 待 push + L2/L3 金标 |
| `run-all-bstrict` 冷构建 | 默认 bootstrap 会盖 L2；验产品用 `SHUX_BSTRICT_SKIP_BUILD=1` |
| Darwin 默认 `-o` | 走 C backend；asm 微测以 Linux x86_64 为准 |
| hello 生成 C 仍大 | preamble/co-emit 大量 `extern std_io_*`（~598）；**链接**已按需，体积/扫描成本另题 |
| ASM bulk std | 见上 residual |

### 0.17 上帝视角问题地图：unsafe 后赋值 cast（2026-07-16）

#### 0.17.1 完整数据流（谁产生 / 谁存 / 谁消费）

```
std/*.x 源
  → parser（函数体 / unsafe region / let / assign）
  → typeck check_block（stmt_order：let → region(unsafe push/pop) → expr_stmt assign）
       ├ pipeline_typeck_check_block_one_region_c  push/pop g_typeck_unsafe_depth
       ├ typeck_check_expr_assign → check 左 VAR / 右 EXPR_AS
       └ 失败时：函数返回类型塌成 `?` 或体被静默丢弃
  → codegen emit_func
       ├ 坏函数：体不发射；部分 let 出现在 top-level init_globals（假绿）
```

#### 0.17.2 最小复现矩阵（产品 `shux_asm -check`，Ubuntu 已测）

| ID | 模式 | 结果 |
|----|------|------|
| gA | `let s=0` → `unsafe{memset}` → **`s = p as *T`（块外）** | **红**（main 见返回 `?`） |
| gB | 同上但 **assign 写在 unsafe 内** | 绿 |
| gC | `unsafe { s = calloc(...) as *T }` | 绿 |
| gD | `unsafe{memset}` → **`let s = p as *T`（一次绑定）** | 绿 |
| gE | gA 但 `*u8` | **红**（非仅 struct） |
| gF | gD + 字段写 `s.magic=` | 绿 |
| gG | 无 unsafe，仅 `s = p as *T` | 绿 |
| gH | **空** `unsafe{}` 后 `s = p as *T` | **红**（与 memset 无关） |
| gMulti | 坏函数 + 好函数；main **不调用**坏函数 | **假绿**：typeck OK，坏函数体消失，`s` 进 `init_globals` |

**精确触发条件**（当前证据）：

> 函数体内 **任意** `unsafe { ... }` region **关闭之后**，再对已绑定局部做 **`s = <expr> as *T` 赋值** → typeck 失败 / 体静默丢失。  
> **同一赋值写在 unsafe 内** 或 **改用 `let s = ... as *T` 新绑定** → 正常。

#### 0.17.2b 解析层铁证（比 typeck 更底层，2026-07-16 补）

| 源形态 | `parse OK num_funcs` | 生成 C |
|--------|----------------------|--------|
| 无 unsafe：`let s=0; s=p as *T; return 0` + 另一函数 | **2** | 两函数体完整 |
| 空 `unsafe{}` 夹在 let 与 assign 之间 + 另一函数 | **1**（坏函数**未入 AST**） | 仅好函数；`s` 成 **top-level** `init_globals` |
| `unsafe{memset}` 后 assign + 另一函数 | **2** | 坏函数体仍丢；`s` 仍可能进 init_globals；typeck 可假绿 |

结论（更新产生点排序）：

1. **parser（第一嫌疑）**：`unsafe { }` region 关闭后，函数体后续 stmt 解析/挂接失败 → 函数未登记或体残缺，局部 let 被抬成 module top-level。  
2. **typeck（第二）**：对残缺/半挂接 AST 报 `?` 或整模块假绿（无调用点时）。  
3. **codegen（第三）**：忠实发射残缺 AST（init_globals + 缺符号），**不是**首恶。

根修必须先查 **parser 对 function-body 内 unsafe region 的 stmt_order / 括号恢复**，再修 typeck 假绿；禁止只在 std 或 codegen 打补丁。

## 1. 核心阻塞 A：`core_mem_placeholder`（已修）

### 1.1 现象

```
Undefined symbols: _core_mem_placeholder
ld: symbol(s) not found
```

`tests/mem/main.x` 经 `import("core.mem")` 调用 `mem.placeholder()` 等；链接期望 `core_mem_*`。

### 1.2 错误修复路径（已证伪）

| 尝试 | 为何无效 |
|------|----------|
| 仅在 `rt_run_compiler_parsed.from_x.c` 设 `current_codegen_prefix` | 产品 `-x -E` **不走** `driver_run_compiler_parsed`，走 `driver_run_x_emit_c` |
| 假定 `driver_c_frontend_smoke` 做 codegen | `SHUX_NO_C_FRONTEND` 下 smoke 被 `#if` 裁掉；且 smoke 本身不做 codegen |
| 入口 inherit `current_codegen_prefix` | pipeline **先** codegen 各 dep，会把 `current_*` 写成最后 dep 前缀，入口被污染 |

### 1.3 真正数据流（权威）

```
main.x (-x -E) 
  → driver_run_x_emit_c          # seeds/runtime.from_x.c
  → pctx_e 钉 entry 前缀
  → shux_pipeline_run_x_pipeline
  → codegen_x_ast(dep_i)         # 各 dep 写 current_codegen_prefix = dep 前缀
  → codegen_x_ast(entry, -1)     # 必须读 entry_module_import_path_*，不能读 current_*
  → emit_func / 同模块 CALL
```

库模块 `.o`：`shux_compile_std_module.sh` 优先 `-o`，失败回退 `-x -E`；符号面须与 import 调用端 `core_mem_*` 一致。

用户程序（`tests/mem/main.x`）：**co-emit** 会把 `core.mem` 函数体写进用户 C；若 dep 侧 skip 掉 `placeholder`，用户 `.o` 缺该符号且 on-demand 因其余 `core_mem_*` 已定义而不链 `mem.o` → 仅 `placeholder` undefined。

### 1.4 根因列表（已修）

1. **`-x -E` 未设入口前缀**  
   修：`driver_run_x_emit_c` 与 `driver_run_compiler_parsed` 均写：
   - `current_codegen_prefix_mirror/len`
   - **`entry_module_import_path_mirror/len`**（钉死，dep 不可覆写）

2. **`codegen_x_ast(entry)` 清空/误用前缀**  
   修：`dep_index < 0` 时用 `entry_module_import_path_*` 填 `prefix_buf`。

3. **同模块 CALL 只认 dep 池路径**  
   修：`codegen_emit_prefix_len_from_ctx` + 同模块 CALL 路径统一走该函数；入口优先 entry pin。

4. **`placeholder` 全局 skip**  
   修：`prefix` 以 `core_mem_` 开头时 **不** skip 裸名 `placeholder` / `string_new`（仍 skip `core_types_` / `std_string_` 的弱桩场景）。

5. **局部类型 `struct String` 无前缀**  
   修：user struct 的 `emit_type` 入口也走 `codegen_emit_prefix_len_from_ctx`（入口读 pin）。
