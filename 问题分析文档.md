### 问题 A（hello 重载 mangle）— **已修**

**现象**：`std_fmt_print(msg, 12)` 裸名 → 应为 `std_fmt_print_u8_ptr_i32`。

**真根因**：METHOD_CALL 在 typeck 已 resolved 的快速路径里调用  
`codegen_emit_func_link_name(out, **arena**, dep_mod, func_ix)`，  
`arena` 是**入口** arena，而 param type_ref 属于 **dep** arena → 后缀为空 → 退化为裸名。  
声明侧用 dep_arena，所以 extern 有 mangle、调用点无。

**修复**：对齐 `codegen.x`，resolved 路径使用 `pipeline_dep_ctx_arena_at(ctx, dep_ix)`。

### 问题 D（io_driver Buffer vs ptrdiff_t）— **已修**

**现象**：`std_io_driver_submit_read(Buffer, …)` 与 `ptrdiff_t` 形参冲突。

**真根因**：seed 的 METHOD_CALL 缺 `codegen.x` 已有的  
`codegen_emit_io_driver_buf_call_name` + `(intptr_t)(void*)&arg` 路径；  
且 seed 字节串误为 `shu_io_*_buf`（缺 `x`）。

**修复**：回写 seed，符号改为 `shux_io_submit_read_buf` 等。

### 问题 B（stdlib-import incomplete Result）— **已修（preamble 权威）**

**现象**：`struct core_result_Result_i32` / `Result_u8` 不完整类型。

**真根因（双权威漂移）**：

1. 产品路径 `SHUX_RT_PREAMBLE_FROM_X` 使用 `rt_preamble.from_x.c`，原先**只有** `Option_i32`，**没有** `Result_*`。  
2. `runtime.from_x.c` 的 `write_io_net_abi_inline` 有 `Result_i32`，但该路径被 `#if !SHUX_RT_PREAMBLE_FROM_X` 裁掉。  
3. multi-dep 时 `core.types` 上有同名幽灵 layout → forward 变成 `core_types_Result_*`，真正 `core_result_*` 完整定义发不出来。

**修复**：在 `rt_preamble.from_x.c` 写入 `Result_i32` / `Result_u8`（与 runtime 布局一致），codegen 对二者 abi-dup skip。

> 备注：owner「优先有字段」已写入 seed/codegen.x，但 multi-dep 下幽灵 layout 仍需 preamble 兜底；后续应消灭 core.types 幽灵 layout（parser/typeck 污染）。

## 3. 剩余问题

### 问题 C：bstrict `std/process` libc 符号冲突（**当前最高优先级**）

**现象**：

```text
shux_compile_std_module.sh: cc -c failed for ../std/process/mod.x
conflicting types for 'getcwd'; have 'int32_t(uint8_t *, int32_t)'
```

同类还有 `pipe` / `exit` / `chdir` / `setenv` 等（unistd / stdlib 声明 vs 裸 export）。

**根因**：

- 脚本约定：`mod.x` 应产出 `std_<module>_*` 前缀符号。  
- 实际 `-x -E ../std/process/mod.x` 产出**裸名** `getcwd` / `pipe` / …  
- preamble 已 `#include <unistd.h>` → **编译期** 与 libc 原型冲突。  
- 现有 `objcopy --redefine-sym` 只覆盖 `free/open/close/malloc/realloc/calloc`，且发生在 **cc 成功之后**，救不了编译期冲突。

**修复方向（按根源优先）**：

1. **权威**：entry 为 `std/*/mod.x` 时强制 path lib 前缀（`std_process_`），使 -E 不再裸名。  
2. 短期：在 `shux_compile_std_module.sh` 于 cc 前对生成 C 做安全改名（勿伤 `process_getcwd_c`），并把 clash 列表扩到 `getcwd/pipe/exit/chdir/...`。  
3. 真前缀收敛后可删 objcopy 段（脚本已注释说明）。

## 5. 一句话总结

**产品四件套（return-value / option / stdlib-import / hello）已在 Ubuntu 金标准冷 seed 路径下 -o 全绿。**  
根因分别是：METHOD_CALL 用错 arena、driver buf 符号拼写、产品 preamble 缺 Result_*。  
**挡住 bstrict 的是 std 模块 entry 仍产出裸 libc 名（getcwd/pipe/…），与 unistd 编译期冲突**——需从 entry lib 前缀或 compile 脚本根源收敛，而不是只扩 objcopy 列表。
