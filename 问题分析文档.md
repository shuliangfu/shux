## 本轮根因与修复

### C（原 getcwd 符号冲突）→ 扩大为 std 库模块构建链

1. **entry `-x -E` 截断**（4KiB 对齐、半个函数体）  
   - 现象：process/string/time 等库模块 `-E` 输出不完整。  
   - 对策：`shux_compile_std_module.sh` **优先 `-o` + 直接 .o**；失败再 KEEP_C / -E。  
   - process 特例：`runtime_process_args_thin.from_x.c` + glue（不依赖 -E）。

2. **libc 裸名 / Arena64 双标签**  
   - Arena64：`#define heap_libc_Arena64 std_heap_libc_LibcArena64`。  
   - process 不再进 -E，避免 getcwd 与 unistd 冲突。

3. **脚本改动触发全量 std 重建**  
   - Makefile：compile 脚本改为 **order-only** 依赖。

4. **heap 别名**  
   - 始终补 `std_heap_libc_heap_alloc_c` → `heap_alloc_c`（http.o 需要）。
