## 已修复问题（本会话）

### 1. typeck 变量名跨函数泄漏 ✅

**commit**: `cde457c8`

**根因**: parser bug 将函数局部 let 误加入 `module.num_top_level_lets`。typeck 的 `typeck_check_expr_var` 先查 top_level_lets 再查函数参数，导致同名变量被解析为错误类型（如 `msg` 参数 `*u8` 被解析为另一函数的 `msg` 局部 `u8[240]`）。

**修复**: `patch_typeck_gen_lang007.py` 增加 `swap_var_param_order()` — 将函数参数检查移到 top_level_lets 检查之前，确保函数参数优先于误置的 top_level_let。

### 2. typeck implicit_tail_region 处理 ✅

**commit**: `cde457c8` + `de1111b5`

**根因**: seed `typeck_func_body_tail_expr_ref_for_implicit_rule` 缺少 region（unsafe block, stmt_order kind 5/6）处理。当函数体末尾是 `unsafe { return ... }` 时，seed 不识别 region 内的 return，误报 "implicit tail return"。

**修复**: `patch_typeck_gen_lang007.py` 增加 `patch_implicit_tail_region()` — 在 `typeck_func_body_tail_expr_ref_for_implicit_rule` 中添加 region 检查，递归查找 unsafe region 内部的 return 表达式。

### 3. `unsafe { return ... }` 模式全面重构 ✅

**commit**: `9144629e` + `98522ece` + `7c1ed586`

**根因**: parser 将 `unsafe { return ... }` 解析为 EXPR_BLOCK，但 `block_ref` 字段未设置，导致 typeck 的 EXPR_BLOCK 递归检查无法找到内部 block 的 return 表达式。

**修复**: 将所有 std/*.x 中的 `unsafe { return expr; }` 模式重构为：
- 多行格式：`let _rc: T = 0; unsafe { _rc = expr; } return _rc;`（188 函数，29 文件）
- 单行格式：`unsafe { return expr; }` + `return 0;`（618 处，39 文件）

### 4. `shux_compile_std_module.sh` -E 回退 ✅

**commit**: `4fb67eaf`

**根因**: `-x -E` 路径使用 .x pipeline typeck（seed），有 unsafe { return } bug。`-E` 路径使用不同 typeck 路径。

**修复**: `-x -E` 失败时回退到 `-E`（不含 `-x`）。

### 5. `shux_compile_std_x.sh` 使用 -E+cc ✅

**commit**: `d2f13781`

**根因**: `shux-c -o` 使用 ASM backend，对含指针运算/数组的 .x 文件失败。

**修复**: `shux-c` 分支改为 `-E + cc -c`。

## 当前阻塞问题：`std/net/tcp.x` C 编译错误

### 现象

`run-all-bstrict.sh` 在 `std/net/net.o` 编译阶段失败：

```
../std/net/tcp.o.gen.c:209:18: error: implicit declaration of function 'socket' [-Wimplicit-function-declaration]
```

生成的 C 代码调用 `socket()` 但缺少 `#include <sys/socket.h>`。

### 根因方向

`tcp.x` 中 `socket()` 是 extern function，codegen 生成的 C 代码使用裸名 `socket`，但没有对应的 `extern` 声明或 `#include`。`shux_compile_std_x.sh` 的回退路径只添加 `#undef htonl/htons/ntohl/ntohs`，不添加 `#include <sys/socket.h>`。

### 下一步

1. 检查 `tcp.x` 中 `socket` 的 extern 声明是否正确
2. 在 `shux_compile_std_x.sh` 回退路径中添加 `#include <sys/socket.h>` 等 network 头文件
3. 或修改 `tcp.x` 将 `socket` 重命名为避免与系统函数冲突的名称
