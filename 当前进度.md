## 本波次：W-extern-unsafe ✅

**根因**：LANG-007 要求 extern 在 `unsafe` 内；C 路径 `-E` 本就调用 `typeck_set_allow_legacy_extern_calls(1)`，但：

1. `typeck_set_allow_legacy_extern_calls` 仅有 weak stub（不存 flag）  
2. X driver 的 `-E` 走 `driver_run_x_emit_c`，**从未** set allow_legacy  
3. 边界检查 `pipeline_typeck_check_extern_call_unsafe_boundary_c` 不读 flag  

**修复**（边界表 + 冷启动路径，非逐文件 800 处 wrap）：

1. `patch_typeck_gen_lang007.py`：强符号 set/get + `typeck_check_expr_call` 在 allow 时跳过 S0 边界  
2. `seeds/runtime.from_x.c`：`driver_run_x_emit_c` 入口 set(1)、出口 restore  
3. `pipeline_glue.c`：boundary 尊重 `typeck_get_allow_legacy_extern_calls`（standalone 链）  

**纪律**：非 `-E` 仍强制 `extern requires unsafe`（用户代码 / 默认编译路径不变）。

**解锁**：compile / lsp / lsp_diag / typeck 等 -E 地图 typeck OK；T 9→14。
