## 未解决问题

### 问题 1：`shux-c -E src/runtime_driver_diagnostic.x` typeck 仍失败

**现象**: 冷启动重建后，独立 `-E` 模式仍报：
```
typeck error: assignment type mismatch: expected *u8, found ?
typeck error[XT001]: .x type check failed in function #90 driver_diag_report_prefixed (check_block failed)
```

**调试进展**:
- `swap_var_param_order()` 将函数参数检查移到 top_level_lets 之前 — 错误从 `found [240]u8` 变为 `found ?`（参数查找返回 null）
- `SHUX_DEBUG_VAR` 调试显示 `pipeline_module_func_param_type_ref_for_name` 对 `fix=90, vnlen=3` 返回非零值 `pr=499` — **参数查找本身是成功的**
- 但 typeck 仍报 `found ?` — 说明 `msg` 变量的 resolved_type_ref 未被正确设置
- **待排查**: `let m: *u8 = msg;` 中的 `msg` 表达式 kind 是否为 EXPR_VAR（kind=3）；`typeck_check_expr_var` 的返回值是否被正确传播到 `typeck_check_block_one_let` 的 `init_ty`

**复现**:
```bash
cd /home/shuliangfu/worker/shu/shux/compiler
SHUX_DEBUG_VAR=1 ./shux-c -E src/runtime_driver_diagnostic.x 2>&1 | grep 'DBG-VAR.*fix=90'
# 输出显示 pr=499（非零），但 typeck 仍失败
```
