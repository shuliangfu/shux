## 本次修复（2026-07-14）

### 问题 1：`expected *u8, found ?` — ✅ 已修复

**原文档误诊**：以为 `let m: *u8 = msg` 形参类型未写入。  
**真实根因**：`m = ""` / `m2 = ""` 的 **STRING_LIT（kind=59）** 在 typeck 中未定型，assign 路径报 `found ?`。

**数据流**：
```
m = ""
  → ASSIGN → typeck_check_expr_assign（seed）
  → typeck_check_expr(RHS, expected=*u8)
  → seed check_expr_impl 原无 kind=59 分支 → resolved=0
  → assign：lt=*u8、rt=null → expected *u8, found ?
```

**修复**：
1. `patch_typeck_gen_lang007.py`：`patch_string_lit_dispatch` — seed `typeck_check_expr_impl` 增加 kind=59；有期望类型则回填，否则 `u8[]`
2. `pipeline_glue.c`：assign 路径 string_lit 回填 + `pipeline_typeck_check_expr_impl_c` 分派（全量 glue / 非 strict_minimal 路径）
3. 常量 `GLUE_EXPR_STRING_LIT_ORD 59` 与 parser_asm / backend 对齐

**验证**：
```bash
./shux-c -E src/runtime_driver_diagnostic.x   # exit=0
# 最小：function f(m: *u8): void { m = ""; }  → exit=0
```

### 问题 2：`socket` 隐式声明 — ✅ 已修复

**原文档方向**：加 `#include <sys/socket.h>` 或改名 `shux_socket`。  
**真实根因**：参数名 **`type` 是 .x 关键字**，`extern "C" function socket(..., type: i32, ...)` 声明未正确进入模块，codegen **不发射** `extern int32_t socket(...)`。

**修复**：
1. `std/net/tcp.x` / `udp.x` / `ipv6.x`：`type` → `sock_type`
2. `tcp.x`：`if (unsafe { net_set_blocking_c(...) } != 0)` 改为 `let rc; unsafe { rc = ... }; if (rc != 0)`（表达式 unsafe 的 codegen 会生成 void 与 int 比较）

**验证**：
```bash
./shux-c -E -L .. ../std/net/tcp.x | grep 'extern int32_t socket'
# extern int32_t socket(int32_t domain, int32_t sock_type, int32_t protocol);
bash scripts/shux_compile_std_x.sh ./shux-c ../std/net/tcp.x /tmp/tcp.o  # .o 生成成功
```
