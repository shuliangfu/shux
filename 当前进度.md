### 修复路径

1. **panic 函数名** `shulang_panic_` → `shux_panic_`（commit `52c13961`）
2. **panic noreturn codegen** — EXPR_PANIC (ord=42) 跳过 `return` 前缀（commit `4f2b7bc1`）
3. **EXPR_BLOCK 递归** — `pipeline_glue.c` 中 `pipeline_typeck_func_body_has_implicit_return_tail_c` 加 EXPR_BLOCK (ord=26) 递归（commit `4f2b7bc1`）
4. **Ubuntu 生效** — 从 macOS scp `codegen_gen.c` + 重建 `codegen_x.o` + `pipeline_x.o`

### Ubuntu 重建步骤（codegen 修改后必须执行）

```bash
# macOS 端
scp compiler/codegen_gen.c ubuntu-server:~/worker/shu/shux/compiler/codegen_gen.c

# Ubuntu 端
cd ~/worker/shu/shux/compiler
make codegen_x.o
rm -f pipeline_x.o && cc -I. -Iinclude -Isrc -DSHUX_USE_X_DRIVER -DSHUX_USE_X_PIPELINE -DSHUX_USE_X_TYPECK -DSHUX_USE_X_CODEGEN -c pipeline_gen.c -o pipeline_x.o
```
