## 本轮新完成的工作（Windows 编译链打通）

### 关键里程碑：Windows 上 `shux-c.exe hello.x -o hello.exe` 成功输出 "Hello World" ✅

macOS + Windows 双平台一致通过，无 regression。

### 1. Windows fork 调用全面替换为 _spawnvp（commit 534bc2cc, 05afeb53）

| 文件 | 替换点 |
|------|--------|
| `compiler/src/runtime.c` | 2 处 fork → `_spawnvp(_P_WAIT, ...)` |
| `compiler/src/runtime_link_abi.c` | 5 处 fork → `_spawnvp(_P_WAIT, ...)`，其中调用 `cc` 编译和 `ld` 链接的分支在 Windows 改用 `gcc` |

POSIX 路径保留 `fork/execvp`，Windows 路径用 `_spawnvp` 同步等待，避免 fork 不可用导致的子进程失败。

### 2. C 前端字符串字面量全链路（commit 0f6ce4f4）

shux 语言原本只有 `let s: *u8 = "..."` 在 AST 层支持，C 前端 parser/typeck/codegen 都没有处理 AST_EXPR_STRING_LIT。

| 文件 | 修改 |
|------|------|
| `compiler/src/parser/parser.c` | `parse_primary` 加 TOKEN_STRING 分支：解码 lexer 存入 str_buf 的内容，strndup 到 ast，构造 AST_EXPR_STRING_LIT 节点 |
| `compiler/src/ast/ast.c` | `ast_expr_free` 加 AST_EXPR_STRING_LIT 分支：释放 `string_lit.bytes` |
| `compiler/src/typeck/typeck.c` | `typeck_expr_sym`：resolved_type = `*u8`；`get_expr_type`：返回 AST_TYPE_PTR + elem=u8；`is_const_expr`：加入 case |
| `compiler/src/codegen/codegen.c` | 加 AST_EXPR_STRING_LIT case：生成 C 字符串字面量，转义 `" \ \n \r \t \0` 与非可见字节 |

### 3. typeck 整数隐式转换修复（commit 0f6ce4f4）

`typeck_integer_widen_ok` 的 i32 分支原本只允许升格到 i64/u32/usize/u64，缺少 isize。`std/io/stubs.x` 中 `return -1` 在返回 isize 的函数里报 "type mismatch"。

修复：i32 分支增加 `dest == AST_TYPE_ISIZE`。

### 4. hello.x 调用 API 修正（commit c12f26ac）

原 `fmt.println("Hello World")` 只传 1 参，但 std.fmt 的 println 重载需要 `(ptr: *u8, len: i32)` 或 `(s: u8[])`。改用 `fmt.print(msg, 12)` 直接匹配。

### 5. WSAStartup 参数类型修复（commit 71c0d5e9）

`std/io/win32.x` 中 `WSAStartup(wVersionRequested: u16, ...)` 用 u16 类型，但 shux 无 u16 关键字（u16 是 NAMED 类型），typeck 不允许 i32→u16(NAMED) 隐式转换，导致 `WSAStartup(514, &wsa[0])` 报 "no matching overload"。改为 `wVersionRequested: i32`。

### 6. shux-c -backend 参数修复（commit b116423c, c010442d, d3e96a32）

**根因**：shux-c（SHUX_LEGACY_C_FRONTEND=1 构建，无 SHUX_USE_X_PIPELINE）不支持 `-backend` 参数（`runtime.c:1219` 报 "build error[BLD001]: -backend asm not available"）。但 `bootstrap-link-shux.sh`、`run-ub.sh`、`p0-gate-shux.sh` 三处对 shux-c 强制 `-backend asm`/`-backend c`，导致 run-ub.sh 和 C5 gate 在所有平台都 FAIL。

**修复**：
- `tests/lib/bootstrap-link-shux.sh`：只对 `shux_asm`/`shux_asm2`/`shux_asm_stage1` 加 `-backend asm`，shux-c 不加
- `tests/run-ub.sh`：SHUX_LINK_BACKEND_ARGS 为空时不强制 `-backend c`
- `tests/lib/p0-gate-shux.sh`：`p0_gate_try_run_o` 对 shux-c 不加 `-backend c`
