### Bug 修复链（按修复顺序）

#### 1. bug ③ dep 前缀模型（commit c28d83e9 + db201393）

- **根因**：codegen.x line 6898/6927 的 `dep_index < 0 &&` 条件阻止 dep 模块 emit 顶层 const/let
- **现象**：dep 模块函数体通过 EXPR_VAR 裸名引用同模块 const，但 const 定义受 dep_index<0 限制不 emit → 引用未定义符号
- **修复**：去掉 dep_index<0 条件，dep 模块 const 以裸名 emit（与 EXPR_VAR 一致，不加 dep 前缀）
- **验证**：macOS arm64 + Linux x86_64 双平台 main3.x import 测试返回 42

#### 2. INDEX 步长 bug（commit fae16e5b）

- **根因**：`pipeline_asm_index_elem_byte_sz_c`（pipeline_glue.c:9100）通过 `pipeline_expr_resolved_type_ref` 获取基址类型；typeck skip 时返回 0，默认 `return 4`（i32 步长）
- **现象**：`buf[i]`（buf: *u8）按 4 字节步长索引，读取错误内存
- **修复**：typeck skip 时回退到 VAR 声明的 type_ref 推断元素大小
- **验证**：NL-03 heap gate 绿

#### 3. shux_sys_open 指针截断 bug（2026-07-09）

- **根因**：freestanding_io_x86_64.s 中 shux_sys_open 有 `mov %edi,%edi`（zero-extend 32-bit → 64-bit），但 open 首参是 path:*u8（64 位指针），该指令将高 32 位清零导致 EFAULT
- **修复**：仅 shux_sys_open 删除该指令（其他 stub 首参是 fd:i32，`mov %edi,%edi` 零扩展正确）

#### 4. 栈布局 bug —— dep struct slot size（commit b310bf0c）

- **根因**：`asm_local_slot_bytes_mod`（ast_pool.c:10532）对 dep 模块 struct（PageMmapHeap）查不到 layout，返回默认 8B（实际 24B）
- **现象**：struct 末字段 off 越界覆盖相邻局部变量 path 数组（h.off 写入 path[0] 使 open("\20") ENOENT → exit 5）
- **修复**：添加 pipeline_asm_emit_dep_pipe_c + pipeline_dep_ctx_ndep/module_at 遍历 dep 模块 struct layout

#### 5. 栈布局 bug —— 名称前缀不匹配（commit 45f97d71）

- **根因**：type name 带模块前缀 "heap.PageMmapHeap"（len=17），struct layout name 是裸名 "PageMmapHeap"（len=12），`ln != nlen` 直接 continue
- **现象**：dep 模块遍历命中 nsl=1 但 sz=0（名称不匹配），栈槽仍算 8B
- **修复**：`asm_slot_bytes_named_in_mod` 比较前取 type name 最后一个 "." 之后的部分
- **验证**：NL-04 fs gate 绿，NL-02~07 全绿

### 关键技术发现

#### let 声明栈槽分配链

```
asm_local_slot_reg_offset（ast_pool_bootstrap_glue.c:214）
  → asm_local_slot_bytes → asm_local_slot_bytes_mod（根据 type_ref 算变量槽）
  + pipeline_asm_let_init_stack_reserve_bytes → asm_struct_lit_reserve_stack_bytes（根据 init_ref 算 temp 区）
```

- `asm_local_slot_bytes_mod`：kind==8（TYPE_NAMED）时查模块 struct layout
- typeck skip 时入口模块无 dep struct layout，须遍历 dep_ctx
- type name 带模块前缀，struct layout name 是裸名，比较时必须去前缀

#### ast_pool.c 编译路径

- `pipeline_glue.c:19028` 通过 `#include "ast_pool.c"` 引入
- Linux：编译进 pipeline_x.o（pipeline_gen.c #include pipeline_glue.c）
- macOS：编译进 pipeline_glue_standalone.o（pipeline_glue_standalone.c #include pipeline_glue.c）
- 链接时用 `--allow-multiple-definition`，standalone/glue 在末尾覆盖 pipeline_x.o 旧符号

#### typeck skip 机制

- import 程序用 `-o` 模式编译时，`runtime.c:3817-3819` 调用 `driver_x_pipeline_skip_typeck_set(1)` 跳过 .x typeck
- 原因：.x typeck 对 import 程序 SIGSEGV
- 副作用：dep 模块 struct layout 未填充，需 ASM 后端自行遍历 dep_ctx 查找

### P0a/P0b extern "C" 语法（2026-07-08）

- P0a：parser 识别 `extern "C"` / `extern "X"` / `extern`（默认）三种语法
- P0b：381 个 C ABI 声明标注 `extern "C"`（跨 47 个 .x 文件），2479 个 `extern function` 保持默认 X ABI
- X ABI 路线图：P0a（parser）→ P0b（源文件标注）→ P1（自举后激活 typeck 契约）→ P2（codegen 优化）
