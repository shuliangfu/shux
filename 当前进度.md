### 问题 1：`shux-c -E` 独立模式 typeck 失败 — 变量参数类型未设置

**状态**：❌ 未修复

**现象**：
```bash
cd /home/shuliangfu/worker/shu/shux/compiler
./shux-c -E src/runtime_driver_diagnostic.x 2>&1 | head -3
# typeck error: assignment type mismatch: expected *u8, found ?
# typeck error[XT001]: .x type check failed in function #90 driver_diag_report_prefixed (check_block failed)
# pipeline error[XP003]: .x pipeline failed for 'src/runtime_driver_diagnostic.x'
```

**根因分析**：
- 函数 `driver_diag_report_prefixed(msg: *u8)` 中 `let m: *u8 = msg;` 的 `msg` 变量类型未被解析
- `SHUX_DEBUG_VAR` 调试显示 `pipeline_module_func_param_type_ref_for_name(module, 90, "msg", 3)` 返回 `pr=499`（非零）—— **参数查找成功**
- 但 typeck 仍报 `found ?`（resolved_type_ref = 0）—— **参数类型未被写入 expr 的 resolved_type_ref**
- `swap_var_param_order()` 已将函数参数检查移到 top_level_lets 之前，错误从 `found [240]u8`（跨函数泄漏）变为 `found ?`（未解析）

**待排查**：
1. `let m: *u8 = msg;` 中 `msg` 表达式的 kind 是否为 EXPR_VAR（kind=3）
2. `typeck_check_expr_var` 返回 0 后，`pipeline_expr_set_resolved_type_ref` 是否被正确调用
3. `typeck_check_block_one_let` 中 `init_ty = typeck_expr_type_ref(arena, ld_ir)` 是否读到正确的 resolved_type_ref
4. 是否存在 C delegate `pipeline_typeck_check_expr_var_c` 覆盖了 seed 的 `typeck_check_expr_var`

**关键文件**：
| 文件 | 位置 | 说明 |
|------|------|------|
| `compiler/src/runtime_driver_diagnostic.x` | L447 | `driver_diag_report_prefixed(msg: *u8)` 函数 |
| `compiler/scripts/patch_typeck_gen_lang007.py` | `swap_var_param_order` | seed patch（参数检查顺序） |
| `compiler/seeds/typeck_gen.linux.x86_64.c` | L3186 | seed `typeck_check_expr_var` 定义 |
| `compiler/ast_pool.c` | L1331 | `pipeline_module_func_param_type_ref_for_name` |
| `compiler/pipeline_glue.c` | L22062 | C 侧 `pipeline_typeck_check_expr_var_c` |

**复现**：

### 问题 2：`std/net/tcp.x` C 编译缺少 `socket()` 声明

**状态**：❌ 未修复

**现象**：
```
../std/net/tcp.o.gen.c:209:18: error: implicit declaration of function 'socket' [-Wimplicit-function-declaration]
```

**根因**：
- `tcp.x` 中 `socket()` 是 extern function，codegen 生成裸名 `socket`
- `shux_compile_std_x.sh` 的 `-E + cc -c` 回退路径只添加 `#undef htonl/htons/ntohl/ntohs`，不添加 `#include <sys/socket.h>`
- cc 编译时找不到 `socket()` 声明

**待修复方向**：
1. 在 `shux_compile_std_x.sh` 的 cc 编译前添加 `#include <sys/socket.h>` / `<netinet/in.h>` / `<arpa/inet.h>` 等 network 头文件
2. 或修改 `tcp.x` 将 `socket` 重命名为 `shux_socket` 避免与系统函数冲突

**关键文件**：
| 文件 | 说明 |
|------|------|
| `compiler/scripts/shux_compile_std_x.sh` | shux-c 分支的 `-E + cc -c` 路径 |
| `std/net/tcp.x` | `socket()` extern 调用 |
