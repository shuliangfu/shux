# 标准库按需深化（P3，非主线）

与根目录 [NEXT.md](../../NEXT.md) 一致：在自举减 C（P1/P2）稳定后，按**编译器依赖**与**已有 run-*.sh** 小步加深，不批量增加仅占位 `import` 的模块。

## 当前优先级

| 优先级 | 模块 | 测试 | 说明 |
|--------|------|------|------|
| 高 | `std.io` / `std.io.driver` | `run-io.sh`、`run-io-driver.sh` | 编译器与 LSP 路径依赖 |
| 高 | `std.fs` | `run-fs.sh` | 多文件与 `-L` 解析 |
| 高 | `std.vec` | `run-vec.sh` | 动态数组；与 typeck/codegen 数组路径相关 |
| 中 | `core.fmt` | `run-fmt.sh` | 格式化；CI 不再 SKIP，须保持绿 |
| 低 | `std.http`、`std.async`、`std.simd` 等 | 无强制 | 仅占位 mod，无用户故事不开工 |

## 单模块验收

每个 PR：`mod.su`（+ 可选 `.c`）+ 扩展或新增 `tests/run-*.sh` + 已列入 `tests/run-all.sh`。
